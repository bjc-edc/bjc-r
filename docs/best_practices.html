Best Practices for Coding HTML on BJC4NYC

See also: EDC Template and Style Guide.

This page was drafted up by Mary after losing a morning to troubleshooting HTML errors. There isn't necessarily one right way to tab nested tags in an HTML file, but we should have some way to make consistent and legible code to share with each other. I'm suggesting the way I do it as a coding guideline. Please read (critique if you must) and follow these guidelines. Thanks, :) Mary

  1. Write tidy code with legible formatting:
    • Use line breaks to split up code blocks—not blocks of text;
    • Use tabs to nest tags within tags (i.e. level of nested tags; e.g. look at the code for this page or this simple example);
    • When in doubt, think of our teammates trying to read and troubleshoot the code. :)
  2. Check for good HTML before committing:
    • See that all opened tags are closed (except <img /> and <br /> which have a closure bar at the end)—especially likely to be an issue with a lot of nesting (good tab formatting helps greatly here);
    • Check that the whole page looks right in multiple browsers (can skip for text-only edits, but not for edits that involve formatting).
  3. Follow our Style Guide:
    • Copy the HTML and CSS from the Style Guide or from other instances (less-safe, as it could have been modified)
    • Use adaptations from the Style Guide to add features—not to replicate ones that aren't working. If something from the Style Guide isn't working, fix it, log an issue on GitHub, or get help from someone. Please do not build a non-extensible workaround when something that should be working isn't.
      For example:
      • ex 1: <div><span><strong>Sasha:</strong></span> I see another way...
        In our <div class="dialogue"> tag, the <div> tag is for each new character or stage direction. The <span> tag within one of these <div> tags is designed specifically to put the bold (i.e. "strong") styling on the character's name. If it doesn't, this is a sign that something is wrong with the HTML. In cases like this, please fix the problem or alert someone else to the issue; don't apply the desired formatting manually as the underlying bug could cause problems later.
      • ex 2: <div class="sidenote"> &nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp; What exactly is Tony's <code>map</code> block doing?</div>
        Here, having to insert a whole bunch of spaces to make the sticky note work is a clue that there is a problem. Please fix it, track it, or pass it on.
  4. Use server-friendly filenames. No fancy symbolic characters (like "?"), no caps, and no spaces (use "-" instead). Unlike your computer, THE SERVER IS CASE SENSITITIVE. So, references must match filenames exactly.
    • e.g. instead of "SpaceKeyPressed? 214 × 28 pixels.png" use "space-key-pressed-214x28.png" (note the letter x instead of the times sign)
