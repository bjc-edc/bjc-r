File Naming Conventions

Our server is cAsE SeNSiTIve so even if things look ok on your browser, they might not on bjc.edc.org. The most essential thing is that the case of the reference has to match the case of the file exactly.

Here are some conventions that we should all be using:

  • Image files should be in all lowercase, e.g. hierarchical-3.jpg or map3x+7.png, and they can't have any spaces (use a dash) or weird characters like ×. Exception: abbreviations, e.g., URL-structure.jpg, TCP.png, or GPS-map.png. In all cases, the references must match the filename exactly.
  • Project files should begin with the unit and lab numbers followed by a dash and then have a few Title Case words that describe the project separated by dashes, e.g., U2L3-GuessNumber.xml, U3L2-DrawShape.xml, U4L3-BigNumbers.xml. The title of the project in Snap should be the same as the filename minus the extension, e.g. U2L3-GuessNumber, etc. The references must match the filename exactly.
  • Lab pages should begin with the lab page number followed by a few words that approximate the lab page title, no spaces (use dashes), all lowercase, e.g., 1-shopping-list-app.html, 3-read-web.html, etc.
  • TG pages should follow the same format as lab pages except starting with the lab number rather than the lab page number.

LaTeX

You can now insert inline LaTeX (such as \| f + g \| \le \|f\|+\|g\|) using the following syntax:

<span class="katex">\| f + g \| \le \|f\|+\|g\|</span>

Block-based LaTeX (that renders on its own line) can be used with the syntax class="katex-block". For example:\| f + g \| \le \|f\|+\|g\|

Please do not use the dollar-sign delimiters any more. The former replaces single dollar-sign delimiters and the latter replaces double dollar-sign delimiters.

The name "\KaTeX" is the current LaTeX library, which is developed by Khan Academy.

How to Make a Picture of Blocks

  1. Build the script you want in snap
  2. If you want to make a big picture of a short script, choose "Zoom blocks" from the settings menu and enter "1.5"; otherwise, just leave it at "1" for single line blocks.
  3. Scroll around to locate the script
  4. Right click the script and choose "script pic..."
  5. Save the file to the unit's new image directory, e.g. /bjc-r/img/3-lists or to a shared directory, as appropriate. (If it's a script or a block with inputs that is not likely to be used again, please store the image in the unit image directory; if it's a blank block or a block with default inputs, then it should go in the "blocks" image directory.)

How to Make an Animation 

See https://www.youtube.com/watch?v=MD9wiyBlJ9Y

Or just follow these instructions (be sure to also read about animation controls below):

  1. Make a screen recording (suggested method: Quicktime Player).
  2. Open Photoshop, go to File > Import > "Video Frames to Layers," and select your movie file.
  3. Limit to every 2 or 3 frames depending on how fast your video moves (as we like to say, experiment!).
  4. If needed, show the timeline by going to Window > Timeline.
  5. If needed, change the speed by selecting all frames in your timeline (using shift), clicking the drop-down menu on one of the frames, and setting the frame delay to 0.1 seconds (or whatever looks good).
  6. Make the last few frames transparent (or add a few blank frames) so that there is a pause in the video as it loops.
  7. Go to File > "Save for Web," choose GIF, set the dimensions of the image as you want them, be sure to set it to loop forever, and save it to the image folder of your unit.
  8. Don't forget to actually link to the image... ;)

How to Link to a Project

  1. Build the script(s) you want in snap If the project contains custom blocks, be sure to not just build them, but leave them up on the stage. Opening a project that looks empty with a custom block buried in the palette has created confusion for folks.
  2. Choose "Export project" from the file menu
  3. Save the XML file to the unit's PROG directory, e.g. /bjc-r/prog/3-lists
  4. In the lab page HTML file where you want to link, place a picture of the script(s) wrapped with an "a href" tag linking to the XML file with class="run" in the "a href" tag. For example:

    Note that clicking the link starts a brand new project! Any work that the student has done prior to this link will NOT be available!!
Principles for when to include a script:
  • Early on, and for simple scripts (and minimal or no typing), getting them to build is useful practice.
  • For complicated scripts that we want them to test and/or modify, let them load it. Saves time and guarantees they're testing what we actually intended.
  • For lists with lots of typing, let them load initial versions, especially for getting started or experiments. Again, reduces overhead. Also, they can always tailor if they want.

From Jens: you can use #cloud to open a shared project in edit mode without running it right away.

There are also three switches you can add to the #present: url to get more control over what will be displayed:

  1. &editMode: this will also NOT switch to presentation mode, but it will still run the project right away.
    • http://snap.berkeley.edu/snapsource/snap.html#present:Username=jens&ProjectName=Mesmerize&editMode
  2. &noRun: this switch will prevent the project from running immediately (it won't fire the green flag event).
    • You can use it together with other switches, e.g. http://snap.berkeley.edu/snapsource/snap.html#present:Username=jens&ProjectName=Mesmerize&editMode&noRun.
    • but also alone: http://snap.berkeley.edu/snapsource/snap.html#present:Username=jens&ProjectName=Mesmerize&noRun
  3. &hideControls: this switch hides the green-flag, pause, stop, and edit/presentation (useful for "hiding" the code) buttons. This is useful when embedding Snap projects into other web pages using iFrames. You can also use this in combination with other switches, but it makes the most sense in presentation mode.
    • http://snap.berkeley.edu/snapsource/snap.html#present:Username=jens&ProjectName=Mesmerize&hideControls

How to Link to a Specfic Part of a Page Using Anchors

There are two parts of a link with an anchor (the link and the anchor):

  1. First, place the anchor on the target page:
    • <a name="picture" class="anchor">&nbsp;</a>
    • Put that &nbsp; (non-breaking space) in there, and use the whole thing (opening tag, &nbsp; and closing tag) as a space, replacing some space in the line your want to link to.
  2. Then, create a link, just like usual except add a hashtag and the anchor name to the end of the link:
    • For example: <a href="/bjc-r/docs/style_guide.html#picture">This would link to the "How to Make a Picture of Blocks"</a>, section above. Try it!

The #moveturn in the reference (a href) scrolls to that anchor (a name). (The anchor class is a workaround to scroll past the heading at the top of each lab page; it goes in the "a name" only; we hope to fix that one day.)

Dreamweaver will maintain these links over time if you move and rename files in DW. Otherwise if you move or rename a files (outside DW) you will have to fix all the links yourself.
