This lab serves as introduction to course. After a quick video from the curriculum developers, the class shares their experience and interest in computer science and then explores a simple offline activity designed to introduce the basics of how computers process data in functional programming. Students then create Snap! accounts and learn about the Snap! programming language, interface, and terminology (reporter blocks, input parameters) by using various blocks that greet the user in different ways and changing the input parameters to explore the impact on outputs.

In this course, the curriculum will provide students with starter projects for them to save to their accounts. It's also possible to create new projects, and students may wish to explore that on their own time.

Pacing

This lab is designed for 3–5 class periods (120–200 minutes).

  • Activity 1: BJC Culture and Norms 35–65 minutes
  • Activity 2: Draw Like a Computer approximately one class period
  • Activity 3: Creating a Snap! Account 15–30 minutes
  • Activity 4: Say Hello to Snap! 10–20 minutes

Activity 2: Draw Like a Computer. 

  • Materials:
    • Draw Like a Computer student page
    • Each student will need several pieces of paper and a writing implement.
  • Prepare:
    • Watch Josh Darnit's videos Peanut Butter and Jelly and Exact Instructions Challenge to appreciate the spirit of the activities; you'll be doing the drawing activity in class today.

      • This is a re-imagining of the CS Unplugged activity called "Marching Orders" done as a functional-programming activity.

      As you can see, the key is that the person doing the drawing tries to mess up as much as possible while still following the directions specified. For example, if the programmer (the student creating the instructions) says to draw a square, the computer (the student drawing) could draw a very tiny square if the programmer didn't specify how big it should be.

  • Learning Goals:
    • Appreciate that you need precision to be able to describe something to a computer.
    • Appreciate that functions take inputs, process them in various ways, and produce an output.
  • Activity Plan:
    1. Work together as a class before students open their computers.
      1. Explain the activity, describing the roles of programmer (provide a list of instructions that is as precise as possible for the computer to replicate some original drawing) and computer (deviate as much as possible from the programmer's intentions while still following the instructions precisely).
      2. Ask the students to take out a piece of paper out and play the role of the computer. Read the following instructions and ask students to follow them, trying to mess them as much as possible while still doing what exactly what they say.
        • Draw a square.
        • Draw a circle on top of that.
        • Draw a triangle on top of that.
        3. Here's an example of what might happen.

        Programmer's Original Drawing

        Two Possible Computer Drawings

Activity 3: Creating a Snap! Account 

  • Materials: Creating a Snap! Account student page
    OLD: Creating a Snap! Account Google Doc
  • Prepare:
    • Before leading the activity with students, create your own Snap! account (if you haven't already) and customize your own settings.
    • You may find it helpful to review the beginning of the Beauty and Joy of Computing: Snap! Crash Course and perhaps also to familiarize yourself with the structure of the Snap! Reference Manual, which is accessible by clicking the Snap! logo in the top left corner of any Snap! window.
  • Learning Goals:
    • Create a Snap! account.
    • Learn to navigate the Snap! interface.
    • Customize the Snap! programming environment.
  • Activity Plan:
    1. Students create a Snap account.
    2. Students then customize several basic Snap! settings (language and appearance).
  • Tips:
    • During the signup process, new users are asked to click a link in their email to verify the account, but that doesn't have to happen right away.
    • Snap! users can change their password at any time using the Cloud menu.
    • Users can tell whether they are logged in by checking the Snap! Cloud menu:
      • If it says "Logout" and then a user name, then that user is logged in.
      • If it says "Login..." then no one is logged in.
      If students are using shared computers, they should log out of Snap! when they are done.
    • Make note of any questions or difficulties that arise, and please share them with the curriculum developers.
  • Standards:
    • None covered in this activity since it focuses on getting set up with Snap!.

Activity 4: Say Hello to Snap!. 

  • Materials: Say Hello to Snap! student page
  • Prepare:
    • Take a few minutes to explore the blocks in the Snap! project so you understand what they do before you share the Google doc with students.
  • Learning Goals:
    • Understand that a block is a piece of code that can be run by a computer and it can be run by clicking it.
    • Understand that reporter blocks are blocks that report outputs.
    • Understand that block can have inputs, those inputs can change the output, and the output of one block can be used as the input to another.
    • Explore changing inputs by typing different text into the input slots, by selecting different dropdown menu options, and by dragging other blocks into the input slots.
  • Activity Plan:
    • Students explore two blocks:
      • , which takes a name and a language as input and reports a greeting to that person in the selected language.
      • , which takes text as input and reports the same text with any letters changed to fancy Unicode letters
    • Students then nest the fancy block inside the say hello block and observe how the input of fancy becomes the input to say hello.
    • Finally, students learn to save their project to the Snap! cloud, as they should every day and periodically while working.
  • Tips:
    • In this first hands-on exploration with Snap!, students may struggle with basics such as clicking a block to make it run, difficulty typing into input slots etc. The simplicity of this project allows for debugging these simple challenges of the absolute beginner so that they are resolved by the time students move on to more complex use of the Snap! interface such as dragging blocks into each other, creating input parameters, and creating and editing blocks.
  • Standards:
    • None covered in this activity since it focuses on the basics of using Snap!.
