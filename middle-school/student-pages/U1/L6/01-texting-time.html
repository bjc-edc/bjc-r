Solutions Worksheet Students may word things differently and still have a correct description. The lowercase block turns every uppercase English letter into a lowercase letter, and does not affect other letters. The songify block adds random music emojis at the start and end of the text. The emojify block replaces particular words with corresponding emojis, and leaves the rest of the text unchanged. The clapify block inserts a clap emoji between each of the letters in the text. The binarize block replaces each letter with the binary number that represents that letter on the computer. That's according to the ASCII/Unicode encoding, but students do not have to understand that. The mystery block keeps only the letters from the text that are English consonants. In this activity, you’ll transform text messages into a rich emoji-filled conversation.

Styling Your Text Messages

Everyone has their own style of writing text messages—some all lowercase, SOME ALL UPPERCASE, some filled with emojis 🎉🤩🐱!

In this project, you'll use reporters to add style to some text messages.

  1. Open this Texting Time project, and save it to your account.

All of the blocks take text as input, so you can just type in anything on your keyboard. For example, the uppercase block capitalizes all the English letters in the text:

If you click that block in the project, you should see "BE MORE ASSERTIVE!"

  1. Type different text in each of the other blocks and see what happens. Write some messages that you might actually write in your actual conversations.
  2. Do the blocks always work like you expect, or can you find some strange inputs that cause them to output wonky things?
  3. This was Worksheet: Text Processing Blocks
    For each of the following blocks, write down:
    1. The input you gave it and the output reported
    2. A description of what you think the block does, based on the input/output

    Programmers often write descriptions of functions, to help other programmers understand how to use them.

Mix & Match the Blocks!

In Snap! programming, you can nest other blocks inside of each other as long as their shapes fit. All these processing blocks have rounded corners (indicating they're "reporter" blocks), so they can be nested inside each other.

  1. Try dragging a block inside another block and see what happens. Some questions to explore:
    1. What happens to the original text inside the block if you move a block inside it?
    2. Does it matter if you change the order, so that one block is on the inside versus the outside? Does it matter more for some blocks versus others?
    3. How many blocks can you put inside each other? Is there a limit?
  2. Share your remixes with your classmates and discuss your answers to the questions.
  1. If you want a challenge, see if you can nest the blocks to turn these input strings into the target output string:
    1. From input: just do it
      ↳ Output: J👏🏽U👏🏽S👏🏽T👏🏽 D👏🏽O👏🏽 I👏🏽T👏🏽

      ↳ Output: 11000101100011

Draw a Conversation

Let's make it look like an actual conversation on a phone!

  1. Find this block in the project:

    This is a command block that draws the conversation on the stage, just like an iPhone conversation. The block accepts any number of messages, but it can only draw as many that can fit on the iPhone screen at once.
  2. Drag one of your messages into the block. Remember that your message might be made up of a block inside another block.
  3. Click the right arrow on the block to add more slots and drag more messages.
  4. Once you're done, show your classmates your conversation and check out theirs. A lot can happen in a short number of messages!
