Can you get a result that's all 1s? Every decimal that is one less than a power of 2 is all 1s. 1 = 1 (same as 2 - 1) 3 = 11 (same as 4 - 1) 7 = 111 (same as 8 - 1) etc. That's because powers of 2 are always 1 followed by a bunch of zeros, so the number before is 111. It's analogous to 9, 99, 999 - the number that is all 9s is always one less than the power of 10. Can you get a result that's 11110000? The really easy way is to type "11110000 in decimal" in Google, and it will calculate it for you! However, this is how we'd manually calculate it. First, write out the powers of 2 under each bit in the number: 1 1 1 1 0 0 0 0 128 64 32 16 8 4 2 1 Then add up the powers that have a 1 in the place. 128 + 64 + 32 + 16 = 240 Storing bits as electricity What decimal numbers result in no capacitors charged at all? Only the decimal number 0 results in no capacitors charged. Every other number would need at least one capacitor charged. What decimal numbers make all the capacitors light up with charge? This is the same as the question about making numbers with all 1s from the previous section. Any decimal number that's a power of 2 minus 1 will result in a row of charged capacitors. All the numbers that we typed above were decimal numbers.

: Decimal Numerals

A decimal numeral is part of the base-10 system, the system most of us started learning as a toddler by counting numbers on our ten fingers and ten toes. The number system you use in daily life uses decimal numerals.

A decimal numeral can have any digit from 0-9 for each place value. The first place is the ones place (100=1), the second place is the tens place (101 = 10), the third place is the hundreds place (102 = 100), the fourth place is the thousands place (103 = 1,000), etc.

Computers actually store all numbers as binary numbers. All of their calculations are on binary numbers, too. They don't even know how to add two decimal numbers together! Instead, they first convert the decimal to binary and then do the adding.

: Binary Numbers

A binary number can only have either the 0 or 1 digit in each place. The first place is the 1s place, the second place is the 2s place, the third place is the 4s place, and each place doubles after that.

We can figure out the decimal equivalent by summing up all the places that have 1s - (8 + 2). That's the decimal number ten!

We can also call each place a "bit" (short for binary digit), so 1010 is a 4-bit number. A number that's only a few digits long in decimal may be many bits long in binary, but that's okay, because bits are the only thing that computers know how to store.

Storing Bits as Electricity

At this point, you might be wondering if you could dissect your computer and discover a bunch of 0s and 1s in the carcass. That'd be cool/creepy, but actually, computers don't really store the digits 0 and 1.

Computers are electronic devices, so they store binary data using electricity. Generally, a 1 is stored as a positively charged electric signal and a 0 is stored as no charge at all.
