Commands Versus Reporters

Before this unit, you've been nesting blocks inside each other so that the output reported by one block becomes the input to another. This style of programming is known as functional programming, and in Snap!, it primarily uses reporter blocks. A reporter computes a value and reports it (to you or to other blocks).

Is "primarily" needed? I think technically true FP is entirely reporter blocks, no? -PF 2/10/22

In this unit, you've been stacking one atop the other blocks that change things. This style of programming is called imperative programming. In Snap!, imperative programming uses command blocks. A command tells the computer to do something, (play a sound, draw, move, change something, etc.). Commands have effects.

You learned about reporters in Unit 1 Lab 1 Activity 4: Say Hello to Snap!.

A reporter block does a calculation and reports the result. Reporters have an oval shape.

You learned about predicates in Unit 1 Lab 6 Activity 2: L33t Text.

A predicate is a special kind of reporter that asks a true/false question and reports either true or false. Predicates have a hexagon shape.

You'll use commands much more in Unit 2.

A command block tells the computer to do something without reporting a value. Commands have a puzzle shape and can be snapped together one after another.

Unlike reporters, a command block doesn't report anything, so you can't use its result as the input to another block—there is no result!
