  1. Build your project!

    One approach is have the program trigger different events depending on which image class is detected. Another is to use the probabilities directly in your code (For example, you could use it as the position, rotation amount, or volume.)

    You may find it helpful to review the Snap! project from the example video shown on Activity 1 or to review this example code that uses the model's prediction to decide whether to move the sprite up, to the right, or back to the lower left.

  2. Test your project with different input images, or have someone else test it out.
  3. If needed, make improvements. You can even go back to Teachable Machine and re-train your model and get a new URL if you need to and have time.
