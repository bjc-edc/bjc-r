In machine learning, a model is like a recipe that helps a computer find patterns in data. In this project, your model will learn to sort data into classes (like "spam" vs. "not spam," or "cat" vs. "dog").

Models need to be trained. Training a model means teaching the computer how to sort the data by showing it many examples already labeled with the right class. Over time, the model learns rules to sort new data correctly.

Setting Up Your Classes with Training Images

In order for your project to know the difference between the classes you have chosen, you'll need to tell it what your classes are and give it examples to learn from.

  1. Open Google's Teachable Machine.
  2. Click “Get started,” then “Image Project,” and then "Standard image model."
  3. Set up your classes:
    • Rename "Class 1" and "Class 2" to describe your first two classes.
    • Add and rename additional classes if needed.
  4. Gather the things you plan to show the camera to create each class.
    5. TODO: add more support for webcam. --MF, 1/14/26
  5. For each class:
    1. Click "Webcam," place the item in front of the camera, and continuously press "Hold to Record" to gather some images.

      You might need to give your browser permission to access the webcam.

    2. Include some variation with different angles or backgrounds. Mix it up.

      For example: Record some images with the object rotated or showing a different side. Maybe have the person point their face in a different direction. Maybe record the same objects with a different background.

    3. Repeat for each class until you have at least 100 image samples for each class. You can delete images that don't match their classes well.

Training (and Re-training) Your Model

Reminder: supervised learning is a type of AI that uses a pre-labeled training set to categorize new data.

Training the model is the supervised learning part of this project. You've made a dataset (of images in classes). Now you'll train the computer on this dataset and the computer will make a model (a recipe) for which types of images belong to which classes. That way when you import the model into Snap!, it will be "trained" to categorize new images it has never seen before into these same classes.

  1. Click "Train model." Do not switch tabs while the model is being trained. If you see the message "page is unresponsive," click "wait."
    The model takes time to train, and your browser might think the website has crashed. You'll have to be patient for a minute to two.
  2. If a pop-up tells you to export the model and blocks the "Preview" window, close the pop-up. You can export later.
    Use the "Preview" window to test your model. Add or edit your image samples and re-train the model as necessary so that most of the time, it indentifies the correct class as output for the corresponding camera input.
  3. Discuss:
    1. Review your plan. What do you want your project to do with image inputs?
    2. Consider your model. Do you have enough classes for your plan?
    3. Do you want other people to use your program? Would it help to train your model with other people, objects, or backgrounds?
  4. If needed, add or delete images, and train the model again.
  5. When the model works well, click "Export model," select the "Upload (shareable link)" option, and click "Update my cloud model." The shareable link should turn blue, and you can “Copy” the link. If you don't have time to add this to your Snap! project today, you must save this link somewhere to keep your work from today! For example, you could paste it into a document you save or an email you send to yourself.
  6. Discuss with other students:
    1. What issues did you run into?
    2. How did you improve your model?
    3. How was your model training process similar? How was it different?
