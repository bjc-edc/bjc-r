Q-Learning

You may have noticed that there are common elements between how we kept track of good and bad moves in hexapawn vs the coin game. Here are some formal definitions that will help you understand how both processes are related.

  1. State - This can be known as the game state or board state. This is the information that represents a unique state in the world we are trying to represent. In the coin game, a state was how many coins were left. In hexapawn, the state was the current board configuration of the pieces on the board.
  2. Actions - For each state, there are actions, or moves that can be taken. In hexapawn, this was straight, diagonal, or capture. In the coin game, the actions were take 1 or 2 coins.
  3. Reward and penalties- Rewards and Penalties are how an RL algorithm knows it is doing the right thing. When an RL algorithm develops a policy of how to move through a world, it will use the rewards and penalties of different states on the board to develop this policy. In the coin game, we saw how rewards are given to game states that lead to a win. In Hexapawn, everytime the machine lost, the moves it made were given a penalty so it doesnt try those moves again. In the two examples we have seen so far, rewards and penalties were straightforward and easy to assign. In more complex worlds with more complex problems to model, this decision may not be as easy.

Q Learning, or “Quality Learning” is a common reinforcement algorithm that aims to keep track of the Q-Value of each move for each board state. If this sounds complex, dont worry, you’ve already been doing it! The coin game and hexapawn both use a form of Q learning to determine which moves are better than others.

Below is a simplified version of the Q-Value equation.
Q(s, a) ← r + max Q(s′, a′)

Simply put, the the Q-value for the current state “s”, given the chosen action “a” is equal to the reward of the current state + the max Q-value of the next state
