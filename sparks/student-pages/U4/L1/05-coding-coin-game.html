Coding the Coin Game

In this activity, you will code essential blocks for a coin game algorithm.

Soon you will build a computer player that will always win (if it can). Computers "think" by calculating with numbers, so we need the computer decide whether to take 1 coin or take 2 coins based on the number of coins left.

    • creates the list of win/lose values for each position
    • decides which move the computer will make by using the win/lose values
    Read through the code inside each block, and figure out how they will work. (The code won't work yet; three blocks used inside aren't finished yet. You will code them soon.)

  3. Write the code that belongs inside these three blocks:
    • should report a list of the options the player has on a given position (usually 1 and 2 unless the player is at position 1).
    • should report the new position of the game after taking away the input number of coins from the given position.
    • should report true them the game is over (that is, when the position is less than 1).
  4. Play a few games against the computer. It should play the game perfectly so it should be very hard for a human player to win. Fix any issues with your code.
