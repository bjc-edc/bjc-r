Generate a Drawing

In this activity, you will extend the same logic for stories and songs to drawings.

Here's a preview of how the drawing part of our N-gram project works. The user clicks "Train picture model," then draws a picture (this one has a lot of right angles). Then the user clicks on "Generate picture" and the program draws a picture in the style of the user's drawing.

  1. What are some ways you could represent a drawing as data (that we can then arrange into n-grams)?

There are lots of possible ways to represent a drawing as data. This project uses directions. The direction is a number between 0 and 360 that shows the angle at which the sprite would have to move to follow the mouse. In this program, whenever the mouse moves more than 5 steps away from the sprite, a new direction is captured and added to a list of directions.

  1. Try the block to see how it reports a list of directions. For example, click on the sketch block, then draw a straight line from left to right across the stage. Then click the sketch block again to see that it returns a list of directions. What angle (direction) corresponds to moving from left to right across the stage?
  2. Move in other directions to check that you understand how the sketch block reports a list of directions. What direction is going straight down the stage? From right to left? What does the list look like if you draw a circle on the stage?

If you tell the mouse where to start a drawing, a list of directions is enough to recreate the drawing. The next set of blocks in the project shows that you can save the list of directions that sketch reports in a variable (called "drawing") and then use a "draw sketch" block to use those directions to recreate the picture. We don't need the "draw sketch" block for our generator; this just shows that the directions can be used to reproduce the drawing.

  1. Draw a sketch and get a list of directions. What would the first four 2-grams for that data be?
  2. What would the first three 3-grams be?
  3. Look inside the block. How does it compare to the "Train model" and "Train song model" blocks used for stories and songs?

If you open the block, you will see the familiar block. As for stories and songs, all the directions are arranged into n-grams, and then the last few directions are used as context to match up with n-grams to provide candidates for a next direction. One of those is chosen, and the program makes a new line in that direction.

  1. Did you notice that the generated picture draws smaller patterns compared to the original? This was done intentionally to fit more of the generated drawing onto the screen. But you can change this. Open the block and try to change it so that the generated picture isn't so small.
In this activity, you learned that the same logic of n-grams and generating the next "thing" can work for drawings, just like it did for songs and stories.
