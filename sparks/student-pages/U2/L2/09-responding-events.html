Responding to Events

In this activity, you will make your project interactive by making the code respond to user events.

So far, your animation has only played when you (the programmer) click the script. You can use events to tell the the computer to run your code when something happens (such as when the user clicks something).

An event is something that happens, such as clicking on a sprite, pressing a key, or clicking the green flag button () that tells the computer to do something.

Event blocks go only on top of a script (as a way to tell the script to start), so Snap! programmers call them "hat blocks."
