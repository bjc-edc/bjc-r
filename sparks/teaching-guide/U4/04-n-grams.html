Lab 4: Generative AI with n-Grams  

(from CITE AI Proposal)
a. Participants discuss how they think genAI works to produce a new drawing in the style of a user-drawn image, or write a new story word-by-word in the style of a set of children’s stories
b. They use a small set of training data and work through code that organizes the data into n-grams (a collection of subsets of size n). Participants discuss how data organized in this way can be used to generate the next new word in a story (or the next new line in a drawing).
c. Participants work through code that generates the next new word.
d. Participants build a program that uses this code to tell a new story, play a new song, and/or draw a new drawing.
Recording notes/thoughts here (needs a lot of editing!): Activity 1 Starts with a discussion about language. This lab will uses n-grams as a tool for using data to generate output. However, this is a limited (though useful) tool. Importantly, it is a tool that was used in early text generation, and is still used (I think???) in tools such as spell checking and auto-complete (which don't require assigning meaning to words, but are sufficient to rely on relative frequencies of phrases of a certain length). Hopefully, students will see that n-grams are useful and powerful in their own way, and gets us a lot of the way towards generating sensible text, but falls far short of what they see current genAI tools doing. This first activity gets them thinking about far more than just looking at frequencies of words. But then also asks them to think about computers using words as data to get them to understand how language needs to be quantified in a sensible way for a computer to use (and therefore helps students understand how/why n-grams are used). Materials: student page Prepare: Optional - a Shakespeare/Taylor Swift (or similar) online quiz Learning Goals: Get students thinking about the complexity of the problem of generating language. Activity Plan: Discussion Tips: This is not meant to be a comprehensive discussion, but just a taste of the problem to get students to see that generating task is a difficult computational problem. Students are not expected to come up with any "answers," but students may have thought about trying to mimic their favorite artists and may share interesting insights about how different artists use language. If a student doesn't resonate with authors or songwriters and want to think about visual artists, feel free to let them explore that as well - this project will bring in image generation too.
