Lab 4: Generative AI with n-Grams  

(from CITE AI Proposal)
a. Participants discuss how they think genAI works to produce a new drawing in the style of a user-drawn image, or write a new story word-by-word in the style of a set of children’s stories
b. They use a small set of training data and work through code that organizes the data into n-grams (a collection of subsets of size n). Participants discuss how data organized in this way can be used to generate the next new word in a story (or the next new line in a drawing).
c. Participants work through code that generates the next new word.
d. Participants build a program that uses this code to tell a new story, play a new song, and/or draw a new drawing.

Activity 1: Generating Text 

  • Materials: Activity: Texting Time student page
  • Prepare:
    • Consider how you plan to lead the discussion.
    • Review the slides, and consider how you plan to lead the post-activity discussion.
    • There are a number of optional "If There Is Time" exercises on this page. Consider whether you plan to use some or all of them with all students or just with those students who are moving more quickly through the materials or express additional interest.
  • Learning Goals:
    • Determine what several blocks do and write descriptions of them.
    • Create block help text using Snap! comments.
  • Activity Plan:
    • Mary: give teachers the worksheet. --MF, 10/19/21
      Styling Your Text Messages: Students experiment with several text processing blocks and write descriptions of what they do. You may choose to have students write their descriptions on paper or on the computer; later they will refine their descriptions as they write "help text" for one or more of the blocks. (10–25 minutes)
    • Adding Program Documentation: Students learn about program documentation, how to use comments in Snap!, and how to use comments to create help text for blocks. (10–30 minutes)
    • Export Text Messages: Students export the output of their text processing blocks so they can paste the results outside of Snap!. (Optional: 5–10 minutes, if used)
    • Draw a Conversation: Students use a draw convo with messages block to create a simulation of a text conversation on the Snap! stage. (Optional: 5–10 minutes, if used)
    • Post-activity Slides and Discussion: After students have completed the For You To Do prompts on the student-facing activity page, use these slides to guide a class discussion. (Optional: 5–40 minutes) You may wish to:
      • Do a Show & Tell where students share the conversation they drew.
      • Review block nesting (e.g., when/whether nesting order matters, how many blocks can be nested).
      • Discuss the equity issues of software used in different languages and by people of different cultures.
  • Standards:
    • 2-AP-19: Document programs in order to make them easier to follow, test, and debug.
  • Solutions
    • Nest blocks to get target output strings. (Students may nest blocks differently and still get the correct output.)
    • Sample block descriptions. (Students may word things differently and still have a correct description.)
      • The songify block adds random music emojis at the start and end of the text.
      • The emojify block replaces particular words with corresponding emojis and leaves the rest of the text unchanged.
      • The clapify block inserts a clap emoji between each of the letters in the text.
      • The shorthand block removes all of the vowels from the text.
    • Sample help text. (Again, students may word things differently, but it is nice to describe inputs and outputs.)
      • lowercase: This block takes a string of text as input and reports the same text but with all of the uppercase letters changed to lowercase.
      • songify: This block takes a string of text as input and reports the same text but with random music emojis added to the start and end of the text.
      • emojify: This block takes a string of text as input and reports the same text but with particular words replaced by corresponding emojis.
      • clapify: This block takes a string of text as input and reports the same text but with clap emoji inserted between each of the letters.
      • shorthand: This block takes a string of text as input and reports the same text but with all of the vowels removed.
Recording notes/thoughts here (needs a lot of editing!): Activity 1 Starts with a discussion about language. This lab will uses n-grams as a tool for using data to generate output. However, this is a limited (though useful) tool. Importantly, it is a tool that was used in early text generation, and is still used (I think???) in tools such as spell checking and auto-complete (which don't require assigning meaning to words, but are sufficient to rely on relative frequencies of phrases of a certain length). Hopefully, students will see that n-grams are useful and powerful in their own way, and gets us a lot of the way towards generating sensible text, but falls far short of what they see current genAI tools doing. This first activity gets them thinking about far more than just looking at frequencies of words. But then also asks them to think about computers using words as data to get them to understand how language needs to be quantified in a sensible way for a computer to use (and therefore helps students understand how/why n-grams are used). Optional: The page mentions Shakespeare and Taylor Swift. If you look online, you can probably find a few quizzes about distinguishing text from Shakespeare and Taylor Swift - it's harder than you might think! Students may enjoy this. Materials: student page Prepare: Optional - a Shakespeare/Taylor Swift (or similar) online quiz Learning Goals: Get students thinking about the complexity of the problem of generating language. Activity Plan: Discussion Tips: This is not meant to be a comprehensive discussion, but just a taste of the problem to get students to see that generating task is a difficult computational problem. Students are not expected to come up with any "answers," but students may have thought about trying to mimic their favorite artists and may share interesting insights about how different artists use language.
