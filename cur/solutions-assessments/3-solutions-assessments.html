Lab 1: Dealing with Complexity

  • Robot in a Maze

    1. Many answers are possible. Besides the left-wall algorithm described later on the page, a likely student response would be something like this:

    1. From any position there are four possible ways to move: forward, backward, to my left, or to my right. List the ways that are not blocked by a wall.
    2. If this isn't the first move, delete any move that takes us to a position we've been in before.
    3. For each remaining possible move, make a clone and start it going in the appropriate direction.
    4. When a clone moves, have it mark the square it's in as having been visited, for the sake of rule (b).
    5. Hide the original sprite.
    6. If there is no possible move for this sprite, delete it.

    2. The steps of the algorithm, from (a) to (f), are done in sequence. Steps (b) and (f) involve selection. The repetition is harder to find because it works by cloning rather than explicit looping, so this algorithm might not be a good choice for the AP Create Task. But technically, cloning should count as repetition, which happens in the language's scheduler ("give each sprite with work to do a chance to run" cycles through a list of active sprites, like a FOR EACH ITEM).

    3&5. This exercise is a little sneaky, because the optimal move at the beginning is to back up a step and then move to the east. Still, the algorithm above works, because a backward move is allowed if it doesn't take us to an already-visited square. So does the left-wall algorithm, although it's important to note that left-wall doesn't say to give up if a square has been seen before. In this maze, the sprite will visit almost every square in the left half of the maze, then visit it again retracing its steps. Eventually it'll return to its starting position, but facing a different direction, so its left wall will take it into the right half of the maze and a quick solution.

    TIF A. The simplest example is to have an "island" at the entrance to the maze

    At the entrance (^) there's no left wall, so we move forward, hit an obstacle, turn right, and then forever loop going counterclockwise around the island.

Assignments

  • Assessment Idea: Write a program that keeps track of each student's grade in one class.
    1. Create abstract data type to hold each student record:
      • Create a student record constructor—a reporter block that, given a name (a text string) and grade (a number) as input, reports a student record (a list of that name and grade).
      • Create two selectors name and grade that, given a student record as input, report the name or grade for that student record.
    2. Make a variable class that, when the green flag is clicked, is set to an empty list that will store student records.
    3. Add a button (sprite) that uses your constructor to allow the user to create a student record by entering the student’s name and grade.
    4. Add a button that generates a grade report. Use the for each block to go through each record in class and have the sprite say the student’s name and grade for a second. Format the output so that it makes sense to the user.
    5. Submit your code in whatever way your class submits work.

  • Assessment Idea: Write a program that allows a user to enter words and definitions.
    1. Create an abstract data type to hold each word definition:
      • Create a word definition constructor that accepts two inputs: the word and the definition (both as text strings).
      • Create two selectors word and definition.
    2. Create a variable called vocab list that is a list of words and their definitions and that initializes when the green flag is clicked.
    3. Make a way for a user to enter a word and its definition. Use keep to determine whether a new word is already in the list of words and provide an error message to the user if it is.
    4. Make a way for the user to read all the words included in the dictionary. Use map to make the sprite say all the words separated by commas.
    5. Make a way for the user to look up the definition of a word in the vocab list.
    6. Submit your code in whatever way your class submits work.
