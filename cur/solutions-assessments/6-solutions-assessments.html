Take It Further:
  • Exercise A: These are meant to be research projects, so there are no short answers.
    • Touchscreen: It would be great if students discover that the sensors are at the edges of the screen, one sensor for X and one for Y.
    • Register: A small chunk of really, really fast memory inside the CPU. Used for several purposes (addressing, values used repeatedly, fast arithmetic).
    • Cache: A small, very fast memory that keeps a copy of recently-used locations from main memory so that repeated access to the same location (or one right next to it) will be at processor speed instead of slower main memory speed. (This is a really clever abstraction; the user's program doesn't know about the cache at all, and yet it runs much faster than it would without cache.)
    • Keyboard: The physical keyboards are/were great! Touchscreens aren't precise enough for virtual keyboards to be error-free, and so users have to use spelling corrector programs. The only advantage to a virtual keyboard is that it makes the phone cheaper to build (not necessarily that any of that saving is passed on to the consumer, however). Phone manufacturers somehow convinced people, especially young people, to want very thin phones, but that just makes them more fragile.
