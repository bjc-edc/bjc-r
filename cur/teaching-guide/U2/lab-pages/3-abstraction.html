Programming Lab 3: Abstraction

In Lab 3, students begin with a drawing project, "Brick Wall," and then build a complex number-guessing game (which they will extend and improve significantly in a later unit). In both cases, they are focusing almost exclusively on ideas of abstraction to serve the goals or writing clear, debuggable, improvable code—essential for developing complex programs. This is not the first time they used abstraction—they used abstraction when they built the game board for Tic Tac Toe (and other games) and when they built "specialist blocks" for making plurals—but the first lesson in which they begin to analyze the various uses of abstraction, for example, a case in which it is useful to build a new block that seems only to be a new name for an already existing block.

Students are given very specific goals for Brick Wall, guidance on structuring it, and rationale for that structure. For the number-guessing-game, they are again given suggestions for features to add, one at a time, but are on their own to think about how to structure these extensions and modifications so that the code works and remains clear.

There is no new Snap! code in Lab 3. Even the idea of abstraction—analyzing a task to find its chunks or component parts or special cases and then build blocks that specialize in each of them—is not new. But making a habit of analyzing and chunking tasks takes time and experience.

Pacing:

The 2 lab pages could be split across 2 or 3 days (55-110 minutes). Expected times to complete follow:
  • 25-50 minutes (about 1 class period) on Brick Wall
  • 30-60 minutes (about 1 class period) on Developing a Number Guessing Game

Students will:

-->
  • Build some tools for board games: a block to draw square arrays of various sizes for the game board, and a block that determines which cell within that array where the sprite is in.
  • Build a number guessing game in which the user tries to guess the computer's secret number, and the computer tries to respond with as human-like dialogue as the programmer can build.
  • Experience the idea of abstraction in the context of developing blocks that specialize.
  • Learn how to specify the type of input that a block is expecting to receive.
  • Get practice composing functions.

Tips:

  • Page 1: Brick wall. This involves the simplest coding of the three projects, but contains a key idea about abstraction. The suggestion to create a draw mortar block is important. It shows how editing one block can update a program in many places. Also, although the coding is simple, figuring out how to make row B's length come out right is a hurdle.
  • Page 2: Developing a number guessing game. This involves simpler coding, but is very open to students' creative ideas.
