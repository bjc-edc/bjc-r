Lab 3: Making Decisions by Using Predicates

Students build various predicates (reporters that only report Boolean values—true or false) and use them to solve word puzzles. First, they review their understanding of predicates, which they have used with conditinonals such as if and repeat until, and they learn to use predicates with the higher order function keep to report only the items from a list that match certain criteria. Then, the build several predicates to use in solving word puzzles (examples below) and use them with keep to perform various queries on a list of words. Finally, they create predicate expressions using Boolean operators (and, or, and not) to select list items that match multiple criteria.

Along the way, students learn about Booleans and Boolean operators, essential for the AP CSP test; learn how to indicate the intended type of an input variable (whether the intended input is to be a number, a string, a list, etc.); and they are introduced to the mod block which reports the remainder when two numbers are divided.

Lab Pages

  • Page 1: What's a Predicate?
    • Goals:
      • Review and codify understanding of predicates.
      • Broaden and refine understanding of the mathematics of computer science by using mod, and Boolean operators (and, or, and not).
      • Develop several versitile numerical predicates: between?, divisible?, and even?.
    • Tips:
      • Some vocabulary (for you to use in context, not to require of students):
        • A command is a single Lego-brick-shaped block, without its inputs. It tells the computer to do something.
        • A reporter is a single oval-shaped block, without its inputs. It reports data.
        • A predicate is a single hexagonal block, without its inputs. It's a reporter that only reports Boolean values.
        • A Boolean value is or . (Not the text strings "true" and "false"; not 0 and 1.)
        • An instruction is a command block along with its inputs or input expressions.
        • An expression is a reporter block along with its inputs or input expressions.
        • A Boolean expression is a predicate block along with its inputs or input expressions.
        So a reporter or a predicate is a function, and an expression is a function applied to input values (often called "arguments").
      • Mathematics Note: Mod. The mod block returns the remainder when the first input is divided by the second.

        For positive whole numbers, this is the same as the notion of remainder that students learn in elementary school when they learn to divide one whole number by another. So:

        because when 7 is divided by 5, the remainder is 2 (and the quotient is 1).

        The behavior of the mod block may seem mysterious when one or both of the inputs are negative integers. For example:

        The sign of the output of mod agrees with the sign of its second input. But why?

        Let's go back to positive whole numbers for a moment. The fact that mod (7, 5) = 2 comes from 7 \div 5 = 1 \text{ remainder } 2, which comes from 7 = 5 \times 1 + 2 where 1 is the quotient and 2 is the remainder. The quotient is the key to the mystery.

        In Snap!, you can get the quotient by dividing 7 by 5 to get 1.4 and then rounding down. This operation of rounding down is called floor, and it's built into Snap! (you can find it as a drop-down option in the sqrt block in the Operators pallete). For any real number, floor takes the closest integer to the left on the number line (hence the name "floor").

        So, given any two integers a and b (or real numbers, really) you can calculate the quotient \frac{b}{a} and then the floor of their quotient \text{floor}(\frac{b}{a}). Let's call this quotient q. Then, b will equal aq plus the little left over that we lost by using floor (the remainder, which we'll call r). So now, b = aq + r much like 7 = 5 \times 1 + 2.

        That remainder is what Snap! and most programming languages report for mod(b, a) and this explains the signs of the outputs when one or both inputs are negative.

  • Page 2: Keeping Items from a List.
    • Goals:
      • Learn how the keep block uses predicates to select items from a list.
    • Tips:
      • Ask students to articulate the basic idea of the keep block. It's a reporter that reports only the items from an input list which meet the criteria specified by a predicate; the predicate itself must have a blank for the list items to cycle through.
  • Page 3: Solving a Word Puzzle.
    • Goals:
      • Build several predicates blocks by constructing predicate expressions for specific cases first and then generalizing.
    • Tips:
      • If students attempt the "If There Is Time" problem, they will likely need some time to consider how to access the last letter of a word. Use probing questions to support their problem-solving:
        • How can you find the place number of the last letter?
        • Look back at the last problem about the 9th letter, how is that related?
