Programming Lab 2: Script Variables

Students have already created blocks that have input variables, and have used those input variables within the block's script. This lab introduces the script variables (a) block, which allows the programmer to create variables for scripts that are not inside blocks, and extra temporary variables for blocks that need them. Script variables exist only while the script is running. They allow you to save, use, or modify information that the script will need while it is running. Also, script variables exist only inside that script. Blocks used inside a script can set, use, or change the value of that script's script variables; blocks outside of that script can not.

This lab gives students practice building several common mathematical functions using script variables, and also gives them practice building several predicates that they are likely to find useful in future projects. Students get continued opportunities to debug before lab 3, which begins to focus on the development of more complex programs.

Prepare:

  • Getting, renaming, setting, and using a script variable are all new. Some students will need help just building the script shown in problem 1 on the first page of this lab. An animation shows most of what they will need to do, but make sure you are comfortable enough, yourself, to help if that is needed.
  • With other programming, analyzing, and debugging problems, it can actually be useful for you not to look fully fluent, even if you know exactly what to do. Showing how you might puzzle out the problem can be an excellent way of modeling the thinking for your students. You don't have to "fake" not knowing; just asking what various bits of the program do, or what they're supposed to do, or how the student figured out what to put in or where to put it, or pausing for a moment to ponder the result, or trying various inputs to see how the program responds can be enormously helpful to the student because it illustrates a good analysis/debugging technique. Often enough, students then find the solutions themselves.

  • In Unit 6, students will learn how to use recursion to understand factorial and power (and some other functions for which they are currently creating using script variables) in a new way.

As a Class:

  • As in Lab 1, class time can be mostly lab.
  • Students work in pairs, switching regularly. You remain available for help, but encourage students to check with each other, as well, for ideas and help.

Look For:

  • As you notice interesting student work—creative responses to assigned tasks, ideas that go beyond the tasks, or interesting bugs—you may choose to have students share, but allow plenty of time for students to do the programming.
