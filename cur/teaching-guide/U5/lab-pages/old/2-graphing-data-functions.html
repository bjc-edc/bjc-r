from the old Lab 5:

Students learn the value of data visualization in acquiring new information from data, and the iterative way in which new insight is learned from data processing. Students learn the concept behind large data sets, including necessary tools for working with and understanding large data, and the value of working with others in discovering insights from large data.

It is easier to compare the efficiency of the two algorithms if we can visualize the data we get from timing experiments. In this lab, students create a general graphing program that plots data points on a screen whose dimensions (scale) they have determined themselves. They then work with a dataset on a classroom of 27 sixth-graders and isolate various data subsets to create graphs of height vs. weight, age vs. height, age vs. weight, etc.

This graphing program also has the option of taking a function as input and producing its Cartesian graph. On the last page, students explore using the app to plot a function such as -5 + (x-3)^2.

The graphing app that students build differs from many traditional graphing programs in that it requires hovering the mouse over the stage to input the x-values into the function, produce the points of the graph, and plot them. This is done for two reasons, one pedagogical and one practical:

  1. pedagogical: inputting the mouse x value into the function to get the corresponding y values supports understanding both what this graphing app is doing and the underlying mathematics of functions (e.g. unlike the Graph datapoints procedure, which begins with a complete set of x and y coordinates, the Graph function procedure needs to generate y values based on inputed x values—this is known to be a challenge in understanding functions for beginning algebra students)
  2. practical: this approach makes use of what students already know about programming (e.g. how to read the mouse location and how to adjust it based on the graph scale) rather than working through the complicated algorithm of making subdivisions necessary to get a consistent curve

Goals for the Lab:

  • Page 1: Working With Large Data Sets.
    • Investigate a large data set using an interactive visualization tool.
    • Learn about the challenges and opportunities when working with large data sets.
    • Discover and solve problems through analysis of large data sets.
    • Looking Ahead
    • Students will use their graphing app for some of their data-processing explorations later in this lab.
    • Students can also use their graphing app to visualize timing data on their algorithms the Timing Experiments lab.
  • Page 2: Building a Graphing App (Teacher's Choice).
    • Create a graphing program that plots data points.
    • Work with a scaling feature to adjust screen dimensions.
  • Page 3: Improving Your Graphing App (Teacher's Choice).
    • Use their graphing program to plot and analyze data points.
    • Isolate and graph various data subsets.
    • Use visualization tools to communicate new information.
  • Page 4: Graphing Functions (Teacher's Choice).
    • Use their graphing program to graph a function.

As a Class:

  • Page 1: Working With Large Data Sets.
    • Discussion: What are some ways to visualize large data sets?
      • Can you think of any other ways the Baby Name Voyager data might be presented?
      • What new information is gained from the Baby Name Voyager visualization that would otherwise be difficult to learn?
      • What visualizations might be possible for a different type of large data set?
    • Discussion: How can you work with large data sets?
      • Where is the information in large data sets stored? How is it accessed?
      • In what ways is it especially difficult to work with large data sets?
      • What large data sets might be part of a student's life?
  • Page 2: Building a Graphing App (Teacher's Choice).
    • Discussion: Toward the end of class, you may wish to discuss:
      • Why might you want to set the scale of the screen?
      • Describe how this setup works to set the scale of the data on the stage?
  • Page 3: Improving Your Graphing App (Teacher's Choice).
    • Discussion: After students work through the lab page, discuss this question as a group:
      • What can you say about this particular class based on the data? Have students share and describe some of the graphs they created to support their observations.
  • Page 4: Graphing Functions (Teacher's Choice).
    • Discussion: You may wish to address these questions as students work on the page (e.g. after reading, but before going too far with the For You to Do exercise) in order to encourage students to think about how Graph function works:
      • What are some similarities and differences between the Graph datapoints and Graph function procedures?
      • How does the Graph function procedure determine what points to plot?

Tips:

  • Page 1: Working With Large Data Sets.
    • Expect a wide variety of search results from the Baby Name Voyager, and look for opportunities for students to share interesting data with others. Of particular interest may be "fad" names that were popular in specific decades (Lisa, Jennifer), and other names that were popular long ago but are re-emerging (Emma).
    • The Baby Name Voyager site frequently becomes unresponsive if no result is found. When this happens, ask students to reload the page.
    • Some students may get frustrated by not finding their own name in the Baby Name Voyager. Point out to students that only a small percentage of all names are listed. A larger text file (available here) contains all names for at least 5 births. You might check this file to see if your students' names are in it -- or even challenge a student to create an app that tests names against this data.
    • You may want to draw some connections to prior units' social implications labs, since there have been discussions about large data sets, notably a discussion about privacy concerns in personally-identifiable information.
    • During this lab, keep a lookout for news reports that involve the analysis of large data sets, for appropriate Computing In The News material.
