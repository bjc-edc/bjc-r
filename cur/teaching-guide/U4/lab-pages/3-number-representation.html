Related Resources:

  • For page 3, Hexadecimal Representation, you may wish to offer one or more of the following number representation games. At your discretion, use them as homework, for a shorter class period, or for a day before vacation. In the classroom, watch that these do not distract students from the essential content of the unit.
    • Cisco Binary Game: a base 10 to base 2 board-clearing game. This game could help to review base 2 (from Unit 1), though it does have an element of time pressure and might take few minutes to figure out. The instructions are not perfect but helpful.
    • Bin-Hex Challenge: a base 2 to base 16 memory-style game. This game has no time pressure, so it might be the most accessible of the three. Also, the first ten matches (0_{16}-9_{16}) are easier because they are like converting between base 10 and base 2.
    • Flippy Bit And The Attack Of The Hexadecimals From Base 16: a base 2 to base 16 arcade-style shooter. This one is especially challenging and should only be offered to students only after completing the whole Hex lab page.

Tips:

  • Page 3: Hexadecimal Representation.
    • Don't overemphasize number representations. This is a popular topic because it's easy, but it doesn't call for deep understanding; hex is just a notation that people find convenient in a few computing contexts. (Very few, actually; RGB and IPv6 pretty much exhaust them.)
    • Although base 16 is a base in its own right, like base 7 or base 10, it's important only as an abbreviation for the underlying binary. You don't necessarily have to say that to students, but don't allow a mystique to develop around hex; it's bad enough that everyone is so fascinated with binary.
    • Remind students, if necessary, that the purpose of computer science is to abstract away the grubby details. That's why "orange" (which browsers also understand) is a much better representation for that color than "FF7F00."
