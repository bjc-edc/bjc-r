Programming Lab 4: Building and Debugging Your Own Blocks

Students learn to create custom blocks and blocks with inputs and to use custom blocks within custom blocks. Using blocks within blocks continues to presage recursion (calling a procedure from within itself), which will be addressed in Unit 5: Recursion and Unit 6: Recursive Reporters. In this lab, students continue to develop comfort with block programming and the snap interface as they create their own blocks and identify categories and input variables for their new blocks.

Students also build on their skill with debugging, and they consider how a block that works well on its own might cause problems when used within another block if it is assumed that the inner block preserves the state of the sprite but it actually does not. Whenever a program is not exhibiting the expected behavior, it's a good idea to consider that the bug may not be in a sub procedure of the script but in the way that sub procedures combine and relate to each other when embedded.

Pacing

The 5 lab pages could be split across two or three days (55-80 minutes). Expected times to complete Lab 4 follow:

  • 10-15 minutes on Designing Custom Blocks
  • 5-10 minutes on Adding an Input
  • 15-20 minutes on Debugging Custom Blocks
  • 15-20 minutes on Using Blocks within Blocks
  • 10-15 minutes on Debugging Blocks within Blocks

Prepare:

  • Prepare several Computing in the News stories.

Students will:

Looking Ahead
  • Lab 4 continues to presage recursion.
  • Student will work more with custom variables in Unit 2 Lab 3: Developing more complex programs, Input Types.
  • Create procedures with inputs, building on Lab 2 work with drawing a square.
  • Use custom procedures inside procedures, using draw square to draw the petals in draw flower and using draw flower to draw a field of flowers.
  • Debug procedures inside procedures, considering why a script to draw a row of squares by repeatedly calling a draw square block that does not return the sprite to the starting orientation fails.

As a Class:

  • Make a Prediction: Count from 1 to num
    • Ask students to predict what this script does. Students may notice the "hat" block, which here names the procedure being defined, count up (num). From their work with the for block, students will likely be able to determine that this script "says" the numbers from 1 to num each for 1 second.
  • Discussion/Writing Prompts: These "abstraction" prompts can be used in sets or spread across several days and are for either individual or collective reflection:

    • How is abstraction important to programming?
    • How is calling blocks within blocks similar to and different from nesting repeats in repeats?

Assignments:

  • You may wish to specifically assign one or more of the optional "Take it Further" problems:
    • Designing Custom Blocks:
      • Creating a draw triangle block
      • Creating a draw house block (using the draw triangle block)
    • Adding an Input:
      • Extending draw triangle and draw house to accept a size input
      • Creating a draw polygon block
      • Nesting polygons with inputs for number of sides and sidelength

Tips:

  • On the Using Blocks within Blocks student lab page, students are given only a general instruction to draw a field of flowers. Encourage students to get creative! If students need help focusing their ideas, suggest they build up their project one step at a time: first building the draw square block, then the draw flower block (perhaps without inputs at first and adding inputs for number and size of petals depending on available time), and lastly creating a script that moves the sprite randomly around the stage and calls the draw flower block. Students might expand this project by changing the size or color of the pen from flower to flower or by changing the polygons used in the flowers from squares to triangles, etc.
