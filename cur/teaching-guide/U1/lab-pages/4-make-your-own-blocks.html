Programming Lab 4: Building and Debugging Your Own Blocks

Students learn to create custom blocks, custom blocks inputs, and custom blocks used within other custom blocks. Using blocks within blocks continues to presage recursion (calling a procedure from within itself), which will be addressed in Unit 5: Recursion and Unit 6: Recursive Reporters. Here, students continue to develop comfort with block programming and the snap interface as they create their own blocks and identify categories and input variables for their new blocks.

Students also build on their skill with debugging considering, in particular, how a block that works well on its own might cause problems when used within another block if it is assumed that the inner block preserves the state of the sprite, but it actually does not. Whenever a program is not exhibiting the expected behavior, it's a good idea to consider that the bug may not only be in a sub procedure of the script, but also be in the way that sub procedures combine and relate to each other when embedded.

Pacing

The 5 lab pages could be split across two or three days. Expected times to complete Lab 3 follow:

  • 10-15 minutes on Designing Custom Blocks
  • 5-10 minutes on Adding an Input
  • 15-20 minutes on Debugging Custom Blocks
  • 15-20 minutes on Using Blocks within Blocks
  • 10-15 minutes on Debugging Blocks within Blocks

Tips:

  • On the Using Blocks within Blocks student lab page, students are given only a general instruction to draw a field of flowers. Encourage students to get creative! If students need help focusing their ideas, suggest they build up their project one step at a time: first building the draw square block, then the draw flower block (perhaps without inputs at first and adding inputs for number and size of petals depending on available time), and lastly creating a script that moves the sprite randomly around the stage and calls the draw flower block. Students might expand this project by changing the size or color of the pen from flower to flower or by changing the polygons used in the flowers from squares to triangles, etc.
