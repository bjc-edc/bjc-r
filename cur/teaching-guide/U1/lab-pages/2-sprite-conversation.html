What you will do: In this lab, you will program your computer so that two sprites gossip with each other, picking randomly from words and ideas that you specify. Keep it friendly!

What you will learn: You will learn to create your own blocks that you can use just like the blocks that Snap! provides. You will learn how to create lists of data, select data from those lists, and combine data in new ways. In this project, the data will be words and phrases that you supply.

In Labs 2 and 3, students learn a new idea: not only can they assemble existing blocks (code) to create sequences (scripts), they can also encapsulate an entire script into a single new block of their own design, which they can then use as if that block had always existed. This process—one aspect of abstraction—is basic to computer science.

In this Lab 2, students first experiment with some blocks that have been built for them, to see what they do. Then they learn to edit and modify these blocks. Finally they create a new block of their own using the exact same structure they've seen in the blocks they've edited, but with new content. (In lab 3, they will extend this knowledge, learning how to give blocks inputs.)

The context is whimsical—students generally enjoy teaching a computer to gossip and find the results funny—but the ideas they are learning are deep.

  • Abstraction: A multipart script can be "hidden" inside a block so that you don't need to see all its details, and given a name so that you easily know what it does. (Students will learn another aspect of abstraction in lab 3.)
  • Creation of a list and (random) selection from a list: Students have used numbers as "data" (information) when they worked on the ClickAlonzo project. Lists, and the words and phrases that they put in the lists in this lab, are other kinds of data. Lists are extremely powerful and students will learn a great deal more about them in Units 2 and 3 and beyond.
  • Computer-generated language: The gossip example is very elementary, but getting a computer to generate (and understand) natural language has been the serious work of computer scientists for decades. Siri is one of the results.
  • Technique: Learning how to test (experiment with), edit, modify, and create new blocks is central to programming.
  • Process: Learning to try out code to see what it does, and then analyze code to see how it works is also central to programming.

It is probably too early to talk about abstraction and name it. Students will need to see more examples of it before it needs a piece of technical vocabulary. But two of its attributes may be worth mentioning even now. Putting inside a block named makes code easier to read and understand by "hiding the complexity" and using a simple name that describes the purpose of the code. Once we understand what who does and are assured that it works properly, we no longer need or want to see the guts of how it works.
