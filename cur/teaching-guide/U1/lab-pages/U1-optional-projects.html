Optional Projects

These optional programming projects are designed to supplement the required lab pages. There are many options for using these projects, and they depend on your pacing through the curriculum and the students in your class. You may decide to have each student select one project, or you may have them each build several. You may have students work individually, or you may have them pair program. You may have them work on their own time, or you may allocate class time for these projects.

Pacing:

Expected times to complete each optional project follow, but these times will vary significantly depending on your students:
  • 45-60 minutes (1-2 class periods) on Sprite Line Art
  • 45-60 minutes (1-2 class periods) on Build Your Own Clock
  • 45-60 minutes (1-2 class periods) on The Game of Pong
  • 45-60 minutes (1-2 class periods) on Daisy Design
  • 45-60 minutes (1-2 class periods) on Modeling Art

Lab Pages

  • Page 1: Sprite Line Art.
    • Learning Goals:
      • Understand fundamentals of processing keyboard input and interacting with a sprite through the keyboard.
      • Review drawing commands and direction in Snap!.
    • Tips:
      • This less difficult project connects to the Sprite Drawing lab, Exploring Motion page. It could be recommended to students with an interest in art and design.
  • Page 2: Build Your Own Clock.
    • Learning Goals:
      • Prepare for the AP CSP Create Task.
      • Develop parallel code for multiple sprites.
    • Tips:
      • This more involved project connects to the Sprite Drawing lab, Angles and Turning page. It could be recommended to students with an interest in building and modeling things.
  • Page 3: The Game of Pong.
    • Learning Goal:
      • Explore the fundamentals of game programming.
      • Review conditionals and other blocks (random, forever, etc.).
    • Tips:
      • This more involved project can be a follow up to the Sprite Drawing lab, Exploring Motion page. It could be recommended to students with an interest in game design.
  • Page 4: Daisy Design.
    • Learning Goal: Review drawing commands and basic iteration.
    • Tips:
      • This less difficult project connects to the Sprite Drawing lab, Angles and Turning page and Graphics and Art page. It could be recommended to students with an interest in art and design.
  • Page 5: Modeling Art.
    • Learning Goal:
      • The key idea is that very slight modifications to an algorithm can have huge effects.
      • The secondary idea is to learn to experiment to see what an algorithm can do.
      • A key "computational thinking" goal is to see how a computer program can model a human action, in this case, the style of an artist. The language project(s) in unit 2 have the same goal.
    • Tips:
      • Encourage experimentation and creativity.
      • Possibly collect student artworks and display them.
      • The programming here is quite light; the focus is mostly on experimentation and creativity. This project can easily follow the Exploring Motion even though it uses for which doesn't appear until page 6 of the lab, Looping with a Counter.
