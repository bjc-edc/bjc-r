Students use loops to repeat steps of code. Students consider the implications of nesting repeat blocks within repeat blocks and explore short scripts with forever and for blocks.

Nesting repeat within repeat previews Using Blocks within Blocks in Lab 4. Students develop understanding of abstraction as they see how projects can be subdivided into smaller tasks and programmed in chunks. This practice lets programmers build more complex programs by hiding details, and makes the code easier to read and debug.

Pacing

The 3 lab pages could be split across two days (50-65 minutes), and may push into the second week of Unit 1. Expected times to complete follow:

  • 15-20 minutes on The Real Value of the Repeat Block
  • 20-25 minutes on Nesting Repeats

Students will:

  • Use several basic kinds of loops (repeat, forever, and for) and formalize their understanding of them.
  • Learn the idea of nesting loops within loops.
  • Begin to explore sound blocks, namely play note () for () beats and string operator blocks such as letter () of () and length of ().

Tips:

  • In the first page, the analyze structure question has a deliberate ambiguity: Can we run the script more than once? If not, only the square can be drawn, but if we can run that script more than once, it can draw all of the first three pictures. With luck, students will take different sides on that ambiguity and an interesting discussion will ensue. If that doesn't happen spontaneously, you can start the discussion by pointing out the ambiguity yourself. Don't say it was intentional, but feel free to say "Whoever wrote this wasn't clear enough." That leaves room for you, in class to say to students, when it is true, "I'm not sure what you mean. Could you say more?"
