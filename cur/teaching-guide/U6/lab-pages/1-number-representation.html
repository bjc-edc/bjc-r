Lab 1: Number Representation

This lab addreses the requirements of AP CSP Enduring Understanding 2.1.2, "Explain how binary sequences are used to represent digital data." (It does not cover hardware abstraction levels.) The first page introduces binary representation as a place value system just like decimal representation, but with only two digits (0 and 1) instead of ten (0 through 9). The second page explains how a sequence of bits can represent information other than integers, and in particular explains floating point as an approximation for real numbers. The third page introduces hexadecimal representation, which uses sixteen digits (0-9 and A-F), building on students' understanding of binary. On the fourth page, students learn to translate between hex and binary and about the use of hex in RGB colors. The last page is an optional project in which students build a decimal-to-binary conversion procedure based on and reviewing their work with base 2 representation.

Pacing:

The 4 required lab pages could be split across 3-6 days (120-240 minutes). Expected times to complete follow:
  • 30-60 minutes (1-2 class periods) on Binary Representation
  • 35-70 minutes (1-2 class periods) on Binary Sequences
  • 25-50 minutes (about 1 class period) on Hexadecimal Representation
  • 30-60 minutes (about 1 class period) on Using Hex
  • If used, 60-120 minutes (1 class period) on Binary Timer Project*

Prepare:

  • There's no such thing as a "binary number" or a "hexadecimal number." The way we represent a number—as XXXIRoman or 3110 or 111112 or 1F16 or "thirty-one" or "treinta y uno"—is language or notation. The number, itself, is the same, regardless of what language we express it in. It helps the distinction to say "binary numerals" or "binary representation" instead of "binary numbers." Some curriculum materials are less careful. It's not just pedantry; students understand the conversions better when they remain clearly aware that only the notation, not the number, changes. Explicitly making the analogy with Roman numerals may help; students are likely to understand and be able to extend from the idea XXXI and 31 are the same number written different ways.
  • "Hexadecimal" is often abbreviated just to "hex." Either is fine to use generally, but students should recognize both.

Goals for the Lab:

  • Page 1: Binary Representation.
    • Understand how numbers are stored in binary notation.
    • Understand how to translate numbers between binary and decimal notations.
  • Page 2: Binary Sequences.
    • Understand that numbers are represented using binary notation.
    • Understand that there are many different ways to represent a number using binary notation.
    • Understand that the choice of representation has implications for the kinds of numbers that can be stored and how accurately they can be stored.
    • Understand that the standard mechanism of storing numbers has limitations on the size of the number.
    • Understand that floating point is a representation for storing numbers that is analogous to scientific notation.
    • Understand that by default, if a number is above a certain size, it will be stored as a floating point value and therefore be rounded.
    • Understand that the meaning of a string of binary digits depends on the context in which it is used.
  • Page 3: Hexadecimal Representation.
    • Understand how numbers are stored in hexadecimal notation.
    • Understand how to translate numerals between hexadecimal and decimal notations.
  • Page 4: Using Hex.
    • Understand how to translate numerals between hexadecimal and binary notations.
    • Understand that colors can be represented using hexadecimal notation and that this is done in HTML.
  • Page 5: Binary Timer Project.
    • Understand algorithm(s) that are used to convert from decimal notation to binary notation.
    • Develop intuition about how decimal and binary representations relate to each other.
    • Implement timer that displays elapsed time in both decimal and binary notations.
    • Write code to supplement and extend existing code.

As a Class:

  • Page 1: Binary Representation.
    • Don't try to teach students to do binary arithmetic. The goal is for students to learn to translate between representations.
  • Page 2: Binary Sequences.
    • You might have students take turns reading, or you might have all students read a section silently and then ask one or two summarize out loud.
    • Discussion: After students work through the lab page, you might use this question to review:
      • Why did Snap! display 20! in ordinary, whole number representation but 30! in scientific notation?
  • Page 3: Hexadecimal Representation.
    • Don't teach hex arithmetic. The goal is for students to become familiar with the notation of hex. Consider translating between representations for a few numbers together at the board.

Tips:

  • Page 2: Binary Sequences.
    • We explain that other data types, including lists, are represented as binary sequences, but we do not introduce the idea of pointers to memory addresses.
  • Page 3: Hexadecimal Representation.
    • Don't overemphasize number representations. This is a popular topic because it's easy, but it doesn't call for deep understanding; hex is just a notation that people find convenient in a few computing contexts (very few, actually; RGB and IPv6 pretty much exhaust them).
    • Although base 16 is a base in its own right, it's important only as an abbreviation for the underlying binary. You don't necessarily have to say that to students, but don't allow a mystique to develop around hex.
  • Page 4: Using Hex.
    • Remind students that the purpose of computer science is to abstract away the details; that's why, for example, "orange" (which browsers also understand) is a much better representation for that color than "FF7F00."

Correlation with AP CS Principles Framework: 

Enduring Understandings:

  • EU 2.1 A variety of abstractions built on binary sequences can be used to represent all digital data.

Learning Objectives:

  • LO 2.1.1 Describe the variety of abstractions used to represent data. [P3]
  • LO 2.1.2 Explain how binary sequences are used to represent digital data. [P5]

Essential Knowledge:

  • EK 2.1.1A Digital data is represented by abstractions at different levels.
  • EK 2.1.1B At the lowest level, all digital data are represented by bits.
  • EK 2.1.1C At a higher level, bits are grouped to represent abstractions, including but not limited to numbers, characters, and color.
  • EK 2.1.1D Number bases, including binary, decimal, and hexadecimal, are used to represent and investigate digital data.
  • EK 2.1.1E At one of the lowest levels of abstraction, digital data is represented in binary (base 2) using only combinations of the digits zero and one.
  • EK 2.1.1F Hexadecimal (base 16) is used to represent digital data because hexadecimal representation uses fewer digits than binary.
  • EK 2.1.1G Numbers can be converted from any base to any other base.
  • EK 2.1.2A A finite representation is used to model the infinite mathematical concept of a number.
  • EK 2.1.2B In many programming languages, the fixed number of bits used to represent characters or integers limits the range of integer values and mathematical operations; this limitation can result in overflow or other errors.
  • EK 2.1.2C In many programming languages, the fixed number of bits used to represent real numbers (as floating point numbers) limits the range of floating point values and mathematical operations; this limitation can result in round off and other errors.
  • EK 2.1.2D The interpretation of a binary sequence depends on how it is used.
  • EK 2.1.2E A sequence of bits may represent instructions or data.
  • EK 2.1.2F A sequence of bits may represent different types of data in different contexts.
  • EK 6.2.2J The bandwidth of a system is a measure of bit rate—the amount of data (measured in bits) that can be sent in a fixed amount of time.
  • EK 6.2.2K The latency of a system is the time elapsed between the transmission and the receipt of a request.
