  • Page 1: Bits.
    • Exercise 1: With 2 bits there are only 22=2×2=4 possibilities {00,01,10,11}.
    • Exercise 2: With 3 bits there are only 23=2×2×2=8 possibilities {000, 001, 010, 011, 100, 101, 110, 111}.
    • Exercise 3: With 4 bits there are only 24=2×2×2×2=16 possibilities {0000, 0001, ...., 1111}.
    • Exercise 4: 210=1024 to be exact.
    • Exercise 5: There are 7 days of the week and the closest power of 2 past 7 is 8 which is 23. Thus, 3 bits are enough.
    • Exercise 6: There are 10 decimal digits {0, 1, 2, ...,9} and the closest power of 2 past 10 is 16 which is 24. Thus, 4 bits are enough.
    • Exercise 7: 264=18446744073709551616=1.8446744073709551616×1019 many values.
    • Exercise 8: There are 26 letters {A, B, C, ..., Z} and the closest power of 2 past 26 is 32 which is 25. Thus, 5 bits are enough.

    • Exercise 4: red 145, green 0, blue 226 will be a slightly blueish purple
    • Exercise 5: 04FF61 will be a slightly blueish green
    • Exercise 6: The hex RGB codes requires less space. The color names are easier to understand. The two are easy to use in different ways: the RGB code is precise and doesn't depend on human interpretation, but the color names may be more familiar.
    • Exercise 7: Each of these bases are useful representations that simplify things in different contexts: binary is useful in the context of computers, decimal is useful in the context of human interactions, hexadecimal is useful in the context of color representations.
    • Exercise 8: In all of these contexts, abstraction is used to hide complexity and to simplify things. Hexidecimal representation abbreviates binary representation and makes it more readable, just as custom blocks and ADT's simplify complex code and make the code more readable.
