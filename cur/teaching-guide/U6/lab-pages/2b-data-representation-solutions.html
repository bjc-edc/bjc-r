  • Page 4: Floating Point.
    • Take It Further:
      "One significant digit" means that, for example, 200 is representable, but 210 isn't. Here are all the representable values:
      • 0
      • 0.01, 0.02, ... 0.09
      • 0.1, 0.2, ... 0.9
      • 1, 2, ... 9
      • 10, 20, ... 90
      • 100, 200, ... 900
      So there are exactly 46 representable values. Here they are, on the number line:

      It's easy to see the multiples of 10 and of 100, but all the smaller values are a blur. If you could zoom in on the range from 0 to 10, it would look the same, with the multiples of 0.1 and 1 visible, and the multiples of 0.01 just a blur.

      What are the pros and cons of this representation? Only 28 integer values can be represented, so 972 integers less than 1000 can't be. This seems like a severe weakness, but remember, the rules of this game say that you only have enough room in your representation for 46 values altogether. You could choose multiples of 20, to get all your values evenly spaced, but then you get no fractions at all. This system gives you a few really small values, a few medium-sized values, and a few really big values. So physicists can measure the size of an electron, astronomers can measure the size of the galaxy, and the rest of us can budget our money.

      By the way, if you want to make this picture in Snap!, you need a widescreen monitor, and then you can click on the gear (settings) menu, choose "stage size," and set the horizontal size to 1200 so there's some margin around your number line.

    • Exercise 6: The hex RGB codes requires less space. The color names are easier to understand. The two are easy to use in different ways: the RGB code is precise and doesn't depend on human interpretation, but the color names may be more familiar.
    • Exercise 7: Human beings use decimal because we have ten fingers, and computers use decimal when they are interacting with human beings. When they're not interacting with people, computers use binary because transistors have two stable states. Neither people nor computers really speak hex, but when people are forced to deal with binary, we use hex as an abbreviation (one fourth the length of the binary numeral), and so computers use hex to convey binary to humans. RGB color codes are just one instance of that.
    • Exercise 8: In all of these contexts, abstraction is used to hide complexity and to simplify things. Hexadecimal representation abbreviates binary representation and makes it more readable, just as custom blocks and ADT's simplify complex code and make the code more readable.
