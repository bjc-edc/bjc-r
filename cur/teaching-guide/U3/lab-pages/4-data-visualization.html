Lab 4: Building Data Visualization Tools

Students learn the value of data visualization in acquiring new information from data, and the iterative way in which new insight is learned from data processing. Students learn the concept behind large data sets, including necessary tools for working with and understanding large data, and the value of working with others in discovering insights from large data.

Pacing

The 3 lab pages could be split across 5-8 days (225-330 minutes). Expected times to complete follow:
  • If used, 90-120 minutes (2-3 class periods) on Creating Bar Graphs*
  • If used, 45-90 minutes (1-2 class periods) on Building a Graphing App*
  • If used, 90-120 minutes (2-3 class periods) on Improving Your Graphing App*

Lab Pages

  • Page 1: Creating Bar Graphs
    • Learning Goals:
      • Learn how to divide a difficult task into smaller manageable tasks.
      • Learn how to use abstraction to create versatile tools that can work beyond their original context.
      • Understand bar graphs as fundamental tools of data visualization.
      • Learn basics of data processing.
      • Learn how to use labels in Snap!.
      • Develop skills at scaling a graph.
    • Tips:
      • It may be beneficial to start this project with a discussion on climate change and CO2 pollution.
      • It will take several days to complete the Bar graphs project. Most students will be greatly challenged. Be sure to do this project on your own first, so you can anticipate their questions and be prepared to help them.
      • Make sure that, at the end of the project, the students test the versatility of the data visualization tool they have created by inputting a new data set (at least the given CO2 Emissions per capita data) and observing the bar graph adjust to the new information in terms of scaling, labels etc.
      • The versatility of the general data visualization tool students create can demonstrate the great power of computing, so make sure that there is enough time at the end of the project for experimentation and reflection.
  • Page 2: Building a Graphing App.
    • Learning Goals:
      • Create a graphing program that plots data points.
      • Work with a scaling feature to adjust screen dimensions.
    • Tips:
      • Most students will have difficulty understanding the difference between Snap!'s stage coordinates and graph coordinates associated with a given data. The need to scale the graph to fit the Snap! stage will be foreign to them. Be sure to do this project on your own first so you can anticipate their challenges and help them.
      • Discussion: Toward the end of class, you may wish to discuss:
        • Why might you want to set the scale of the screen?
        • Describe how this setup works to set the scale of the data on the stage?
  • Page 3: Improving Your Graphing App.
    • Learning Goals:
      • Use their graphing program to plot and analyze data points.
      • Isolate and graph various data subsets.
      • Use visualization tools to communicate new information.
    • Tips:
      • Discussion: After students work through the lab page, discuss this question as a group:
        • What can you say about this particular class based on the data? Have students share and describe some of the graphs they created to support their observations.

Correlation with AP CS Principles Framework 

Essential Understandings:

Students use their own graphing program to visualize data that they process to gain knowledge and insight about a situation (EU 3.1). They will later use it to visualize the data they get when they time their algorithms.

This lab builds toward this EU by having students develop a data graphing tool as a way to gain insight into information. Students are required to program a graphing app that plots arbitrary (x, y) data from a list and can also accept a function as input and graph the values taken by the function over a range of inputs.

  • EU 3.1 People use computer programs to process information to gain insight and knowledge.

Learning Objectives:

Students interpret and communicate the results of their data-processing by using appropriate visualizations, notations, and precise language (LO 3.1.3).

  • LO 3.1.3 Explain the insight and knowledge gained from digitally processed data by using appropriate visualizations, notations, and precise language. [P5]

Essential Knowledge:

  • EK 3.1.3A Visualization tools and software can communicate information about data.
  • EK 3.1.3B Tables, diagrams, and textual displays can be used in communicating insight and knowledge gained from data.
  • EK 3.1.3C Summaries of data analyzed computationally can be effective in communicating insight and knowledge gained from digitally represented information.
  • EK 3.1.3D Transforming information can be effective in communicating knowledge gained from data.
  • EK 3.1.3E Interactivity with data is an aspect of communicating.
  • EK 3.2.1A Large data sets provide opportunities and challenges for extracting information and knowledge.
  • EK 3.2.1B Large data sets provide opportunities for identifying trends, making connections in data, and solving problems.
  • EK 3.2.1C Computing tools facilitate the discovery of connections in information within large data sets.
