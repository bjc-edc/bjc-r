Programming Lab 2: Three Key List Operations

This lab introduces students to the map, keep, and combine list blocks. Map performs an operation on each item of a list and reports a modified list. Keep filters a list and returns a list of all items from the orginal list that match a given requirement. Combine performs an operation (such as addition) with every item of a list and returns a single result (such as the sum).

In L1, students encountered FOR EACH, which is useful when you want to use each item of a list without changing that list or producing a new list. Snap! provides two powerful tools for processing lists in ways that create new lists of data. This is the topical focus of Lab 2. Students will apply these tools to their ongoing projects. (Obviously, this can’t be the description unless it’s true. It’d be nice to make it true.) MAP takes a list as input and allows you to produce a new list in which each element of the input has been processed in some way: for example, doubled, or made plural, or replaced by a selection (from some other list) that it somehow indicated. KEEP takes a list as input and lets you create a new list keeping only those items from the input list that match some criterion—like begins with a vowel, or is even, or is less than five characters long, or is a vegetable. You need to write the predicate that does the checking, but then KEEP allows you to apply that predicate to each item in the input list and report the list of elements that make the predicate true.
