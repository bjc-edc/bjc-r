Former purpose ramblings

This lab does not introduce new ideas, but gives students practice applying MAP, KEEP, and COMBINE in several contexts, in order to master the use of these powerful tools and to learn to think in terms of what they can do.

Lists of same-size lists can function as tables or matrices. Tic Tac Toe... Magic Squares gives students a game context in which to practice further their use of lists and list processing functions.

Anything that can be handled by creating and manipulating a list can also be done without lists, just by working with the individual elements and keeping track of that process. But list processing—especially list processing with the higher order functions map, keep, and combine (and, later in this course, recursion)—is powerful. It lets the programmer focus on the purpose and meaning of the program—what the program is intended to do—and not on tiny local details of how to do it, details that essentially reinvent a clumsy equivalent of lists and ways to work sequentially through them.

Becoming facile with list processing takes practice, and initially requires contexts in which the meaning of the lists and the purpose and meaning of the functions that process them are clear. One subtlety of working with lists is that it requires thinking simultaneously about the list as a single object and, at the same time, recognizing that much of what one wants to do with the list involves processing some or all of the elements of the list. That is also part of the subtlety of the higher-order functions, which take an entire list as a single object and handle the individual elements of the list in various ways, as determined by the programmer. Composing higher order functions is yet more powerful, but also more challenging, because each one does so much that the programmer needs to picture large effects and the interactions of these large effects.

This lab gives students practice applying map, keep, and combine in several contexts to build their facility with these powerful tools and to learn to think in terms of what they can do. It does not introduce new ideas, but focuses on mastery and application.

Correlation with AP CS Principles Framework 

Essential Understandings:

Students think simultaneously about lists as a single object and, at the same time, as a set of elements to be processed (EU 2.1) as they build encoding/decoding functions and a tic-tac-toe game.

This lab builds toward this EU with a mini-project in which students are required to use lists to represent text to be encyphered or decyphered and the moves in a tic-tac-toe game. Students must also use procedural abstraction; for example, a procedure to test whether player X has won in tic-tac-toe is built out of a procedure that reports a list of cells in which X has moved, a procedure that reports all of the eight winning combinations of squares, and a function that tests whether a list of cells matches a particular winning combination.
