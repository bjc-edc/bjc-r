Lab 5: Big Data

This lab focuses on the remaining content of AP CSP Big Idea 3: Data. Students use existing online data visualization tools to identifying patterns in data and work collaboratively to draw conclusions and summarize their findings. Several AP-style self-check questions are included to help students extend their understanding to the type of questions likely to appear on the exam.

Pacing

The 4 lab pages could be split across 2-5 days (75-190 minutes). Expected times to complete follow:
  • 30-90 minutes (1-2 class periods) on Analyzing U.S. Baby Names
  • 25-50 minutes (about 1 class period) on Analyzing Global Health Data
  • 10-20 minutes (<1 class period) on Self-Check: Big Data
  • 10-30 minutes (<1 class period) on Metadata

Prepare

  • Students will be using the Baby Name Voyager on page 1. This page should load correctly, but check in advance that the site will be usable by students.

Lab Pages

  • Page 1: Analyzing U.S. Baby Names.
    • Learning Goals:
      • Investigate a large data set using an interactive visualization tool.
      • Learn about the challenges and opportunities when working with large data sets.
      • Discover and solve problems through analysis of large data sets.
    • Tips:
      • Discussion: What are some ways to visualize large data sets?
        • Can you think of any other ways the Baby Name Voyager data might be presented?
        • What new information is gained from the Baby Name Voyager visualization that would otherwise be difficult to learn?
        • What visualizations might be possible for a different type of large data set?
      • Discussion: How can you work with large data sets?
        • Where is the information in large data sets stored? How is it accessed?
        • In what ways is it especially difficult to work with large data sets?
        • What large data sets might be part of a student's life?
      • Expect a wide variety of search results from the Baby Name Voyager, and look for opportunities for students to share interesting data with others. Of particular interest may be "fad" names that were popular in specific decades (Lisa, Jennifer), and other names that were popular long ago but are re-emerging (Emma).
      • The Baby Name Voyager site frequently becomes unresponsive if no result is found. When this happens, ask students to reload the page.
      • Some students may get frustrated by not finding their own name in the Baby Name Voyager. Point out to students that only a small percentage of all names are listed. A larger text file (available here) contains all names for at least 5 births. You might check this file to see if your students' names are in it—or even challenge a student to create an app that tests names against this data.
      • You may want to draw some connections to prior units' social implications labs, since there have been discussions about large data sets, notably a discussion about privacy concerns in personally-identifiable information.
      • During this lab, keep a lookout for news reports that involve the analysis of large data sets, for appropriate Computing In The News material.
  • Page 2: Analyzing Health Data.
    • Learning Goals:
      • Investigate and analyze a large data set.
      • Draw and summarize conclusions based on investigation and analysis of a large data set.
    • Tips:
      • You can find similar interactive data sets from Google at google.com/publicdata/directory.
  • Page 3: Self-Check: Big Data.
    • Learning Goal: Prepare for data questions on the AP CSP exam.
    • Tip: These questions are worth considering even for students not planning to take the exam.
  • Page 4: Metadata.
    • Learning Goals:
      • Understand the meaning of the term and several uses of metadata.
      • Prepare for metadata questions on the AP CSP exam.
    • Tips:
      • You may choose to assign this page—particularly watching the video—as a homework assignment.
      • There is an alternate link for the video that may work in schools that block YouTube access.

Correlation with AP CS Principles Framework 

Essential Understandings:

  • EU 3.1 People use computer programs to process information to gain insight and knowledge.
  • EU 3.2 Computing facilitates exploration and the discovery of connections in information.

Learning Objectives:

  • LO 3.1.3 Explain the insight and knowledge gained from digitally processed data by using appropriate visualizations, notations, and precise language. [P5]
  • LO 3.2.1 Extract information from data to discover and explain connections or trends. [P1]

Essential Knowledge:

  • EK 3.1.2F Investigating large data sets collaboratively can lead to insight and knowledge not obtained when working alone.
  • EK 3.1.3A Visualization tools and software can communicate information about data.
  • EK 3.1.3B Tables, diagrams, and textual displays can be used in communicating insight and knowledge gained from data.
  • EK 3.1.3C Summaries of data analyzed computationally can be effective in communicating insight and knowledge gained from digitally represented information.
  • EK 3.1.3D Transforming information can be effective in communicating knowledge gained from data.
  • EK 3.1.3E Interactivity with data is an aspect of communicating.
  • EK 3.2.1A Large data sets provide opportunities and challenges for extracting information and knowledge.
  • EK 3.2.1B Large data sets provide opportunities for identifying trends, making connections in data, and solving problems.
  • EK 3.2.1C Computing tools facilitate the discovery of connections in information within large data sets.
  • EK 3.2.1F Software tools, including spreadsheets and databases, help to efficiently organize and find trends in information.
  • EK 3.2.1G Metadata is data about data.
  • EK 3.2.1H Metadata can be descriptive data about an image, a Web page, or other complex objects.
  • EK 3.2.1I Metadata can increase the effective use of data or data sets by providing additional information about various aspects of that data.
  • EK 3.2.2A Large data sets include data such as transactions, measurements, texts, sounds, images, and videos.
  • EK 3.2.2B The storing, processing, and curating of large data sets is challenging.
  • EK 3.2.2C Structuring large data sets for analysis can be challenging.
  • EK 3.2.2E Scalability of systems is an important consideration when data sets are large.
  • EK 3.2.2G The effective use of large data sets requires computational solutions.
