Lab 2: Tic-Tac-Toe

In this lab, students will delve into the next phase of the Tic-Tac-Toe project where they will teach the computer how to remember the moves made on the board by the users and how to detect a win or a tie. The capacity to remember moves and the ability to analyze them will lead the way to the last phase of the game where the computer will become an intelligent player.

Pacing

The 2 lab pages could be split across 3-4 days (110-160 minutes). Expected times to complete follow:
  • If used, 30-40 minutes (1 class period) on Remembering the Moves*
  • If used, 80-120 minutes (2-3 class periods) on Detecting Wins and Ties*

Prepare

  • This is a challenging lab, make sure that you have mastered both pages yourself so you can be of help to the students.

Lab Pages

  • Page 1: Remembering the moves.
    • Learning Goals:
      • Learn how to extend an existing project to meet more complex specifications.
      • Learn how to create sprite-local variables in Snap!.
      • Understand how to use global variables to shift the focus of a program from a local view to a global view.
      • Gain experience with lists.
    • Tips:
      • The purpose of most of the preparatory activities on this page may not be immediately clear to students. Why do we need to have each square sprite have their own unique square number variable? Why do we create a global board variable when each sprite already knows if it has been clicked? The answers to these questions and more should start becoming clear in the next page where students will learn how to make the computer aware of the global state of the game in order to detect wins and ties.
  • Page 2: Detecting Wins and Ties.
    • Learning Goals:
      • Refine the Tic-Tac-Toe game to declare a winner or announce a tie.
      • Gain experience with the higher order function map and have a taste of it enormous power to simplify code.
      • Understand how having a global view of the game gives the program the ability to analyze moves.
    • Tips:
      • Some students may try to determine if there has been a win in the game by using eight nested if then conditionals as indicated below. Encourage them instead to understand the elegance and simplicity of using map in solving this problem.
      • Once the students have written the status of winning triples block, it will be beneficial for them to compare the status of each winning triple to the corresponding configuration on the board. For example the eighth entry is the status of the diagonal with squares 3,5,7 and we see the status of winning triples block reporting {X, Empty, Empty} agreeing with the board configuration.
      • The elegant solution with map will use the block to rapidly test whether the status of a triple implies a win. It will not be obvious to most students that this block can have a list of lists as an input as shown below.
      • Refining this game builds on students' understanding of lists and list operations including higher order functions. This is likely to feel very satisfying to students, a major accomplishment. It is!

Solutions

Correlation with AP CS Principles Framework 

Essential Understandings:

Students think simultaneously about lists as a single object and, at the same time, as a set of elements to be processed (EU 2.1) as they build encoding/decoding functions and a tic-tac-toe game.

This lab builds toward this EU with a mini-project in which students are required to use lists to represent text to be encyphered or decyphered and the moves in a tic-tac-toe game. Students must also use procedural abstraction; for example, a procedure to test whether player X has won in tic-tac-toe is built out of a procedure that reports a list of cells in which X has moved, a procedure that reports all of the eight winning combinations of squares, and a function that tests whether a list of cells matches a particular winning combination.

  • EU 2.1 A variety of abstractions built on binary sequences can be used to represent all digital data.

Learning Objectives:

Students compose higher order functions to develop a tic-tac-toe script that checks the board for wins (LO 2.2.2).
