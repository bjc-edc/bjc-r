From Y2U3L1

Computers manipulate data and return processed information to the user. Numbers, strings of characters (letters, numbers, symbols, etc.), and Boolean values (true, false) are some of the basic pieces of information. Often, these small pieces are arranged in more complex structures: lists, ordered sets of data.

A list is not just a collection (set) but also a collection in a particular order (this order is tracked by an index number for each item on the list). If it has any elements at all, it has a first element, a last element, and so on. A list can also be totally empty, with no elements; often that is the way we want to set up (initialize) a list before we add elements to it. The length of an empty list is 0.

Lists are powerful. Databases are lists of lists of lists. Many operations can be performed on data in a list:

  • Functional programming:
    • Searching a list to see if it contains some element.
    • Creating new lists that keep only certain items from a list you started with.
    • You can create new lists with more or fewer elements than the list you have.
    • You can pick an item of a list by position or characteristic or just randomly.
  • Imperative programming:
    • Modifying lists to add, delete, or replace items, or to insert an item in a specific spot.
    • You can do something with each item of the list: you can process it in some way, or for each element of the list, you can display it in some way.

From Y2U3L2

Lists can contain anything: numbers, strings, other lists, even blocks. The objects in a list can be all of the same kind (all numbers, for example, or all lists that contain a name and a phone number) but they do not have to be. A list can, for example, contain an apartment number and a list of people at that address. The tabular data we are familiar with in spreadsheets can be represented as a list of same-length lists, the smaller lists representing each row in the table, and the enclosing list representing the entire spreadsheet.

The "ordered pairs" of algebra, the (x, y) coordinates of a point, can be represented as lists. So can vectors and matrices. The list block is the basic building block of these complex structures. To select information in the dataset, we only need to know where it appears.

Blocks can locate items in the nested lists. For example, item (3) of the list of points below gives {10, 120}. As the sprite visits each of these points (in a for each block) the point (10, 120) is the third that it visits.

Students start by constructing a very simple contact list app. The list's elements are themselves lists. This requires not only collecting and storing information, but being able to retrieve it in useful ways. Students use keep to be able to collect only the information they need from a list, and learn to use for each to process lists of points on the plane—points which are themselves lists of coordinates. Data Types, a later page in this lab, returns to the idea of using abstraction to make the (x, y) coordinate lists more recognizable and usable by creating the abstract data type point and constructing and identifying these coordinate lists that way. After some embellishment of the contact app, practicing and extending the ideas of abstraction, and a Lists Review and Self Check, students extend their use of for each and data types to a musical context.

Thus in Labs 1 and 2, students have encountered lists of strings/words, numbers, and lists. They also use abstraction to clarify and simplify code, and see how it makes code more flexible and maintainable.

Lab Pages

  • Page X: .
    • Goals:
      • Use nested list data structures to create abstract data types, which let a user deal with data without having to know the details of how the data are stored and accessed.
      • Use the keep function to filter lists according to particular constraints.
    • Tips:
      • Discussion: You might ask these questions:
        • What kinds of things can you do with a list of lists?
        • Do the lists inside a list need to be the same size?
        • In what other situations might a list of lists be helpful?
      • The name from number and especially phone number of blocks require students to think carefully about accessing items within lists of lists, offering opportunity for them to practice working in these structures and motivating the use of abstract data types.
      • After most students have completed the "Create the Abstract Data Type" section, you may wish to pause to review what an abstract data type is and the role of both constructor and selector.
      • The Contact List project is ambitious but carefully scaffolded. It's helpful for you to remember its ultimate goal, which is to introduce the idea of data abstraction, in which an abstract data type is defined by writing constructor and selector blocks that are the sole program interface to the underlying list structure. We start without data abstraction, so you see lots of item 2 of when a phone number is wanted. By the end of the project, there should be no more item calls in the user interface blocks. Students will be inclined to resist data abstraction, because it makes the program longer. But the project uses a change in the underlying representation to convince them that it's much easier to maintain a large program if such changes affect only the constructor and selectors.

Assignments

  • Assessment Idea: Write a program that keeps track of each student's grade in one class.
    1. Create abstract data type to hold each student record:
      • Create a student record constructor—a reporter block that, given a name (a text string) and grade (a number) as input, reports a student record (a list of that name and grade).
      • Create two selectors name and grade that, given a student record as input, report the name or grade for that student record.
    2. Make a variable class that, when the green flag is clicked, is set to an empty list that will store student records.
    3. Add a button (sprite) that uses your constructor to allow the user to create a student record by entering the student’s name and grade.
    4. Add a button that generates a grade report. Use the for each block to go through each record in class and have the sprite say the student’s name and grade for a second. Format the output so that it makes sense to the user.
    5. Submit your code in whatever way your class submits work.
