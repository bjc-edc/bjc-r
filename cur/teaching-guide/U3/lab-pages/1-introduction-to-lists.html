Tips:

  • Getting Started:
    • This page asks students to build the blocks (rather than providing a link that has the blocks built already). This is intentional, so that students get the experience of finding these blocks in the palette and getting the logic of what blocks go inside of other blocks.
    • Problem 2 is subtle. The first block reports the string "Boise", whereas the second block reports a list containing the item "Boise". If you don't hear students making this distinction, consider reviewing this problem as a class. Distinguishing the logic between what data type is reported is a big help when debugging.
    • Problem 3b gives the result "NaN". This stands for "Not a Number" and results because the sum operation expects numbers as its inputs, and 3b has included a list as one of the inputs.
    • Problem 5 includes a list within a list.
  • Many of the snap projects linked from the student pages already have the tools imported.
    Throughout this Unit, students will have to Import Tools in order to access the blocks they need. This is required every time a student opens a new snap window.
