Programming Lab 3: Three Key List Operations

Functions that can accept other functions as input are called higher order functions. In Lab 1, students worked with the higher order function for each to access each item of a list. Now students focus on three higher order list operations: map, keep, and combine.

While for each is a command block that steps through each item in a list to use or make changes to the items, combine reports a single result, and map and keep both report new lists of data:

  • Map performs a function on each item of a list and returns a modified list. That is, map takes a list as input and reports a new list in which each element of the original list has been processed in some way, for example: doubled, made plural, replaced by something (like a selection from some other list), etc.
  • Keep filters a list and returns a list of all items that match a given requirement. That is, keep takes a list as input and reports a new list keeping only those items from the original list that match some criterion, for example: begins with a vowel, is even, is less than five characters long, is a vegetable, etc.
  • Combine performs one operation between all the items of a list and returns a single result (e.g. a sum, product, concatenated string, etc.). That is, combine takes a list and a binary (i.e., two-input) operation as input and reports the result of performing that binary operation between each of the items in the list. (Like addition, multiplication, and join, combine is associative; however, even non -associative binary operations can be used with combine, for example: subtraction would just alternately cancel itself with each list item.)

These three list processing operations give enormous power all by themselves. In combination, they allow very sophisticated data processing. Students will apply these tools various context including an overarching theme of list of lists to manage and organize data in apps (shopping list, contact lists, tic tac toe, etc.).

Pacing:

The 4 lab pages could be split across 5 to 7 days. Expected times to complete follow:
  • The introductory page, Exploring List Operations, are expected to be relatively quick, less than 1 class period.
  • The Map, Keep, and Combine lab pages each have several sections with different (if any) project files and a quiz.
    • Nesting Functions may take about a day.
    • Map may take 2-3 days.
    • Keep may take 1-2 days.
    • Combine may take 1-2 days.

Students will:

  • Learn about higher order functions.
  • Learn to use for each, map, keep, and combine on their own and together to access and manipulate list data.

As a Class:

  • Page 2: Map.
    • The section, Using Map for Algebraic Functions, presents an example of using Snap! to model this algebraic function: $$f(n)=3n+7$$ Encourage students to identify a function (or several) of interest to them and build a reporter to run it. For example, they might calculate values of a function needed for a math or science assignment (or any other school or non-school context).
  • Page 4: Combine.
    • The constraints of the functional input for combine are straightforward (an associative operation with two blank input slots), but as they are different from map and keep (which require input functions and predicates, respectively, with one blank). You may wish to have a discussion to discuss the combine input requirements and the issue of associativity, which is also addressed in the student lab pages.

Assignments:

  • Encourage students to identify places where map, keep, and combine may help with a program with lists that they have already developed and to take the time to improve their programs.

Tips:

  • Page 1: Exploring List Operations.
    • Problem 1 part b gives the result "NaN". This stands for "Not a Number" and results because the sum operation expects numbers as its inputs, and this expression includes a list as one of the inputs.
  • Page 2: Map.
    • Watch for students who are particularly adapt with translating, dilating, and reflecting computer graphics or with composing and transposing music. Encourage these students to develop these projects further and/or to present their work to the class.
  • Page 3: Keep.
    • Students develop a small database of first name, last name, age (in years), and height (in inches). Encourage students to keep the dataset small (4-6 records) to minimize time lost on data entry.
  • Page 4: Combine.
    • The two quizzes on this page address all three higher order functions in this lab: map, keep, and combine. Encourage students to use these questions to assess how well they have learned these ideas. If necessary, they may wish to review.
