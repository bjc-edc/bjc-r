Programming Lab 3: Three Key List Operations

Higher order functions accept other functions as input. In Lab 1, students worked with for each, which is useful when you want to use each item of a list without changing that list or producing a new list. This lab focuses on three higher order list operations: map, keep, and combine., two of which, map and keep create new lists of data. These three list processing operations give enormous power all by themselves. In combination, they allow very sophisticated data processing. Students will apply these tools various context including an overarching theme of list of lists to manage and organize yearbook data.

Map performs an operation on each item of a list and reports a modified list. That is, map takes a list as input and allows produces a new list in which each element of the input has been processed in some way: for example, doubled, or made plural, or replaced by a selection (from some other list) that it somehow indicated.

Keep filters a list and returns a list of all items from the orginal list that match a given requirement. That is, keep takes a list as input and creates a new list keeping only those items from the input list that match some criterion—like begins with a vowel, is even, is less than five characters long, or is a vegetable.

Combine performs a binary (two-input) operation (such as addition) with every item of a list and returns a single result (such as the sum). That is, combine takes a list and an operation as input and reports the result of performing that binary operation between each of the items in the list (like addition, multiplication, and join, combine is associative).
