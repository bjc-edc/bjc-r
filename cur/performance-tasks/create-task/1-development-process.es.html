Uso de un proceso de desarrollo para organizar tu código

En la práctica de la Create Task (tarea de creación), te familiarizarás con las pautas y el vocabulario de esta tarea y practicarás la misma con la orientación y los comentarios de tu profesor.

Cuando completes la Create Task real, no podrás obtener orientación ni comentarios de tu profesor.

En esta página, reflexionarás sobre tus hábitos al codificar y planificarás el proceso de desarrollo que utilizarás para la Create Task.

La Create Task es una oportunidad para que envíes algo creativo al College Board para incluir en tu puntuación de AP CSP: un programa que diseñes (con o sin un compañero), un video del programa mientras se ejecuta (preparado por tu cuenta), segmentos de tu código en la Personal Project Reference (referencia del proyecto personalizada) (preparada por tu cuenta) y varias respuestas por escrito sobre tu programa y tu proceso de desarrollo (escritas durante el examen de AP).

CRD-2.E.1, CRD-2.E.2

El proceso que utilices para desarrollar tu programa puede ser organizado y estratégico o puede ser un proceso de exploración. Hay diferentes formas de programar. Aquí hay algunas fases comunes por las que pasa un programador:

    CRD-2.F.1
  • Investigar y reflexionar: averiguar los requisitos (lo que el usuario quiere que haga el programa)
  • Diseño: planificación de cómo se verá la aplicación para el usuario y el enfoque general que usarás para codificarla.
  • Creación de prototipos: creación de un borrador del programa
  • Pruebas: probarlo, compartirlo con otros para obtener comentarios, corregir cualquier error y realizar las mejoras necesarias

CRD-2.E.3, CRD-2.E.4

Tu proceso de desarrollo puede ser iterativo (involucrando varios ciclos de desarrollo y luego pruebas y comentarios) o puede ser incremental (lo que significa que construyes y pruebas partes más pequeñas del proyecto antes de combinarlas).

  1. Piensa cómo escribes tu código. ¿Tiendes a codificar iterativamente (codificando y luego probando y luego codificando nuevamente, etc.)? ¿Tiendes a codificar de forma incremental (construyendo y perfeccionando una pieza a la vez)? ¿Hay algún problema de desarrollo con el que te hayas encontrado una y otra vez? ¿Hay alguna forma de cambiar tu proceso de desarrollo para evitar esos desafíos?
    2. CRD-2.E: Develop a program using a development process.
  2. Antes de comenzar la Create Task, decide qué tipo de proceso de desarrollo utilizarás. Escribe tu plan. No pasa nada si cambias el plan a medida que avanzas, pero toma nota de los cambios que le hagas.
