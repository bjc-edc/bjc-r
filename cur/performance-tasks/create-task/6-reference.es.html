Preparar tu Personalized Project Reference

En esta página, prepararás tu Personalized Project Reference sheet (hoja de referencia del proyecto personalizada) basada en tu proyecto de Snap! para utilizar en el examen de AP.

Hay dos lugares diferentes donde enviar imágenes de tu código al College Board:

  • Para el Componente A (código del programa), enviarás una imagen de todo tu código, incluyendo comentarios y referencias.
  • Para el Componente C (Personalized Project Reference), enviarás imágenes de partes específicas de tu código y no puedes incluir comentarios.
Durante la parte de respuesta escrita en el examen de AP, solo puedes utilizar lo que está en tu Personalized Project Reference sheet.

  1. Sigue las pautas provistas por el College Board para identificar dos segmentos de código de procedimiento y dos segmentos de código de lista de tu proyecto. Aquí tienes una revisión de las pautas, pero siempre confirma los requisitos con el College Board:
    1. Los dos segmentos de código de procedimiento deben mostrar:
      1. Un procedimiento desarrollado por el estudiante que muestre el nombre del procedimiento, el tipo de datos devuelto por el procedimiento, si lo hay (como número, texto, lista, etc.), la entrada o entradas del procedimiento y su uso, y el algoritmo dentro del procedimiento que incluye secuenciación, selección e iteración.
      2. Una sección de tu programa que llame a ese procedimiento.
    2. Los dos segmentos de código de lista deben mostrar:
      1. Una sección de tu programa que muestre cómo los datos han sido almacenados en una lista y cómo se llama esa lista.
      2. Una sección de tu programa que muestre cómo los datos en esa misma lista se utilizan para el propósito del programa.
  2. Crea tu Personalized Project Reference sheet con los cuatro segmentos de código que identificaste anteriormente.
    Durante el examen de AP, tu escuela proporcionará una impresión de tu hoja Personalized Project Reference para que la uses en el examen. Para la Create Task de práctica, si te das cuenta que no puedes responder a algunas de las preguntas de escritura en la página siguiente (Write About Your Proyect, Escribir sobre tu proyecto) utilizando solo los cuatro segmentos de código que seleccionaste, entonces puedes revisar tu Personalized Project Reference . En el examen de AP real, no podrás revisar tu Personalized Project Reference, así que utiliza la Create Task de práctica para pensar cómo debe ser un buen segmento de código para incluir en tu Personalized Project Reference sheet. En la tarea de creación oficial, cuando crees tu Personalized Project Reference para el examen de AP, piensa de antemano si podrás responder correctamente a las preguntas de la página siguiente.

    3. Mostrar el código

    Snap! tiene dos formas convenientes de obtener archivos de imagen que muestren parte o todo tu código:

    • Crear imágenes de programas

      Puedes descargar un archivo de imagen PNG de cualquier bloque o programa haciendo clic en él con el botón derecho del ratón (o presionando con la tecla Control y haciendo clic en él en una Mac) y seleccionando "imagen de programa ..." Busca el archivo de imagen en la carpeta Descargas del navegador.

      Si haces clic pulsando a la vez las teclas Shift y la flecha a la derecha (o Shift y Control) en un bloque reportero, puedes obtener una imagen de programa del reportero con la burbuja que muestra lo que está reportando.

    • Exportar todas las imágenes a la vez

      Puedes descargar un archivo HTML de todos los bloques de todo tu proyecto, así como una imagen del escenario seleccionando "Exportar resumen ..." en Snap! Menú Archivo (). El archivo HTML también se guarda en la carpeta Descargas del navegador. Ábrelo (en un navegador), haz clic con el botón derecho (o haz clic con la tecla Control presionada en una Mac) para copiar las imágenes que necesitas y pégalas en el documento donde estás describiendo tu código.
