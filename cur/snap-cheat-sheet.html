Snap! Crash Course

Once you've learned how to program in one language (at least any language that your AP Computer Science Principles class would have used), it's much easier to learn another language. And Snap! will be especially easy to pick up quickly because it's all visual; you just snap blocks together, and you don't have to worry about any syntax (parentheses, commas, semicolons, quotation marks, etc.).

This page is designed to get you up and running in Snap! as quickly as possible. It will help you pick up Snap! quickly if you take the time to read (or at least skim) this whole page.

Creating Variables

There are several kinds of variables in Snap!. Here are the most important:

  • Global variables can be accessed from anywhere in your program. You can create them by clicking the button in the Variables palette.
  • Local (script variables) only exist within the script that creates them. You can create local variables using the block. Place that block at the point in your code where you want to create a variable, click the variable name to change it if you want, click the right arrow (▸) if you want to create additional variables at the same point in your program, and drag the variable off to use it in the code below it.
  • Input variables (parameters) store the input values (arguments) passed to a custom block. For example, size is an input variable to the draw square block shown at right. There is more information about creating blocks with input parameters below.
  • Iteration variables change in value as a block repeats a set of commands. You can change the name of an iteration variable by clicking on it. In Snap!, there are two iteration blocks that have built in variables: for uses i (unless you change the name) to hold the index number of the iteration, and for each uses item (unless you change the name) to hold each list item as it traverses (travels through) the list:

Creating Custom Procedures

To program your own block:

  1. Create a new block. There are several ways to start: click near the end of the palette, click the + sign in the top right corner of the palette (shown below), or right-click (or control-click on a Mac) on any empty spot in the scripting area and choose "make a block..." (shown below).
  2. Choose the color (palette), type the title, select the shape (block type), and click OK.
    • Just as you can use multi-word variable names in Snap!, you can use multi-word names for your blocks (custom procedures).
  3. Program your block. Drag in blocks from the palette, snap them in after the hat block, and click OK.
    • Note that if you create a reporter or predicate block, the block editor will open with a block already attached to the hat block. All of your block code will go between these two blocks or inside the report block; whatever input you give to report will be the value reported (returned) by your new block.

Grouping Data: Lists/Arrays/Etc.

Snap! uses lists to group a bunch of values. If you learned to program in another language, you may have used arrays. In Snap!, lists are ordered, and the first item in a list is item number 1. (Some languages have the first the first item be 0, and the second is 1, the third is 2, etc.). Also in Snap!, list items do not all have to be the same data type. For example, a single list can contain text, numbers, images, sounds, blocks, and even other lists.

Additional Programming Topics

Snap! Libraries

Most languages have libraries of operations you can apply to lists: combine two lists, sort a list, etc. Ours is called "List Utilities."

Higher Order Functions

Snap! provides other ways to deal with lists: higher order functions and recursion. HOFs let you consider a list as a single value, dealing with its items all at once instead of singling out a specific item by its index: MAP... KEEP... FIND FIRST... COMBINE...

Recursion

Recursion lets you deal with a list by left-to-right traversal, but without the use of index variables. Instead, you split the list into its first item and a sublist with all but the first item: ITEM 1 OF foo ALL BUT FIRST OF foo You compute some function of the first item, and combine that with a call to the same procedure you're writing: PLURALS words IF REPORT (LIST) ELSE REPORT (JOIN (ITEM 1 OF words) s) IN FRONT OF (PLURALS (ALL BUT FIRST OF words)) ----------- end of blowing our own horn -----------
