Sharing Your App

  1. Make sure your project is saved.
  2. Then, choose "Open..." from the File menu, select the file from the project list by clicking on its name (in this lab, "U1L1-ClickAlonzo"), and click "Share."
  3. Then, say "Yes" you want to publish. (You can now cancel or open. It doesn't matter.)
  4. Copy the URL (web address) from the top of your browser window. You can use that link to share the app.
    You can also make a tinyurl for your game to make typing the URL easier.
    5. Here's a video that shows how:
  5. Log out and let your partner log in. Open the link you created with the first partner's account, and save the project in the second partner's account so that you both have a copy.
    Note: these are copies, not the same file. So, if you change the project in one person's account, it will not change in the other person's account; you will need to share it again.

At home—or in class, if you have time—load the app on your mobile device.
  • On an Apple iOS device:
    1. On your mobile device, open the project URL from step 3 above in Safari.
    2. At the bottom of the Safari screen, tap the share icon, then select "Add to Home Screen" from the menu that appears.
  • On an Android device:
    1. On your mobile device, open the project URL from step 3 above in Chrome.
    2. Below the screen, tap the ☰ settings icon, then select "Add to homescreen" from the menu that appears.

Now, you can open and play your game on your mobile device.

Improve the game:
  1. So far, Alonzo waits in one place to be clicked. Add code so that after the first time Alonzo jumps to a new location, Alonzo always waits 1 second and jumps again to a random position (even if not clicked).
    Translating ideas into code:
      "always...waits 1 second...and jumps again"
    • "Always" means "do it forever," and uses the . The code that you want to repeat forever will snap inside the forever block.
    • "Waits 1 second" uses . This will go inside the forever block.
    • "and jumps again" uses another copy of the code you have already used for jumping. This will go inside the forever block.
    • What does "after the first time Alonzo jumps to a new location" tell you about where to attach this new bit of code to your original script?
The game on your phone or tablet updates automaticaly, when you improve the game and save it.
