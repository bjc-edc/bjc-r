In Snap!, a sprite can have more than one script controlling its behavior. You currently have just one script, which runs when you click on Alonzo. Now you'll make a second script that runs when you click on the green flag. The new script will get Alonzo to welcome the player to your game.

Clicking the green flag at the top right, , is one way to start a Snap! script. You can stop all running scripts by clicking the red stop sign, .
    Hint: You can do this using just 3 blocks!
  1. Create a second script that makes Alonzo move to (190, -130) and say a welcome message whenever the green flag is clicked. (Alonzo might say, "Welcome to Click On Alonzo! Keep clicking me if you can!") .
  2. Make that message disappear when the player clicks on Alonzo.
    Using the block with an empty input will make the Alonzo sprite stop saying anything.
  3. Check that your app does everything you intend it to do:
    • When the green flag is clicked, Alonzo should move to (190, -130) and say a welcome message.
    • When Alonzo is clicked, the message should disappear, and Alonzo should move to a random spot on the stage and face the other way.
 
  1. So far, Alonzo waits in one place to be clicked. Increase the game's difficulty by adding code so that after the first time he is clicked and from them on, Alonzo moves to a random position on the stage every two seconds (even if not clicked).
    Translating ideas into code: "Every 2 seconds" means wait 2 seconds and repeat.
    • What does "from then on" translate to in Snap! programming?
    • What does "after the first time Alonzo is clicked" tell you about the structure of the code?
      • Snap! scripts can run at the same time, but within each script, the steps run one at a time.
  2. Give the player some sense of how well they are doing:
    1. Put in the script that runs when the player clicks on Alonzo. This makes Alonzo fade out a little every time he's clicked.
    2. Put another block in your program to change the ghost effect by -5 every time Alonzo moves without being clicked.
    3. Add a reset button by creating a sprite with a reset button costume (made in Snap! or using an imported image) that will clear graphic effects or set ghost effect to 0 when clicked.
