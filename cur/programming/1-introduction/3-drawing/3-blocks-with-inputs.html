Blocks with Inputs


On this page: As an example of Abstraction, you will turn a collection of related scripts into a single block that takes an input. Giving blocks inputs lets them do a wider variety of tasks.

  1. Start by loading this file. Make 5 copies of the script and modify those copies so that they make pinwheel designs like these with 3, 4, 5, 6, 12 branches.
The scripts are essentially all the same; only the numbers determining the number of branches change. So, instead of having many scripts, you can make one block that will draw all such designs. Your pinwheel block will need an input for the changing number of branches, just like the go to block has inputs for the changing x and y values .

  1. To create a block with an input:
    • Control-click (right-click) on some empty spot in the scripting area. You will see this menu. Select "make a block..."
    • You will see a dialog box. In that title slot, type "pinwheel, branches:   " and because this block is about moving your sprite, click the blue Motion button. It will turn fully blue:
    • Then click OK. That will open the block editor, which you've seen before. You should see:
    • Drag one of your pinwheel scripts into the editor and snap it to the title block.
    • You don't always want 21 branches. So you want to give this pinwheel block an input. Click on the + (plus sign) immediately following the word "branches: "
    • Click on the "Input name" button and type "number of branches" for the name of your input. Click the "OK" button when you are done.
    • Adjust your code as shown so that the new input "number of branches" replaces the "21" allowing for generality.
  2. Save your work.
