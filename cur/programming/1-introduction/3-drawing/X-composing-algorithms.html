Modify Your Pinwheel

On this page, you will modify your existing pinwheel block so that you can use it to draw a variety of shapes.

Adding Multiple Inputs

Recall how you first generalized your original pinwheel script: you added an input variable called branches that controlled the turning angle of the sprite. By adding more input variables, you can generalize other aspects of your program...

  1. If it isn't already open, open your U1L3-Angles project.
  2. Experiment with the input for the second move block inside your pinwheel block as shown below.
    1. First predict. What do you think will happen?
    2. Then try several inputs between -100 and 0.
    3. What happened? How does it compare to what you predicted? How does this input value impact the sprite's behavior? How does it impact the resulting image on the stage?
  3. Add a second input variable to control the amount of "backing up" that the sprite does before each turn through the full 360°.
    1. First, go to the Settings menu (), and make sure that "Plain prototype labels" are un-checked. (This will allow you to see the "+" signs in the next step.)
    2. Now, edit your pinwheel block, click the "+" sign at the end to add another input to this procedure, and call the new variable backup.
    3. Drag in the backup variable, place it where it belongs in the pinwheel code, and press "OK" or "Apply."
    4. Check that the new input slot in the pinwheel block behaves like the changes in your experimentation above.
    Hint: Use an Operator block.
  1. Change the pinwheel script so that it will accept a positive value (between 0 and 100) for backup.
  1. Add a third input called size to control the input to first move block.
  2. Try out a variety of inputs to your pinwheel program...
    If you did not have time to make pinwheel accept positive values for backup, you will need negative values in the third input slot.
  3. Find inputs to pinwheel that make the result look like a circle.
    4. You may want to reduce the input value for the wait blocks inside the pinwheel script so you don't have to wait so long between trials.

Hiding the Complexity

You can create use your general-purpose pinwheel block to create other, simpler special-purpose blocks that each do a particular job.

  1. Create a specialized block that draws a polygon using specific inputs to the pinwheel block. The polygon block should accept a number of sides and a size as input.
  2. Create a specialized block that draws an asterisk (*) using specific inputs to the pinwheel block. The asterisk block should accept a number of branches and a size as input.

Important Point

Designing a general-purpose tool that can be used in many different special-purpose ways is an extremely useful programming practice. Here, instead of programming a polygon block, then an asterisk block, and then a circle block, you only have to code one pinwheel block that you can use to build all the others with just a few modifications.
  1. Create a specialized block that draws a circle using specific inputs to the pinwheel block. The circle block should accept only a size as input.
    2. You may find it helpful to recall that the circumference (C, the distance around the outside) of a circle depends on the radius (r, the distance from the center to the circle). The symbol pi has a fixed value: \pi = 3.14159265...
    C = 2 \pi r
  2. Use the random block together with set pen color and set pen size to draw a variety of shapes.
