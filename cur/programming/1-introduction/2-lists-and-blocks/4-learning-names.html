Using Blocks in Other Programs

Include purple reference to snap manual. --MF, 3/11/17
  1. Once greet player is working correctly, export it.

    Exporting Blocks

    1. Choose "Export blocks..." from the Snap! file menu ().
    2. Select the block(s) you wish to export (in this case, just greet player), and click "OK." An XML file will download.
  2. Open the project where you'll use greet player, and import it.
    You could use greet player in U1L1-ClickAlonzo or U1L2-Gossip.

    Importing Blocks

    1. Choose "Import..." from the Snap! file menu ().
    2. Locate the XML file (perhaps in your downloads folder).
    3. Locate the imported block in the matching palette on the left. (Custom blocks appear at the end.)
    4. Click the imported bock. (You'll likely see an error message.)

      5. Dependencies

      Sometimes when you take a script out of the program where you built it and try to use it somewhere else, something breaks. It could be that the script needs a custom block that isn't defined in the new program, or it could be (like it is here) that the script needs a variable that isn't defined.

Variables look like reporters, and both do report values, but they are different.

  • Variables always give the same value until the value is changed by the user or programmer.
  • Reporters may report different output but always by same process until the process is changed by the programmer.
Reporters always have the same behavior as was coded in at the beginning. Reporter functions it can't be changed by the user but only by a programmer changing the code. Variables, the other hand, can change while the program is running.
