Drawing with Two Sprites

"U1L2-SpriteOrbit"
  1. Analyze and predict: How are the behaviors of these four scripts the same? How are they different?
  2. Create two sprites.
    1. Give Sprite a script that, when started, draws a blue square, drawing each line at some medium speed.
    2. Give Sprite(2) a script that does this:
                      when started
                draw in red
                forever
                  {
                     point towards (Sprite)
                     move (5) steps
                  }
      3. One "line" of pseudo-code may require more than one line of code in the language you are using. For example, "draw in red" requires two snap commands, one so that the sprite draws when it moves, and one to set the pen's color.
    3. Make sure that your program starts both sprites at the same time (for example, with ).
  3. Experiment with the speeds of each sprite.
    1. What happens if Sprite(2) moves 20 steps instead of 5 steps each cycle through the forever-loop?
      What if it moves only 2 steps?
      What if Sprite draws its square more quickly?
    2. Describe your experiments in writing. Include pictures of the screen to show what happened. Try to explain why the picture came out as it did.

Sprite doesn't need to move in a square. Try one or more of the following experiments.

  1. Change Sprite's turn to 144 degrees.
  2. Change Sprite's turn so that it makes an equilateral triangle.
  3. Change Sprite's turn to 10 degrees and make the distance it moves in each step much smaller than 100.
