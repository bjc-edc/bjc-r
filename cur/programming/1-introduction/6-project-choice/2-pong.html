The Game of Pong (Teacher's Choice)

ST-Fix the student access issue due to hideControls
In this project, you are going to model a simple game called Pong, first developed by Atari in 1972. You will write the code for the paddle which will allow it to move up and down based on user keyboard input and also write the code for the ball which will allow it to bounce every time it hits a wall or the paddle.
  1. Play with the Snap! file linked here to get a feel how the game works. Use the up and down arrow keys to move the paddle. The initial angle at which the ball starts is random, so if it is not to your liking, you can reload the page for a different initial angle.
  2. Discuss with your partner the mathematics behind the ball bouncing off the paddle. How does the angle at which the ball hits the paddle relate to the angle at which the ball bounces off the paddle? The following diagrams may help.

  3. Create the paddle and the ball sprites by drawing the costumes on your own and then try to write the code for the paddle and the ball by yourself. If you get stuck, look at this page for hints on how to proceed.
 
 
  1. Add scoring to your game. Every time the paddles bounces off the ball the player should earn a point and every time the ball hits the right wall the player should lose a point.
  2. Create a second paddle to allow the game to become multiplayer game. Adjust your code to accommodate keyboard input from two users.
  3. Create a second paddle and let this paddle be driven automatically by the computer.
