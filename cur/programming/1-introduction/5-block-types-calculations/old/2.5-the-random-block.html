The Random Block

You can use the random block to generate a random number within a range defined by two whole numbers. For example, each time this block is run, it generates, with about equal likelihood, one number ranging from 1 through 10.

Like other reporter blocks, random reports a value that you can use inside other blocks that take an input. Here are some examples:

    • You'll want to use a or to keep your sprite moving continuously.
    • Make sure the sprite can travel in any direction.
    • Putting the pen down will help you keep track of what the sprite is doing. You'll see a two-dimensional random walk.
  2. Experiment! How does your sprite's actions change if you have it move a fixed (rather than random) amount each time?
  3. Have another sprite chase that sprite but not quite catch it. It might point towards it and move more slowly or it might point almost towards it but veer away by some random amount.

The sprite will say "Hey! You!" once.
Incorrect. Understand what the say block does. In this case, "Hey!" will be said first, independently from "You!". "You!" will be said after "Hey!".
The sprite will say "Hey!" and then "You!" once.
Not necessarily. This is only true if the pick random block picks the number 1, which means that the repeat loop will only loop once. If this is so, then "Hey!" will be said for one second followed by "You!" said for one second.
The sprite will say "Hey!" a random number of times and "You!" a different random number of times.
Not quite, but you are close! Make sure you understand that a repeat loop will do the same number of repeats to each block within it.
The sprite will say "Hey!" and "You!" the same random number of times.
Correct! The pick random block will randomly pick a number between 1 and 5 (the given parameters) and then loop that number of times.
