This script draws a square.

The block certainly makes the script shorter and simpler to construct, but that is not what's most important about it.

The structure—the logic of the script—is clearer when we see repeat 4 (move 100, turn 90) than when we see move 100, turn 90, move 100, turn 90, move 100, turn 90, move 100, turn 90. Making the logic clear is essential for good coding for lots of reasons, including making it easier to catch bugs. In that long list, we'd have to count the parts to see if it's right and you can imagine how easy it would be to make a mistake if we were drawing a 20-sided figure instead of a 4-sided one.

It is also possible to nest one repeat block inside another. Since snap processes scripts from top to bottom, the first repeat will begin first, and the second repeat will have to complete its entire process before the process of the first repeat is called again.
