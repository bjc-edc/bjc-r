The Real Value of The repeat Block

It is, of course, much easier to create this script...

...than to create this one:


Even more important than ease, using repeat makes the structure of the program clearer.

That is essential to good coding. Clearly structured code is easier to understand and easier to debug (to catch and fix errors).

In that longer script (without repeat), we'd have to count the parts to see if it's right. It would be very easy to make a mistake if we were drawing a 20-sided figure instead of a 4-sided one!

    You don't need to say how long each side is or how many degrees to turn.
    You do need to show what repeats and how many times it repeats.
