Tic Tac Toe Phase 1 (Teacher's Choice)


Tic Tac Toe Phase 1: You will create the basic Human versus Human game where the computer simply takes care of displaying the X’s and O’s and the users themselves decide on a win or a draw. Playing with the final product will feel a lot like two players playing the game with paper and pencil. In this phase, you will create the TTT board by cloning a clickable square sprite with three costumes: Empty, X and O.

  1. Play with this example to get a feel for the Phase 1 version of TTT.
  2. Open this starter project to begin your Phase 1 TTT. A sprite, named Square, has already been created for you with three costumes: Empty, X and O.

You will be using the cloning feature in Snap! to create the 3X3 TTT board as it is a speedier alternative to creating and keeping track of 9 separate sprites: Square1, Square2, … Square9.

Study this example file to gain some experience with the basics of cloning in Snap!.
  1. Create a new sprite called Board and use it to build the TTT board with clones of Square using the following blocks.

  1. Experiment until you can position the Square’s clones correctly on the stage building the board row by row without overlap.

    2. After cloning 9 times to create the board, hide the original Square so it does not interfere with the game as a 10th square. Be sure to show the original Square initially, as it will be hidden by a previous run and thus its clones will not be visible.
  2. By default, each Square clone should have the Empty costume. Now create code that will allow each Square clone to put on the proper costume when clicked: X or O depending on whose turn it is. Also think, how should a square behave when it is clicked while already wearing an X or O costume.
    (Hint: If you keep track of the number of moves made so far with a variable named counter, you will be able to use the oddness/evenness of this count to determine whose turn it is. You can use the mod block to in the Operators menu to determine oddness/evenness.)
    mod function gives the remainder when the first input is divided by the second. For example, 6 mod 2=0 as 6 divided by 2 leaves a remainder of 0. And, 7 mod 2 =1 as 7 divided by 2 leaves a remainder of 1. Thus, you can use a number mod 2 to determine if it is odd or even.
