Creating Scripts

In the scripting area, you can snap blocks together (hence the name snap) to build up a set of instructions, a script. Drag one block right underneath another to snap them together. The white bar that appears (see the picture below) shows you where the block will go when you drop it.

A snap program consists of one or more scripts.

While a script is running its border is highlighted. This one runs so quickly that you may have to look closely to notice the brief border flash.
  1. Create this two-block script in the scripting area. Click on it a few times to see what it does.
  2. This script takes longer to run, so it's easier to notice the highlighted border:
    Here is another script to create. Predict what it will do, and then try it out.
  3. Modify your first two-block script so that when you click it enough times, it makes a square.
  4. Modify it again (or build another copy) so that it can make a triangle or hexagon.

Blocks in a script run in a specific order, from the top of the script to the bottom. Generally, snap waits until one block has finished its job before continuing on to the block below it. (One common exception is blocks that play sounds: a block's job can be to start the sound, which means the block below it will execute while the sound is still playing.)

Saving Your Work

In snap, your work is saved online ("to the cloud").

  1. If it's not open already, open snap by visiting http://snap.berkeley.edu/run .
  2. Check if you are logged in by clicking the cloud button in the toolbar: .
    • Log in if you need to.
    • Create an account if you have not done so already.
    3. Any time you want that work back, you will use the same button and select "Open..." to see a list of your files.
  3. To save your work, use the File button in the tool bar, and choose "Save" or "Save as...".
  4. Give your project a name that will help you remember what the project is.
