Recorrer una lista

En este laboratorio, utilizarás tu habilidad con la recursividad para construir algunas funciones avanzadas.

En esta página, investigarás acerca de un patrón de código común.

Decir que agregar "s" a una palabra siempre forma el plural es una simplificación excesiva. Si hiciste el proyecto opcional de Plurales y te apetece, puedes usar tu trabajo del bloque plural que hiciste en la Unidad 2.

Aquí tienes el bloque plurals del laboratorio 1. Toma una lista de palabras como entrada y reporta una lista de los plurales de esas palabras. Aunque hay una manera de construir este bloque usando mapear, ten paciencia con nosotros: hay una buena razón para investigar esta versión recursiva.

La idea principal es poder dividir una lista en su primer elemento y el resto de los elementos, formar el plural del primero y hacer una llamada recursiva para el resto. El bloque delante de puede parecer extraño al principio, con sus entradas asimétricas (un elemento, una lista), pero junta el plural de la primera palabra y los plurales del resto de las palabras.

Este patrón de código se puede usar para crear muchos reporteros recursivos útiles que tienen un caso base simple y una llamada recursiva simple.

    Haz un bloque raíz cuadrada que tome una lista de números como entrada y reporte una lista de las raíces cuadradas de los números.

  2. Crea un bloque exagerar que tome una oración como entrada y reporte una oración en la que cada número en la entrada se duplique, "bueno" se reemplace por "excelente", "malo" por "terrible", "me gusta" por "me encanta" y "no me gusta" por "lo odio".

    ¿Qué debe hacer la función auxiliar con las otras palabras?

    El bloque exagerar lista de palabras será recursivo.

    También necesitarás una función de ayuda exagerar palabra que funcione en una sola palabra, buscando para palabras y tipos específicos para exagerar.
