Plotting points

Plotting a single point, given its two coordinates, should feel this simple.

Then you could graph a whole list of data points by inventing a block like that uses PlotPoint for each item of the list.

If all the (x, y) pairs fit on the stage that seems like all you should need, but what if you want to plot the popularity of a baby name over time, like this?

The x-values can't be plotted because they are completely off the screen. And even if you adjusted them to be on the screen y-values 0, 0.1, 0.2, 0.3, etc., are so close to zero that they would all look the same. We would not see the huge increase in popularity between 1960 and 1980.

A good grapher needs to let the user set the scale of the screen, to specify where to focus attention, where to zoom in.

  1. Load this Snap! project. It contains several blocks already built for your convenience.
    1. Click the set window scale block with the inputs it already has (-2, 4, 30, 300) to see what it does. It sets several variables that your other blocks will need so that they know the intended screen dimensions. It also draws axes (or substitutes) on the screen and labels the place where they cross.
    3. Now click to reset the size of the screen.
  2. Experiment with various inputs to see where it puts the axes and how it represents the "substitutes" for the axes, when the axes can't be shown.
  3. Build . The two operations that convert data coordinates into stage coordinates, so that sets the sprite in roughly the middle of the stage.
  4. Then, build PlotPoint so that it uses the new go to scaled point instead of plain go to.
  5. inally build and try it out.
