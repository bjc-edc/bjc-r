Plotting points

Plotting a single point, given its two coordinates, should feel this simple.

Then you could graph a whole list of data points by inventing a block like that uses PlotPoint for each item of the list.

That seems like all you should need, as long as all the (x, y) pairs fit on the stage, but what if you want to plot the popularity of a baby name over time, like this?

The x-values can't be plotted because they are completely off of Snap!'s stage. Moreover, the y-values 0, 0.1, 0.2, 0.3, etc., are so close to zero that they would all look the same. We would not see the huge increase in popularity between 1960 and 1980.

A good grapher needs to let the user set the scale of the screen, to specify where to focus attention, where to zoom in.
  1. Load this Snap! project. It contains several blocks already built for your convenience.
    1. Click the set window scale block with the inputs it already has (-2, 4, 30, 300) to see what it does. In addition to drawing on the screen, it sets several variables that your other blocks will need so that they know the intended screen dimensions.
    2. Experiment with inputs to these two blocks to get them to report 0. How do your input numbers relate to the window scale?
    3. Use with those same input numbers. Where does it put the sprite?
