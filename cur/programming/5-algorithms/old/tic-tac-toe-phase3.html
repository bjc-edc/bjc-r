Tic-Tac-Toe with Intelligent Computer Player

In this last phase of the Tic-Tac-Toe project, you will learn how to make the computer play the game intelligently against a human player.
  1. Open your Tic-Tac-Toe project from Unit 3. The name suggested was U3L2-TicTacToe.
  2. Save it as U5-TicTacToe so the earlier work is saved separately.
  3. Create a new sprite and give it a name such as Computer Player.
  4. Computer Player will need access to each square clone and tell it to put on a particular costume, so you will need a way of storing the squares in a list as each clone is created. Create a variable squares list and initialize it to an empty list. Using When I start as a clone control block, add each square clone to this list as soon as it is created.
    5. For simplicity, you can assume that the human player will always start first. In the "Take It Further A" below you will be asked to design your game so that human player is given the choice to start first or not.
  5. From your earlier work, the squares already know how to behave when they are clicked by a human player. You will need to adjust the code to make sure that the squares respond to a click only when it is the human player's turn. You will also need to broadcast when human player is done with a move and announce that it is Computer Playerâ€™s turn.
  6. Implement an algorithm such as this for the Computer Player:
       Check each empty square for a winning move
   If possible, make the winning move
   Else, check each empty square to see if can block a win by the opponent
   If possible, make the blocking move
   Else, make a random(*) move to an available square
   Check for a tie
   If there is no tie, pass the turn to human player
    7. (*) In the "Take It Further B" below you can explore more intelligent alternative strategies to the random move.
    • Computer Player will use the (virtual) board to attempt a move and test for a win. If a win is possible, it will make the move on the actual board by telling the appropriate square to put on the relevant costume.
    • If no win is possible with a move, it will undo the move on the (virtual) board. A similar approach will be needed to see if blocking a win by the opponent is possible.
  7. Create a fully functional Tic-Tac-Toe program endowed with an intelligent Computer Player. Test and debug your code thoroughly. Make sure that the Computer Player responds appropriately when you intentionally lose, tie and win the game.
  1. Design your game so that the human player is given the choice whether to start first or not.
  2. Do some research on Tic-Tac-Toe and implement a more intelligent strategy than making a random move in the algorithm given above. Why moving to the center is most desirable? Why getting a corner is second best?
  3. Learn about the game Ultimate Tic-Tac-Toe and implement it in Snap!.
  4. Read about artificial intelligence programs such as Deep Blue and AlphaGo.
