On Unit 2 Lab 3 Page 3, Solving a Word Puzzle, you built a block and used it with keep to find words of a specific length in a word list. But how long does a process like that take?

  1. Run the initialize block to load a list of 10,000 words.
  2. How many five-letter words are in the 10,000 words list?
    3. Need to add a URL to the "import" link. --MF, 4/3/19
    You can import your block from your U2L3-WordPuzzleSolver project.
  3. Use the block provided in this project to determine how long it takes for the computer to answer that question.

    4. The computation time block takes any reporter (with its inputs filled in), computes the result but ignores it, and instead reports how long it took to do the computation (in milliseconds).

    In this example, it took 27 milliseconds to compute the list of integers from 1 to 1000. (The report you see will depend on how fast your computer is and what other programs are running on it.)

    You can look inside computation time to see how it works.

  4. Click on computation time of (10,000 words) three times and note the range of answers. They're not exactly the same because of other things that your comptuer is doing at the same time, so you should always take whatever result it gives you as approximate.
  5. How long do you think it would take to answer that question if the list had 100,000 words?
  6. Use the computation time block to determine how long it takes to find out how many five-letter words are in the 100,000 words list.
  7. Suppose instead, we count the seven letter words. This will take:
    1. More time because seven letters is more than five letters.
    2. Less time because there are fewer seven letter words than five letter words.
    3. The same time because it's not the word length that matters; it's the size of the dictionary.
  8. Experiment to find out for sure.

The only way to answer this problem is to check every single word in the dictionary. So if you have ten times as many words in the dictionary, it takes ten times as long to check them all.

: Linear Search or Sequential Search
  • An algorithm takes linear time if multiplying the input size by ten multiplies the time required by ten.
    • AAP-2.O.5
  • A linear search (or sequential search) algorithm checks each element of a list in order , a process which takes linear time.
