Adding a Contact to the List

On this page, you will develop:

  • An "Add Contact" button for adding each new contact using the contact constructor
  • A "Clear List" button for emptying the contact list

  1. Develop an "Add Contact" button for the user.
    1. Give the sprite an appropriate costume; several are included in the project.
    2. Make the sprite collect information for a new contact from the user and then add it to the contact list using contact.
    EK 4.1.1H Different algorithms can be developed to solve the same problem.

    Collecting Data from the User

    You'll want to ask the user for each piece of data (contact name, address, and phone number) separately.

    • You could use several ask and answer blocks together with script variables to request and then store each piece of user input until you are ready to report them all together using contact.
    • Another way is to use abstraction: create helper blocks (such as and ) that each ask the user for one piece of data and report their answer to the contact constructor.

    Adding Items to a List

    • One way to add an item to a list is with add.
    • Another clever way is to use set together with .
  2. Add a few made-up people to your contact list to make sure everything works.
  3. Write a script for the "Clear List" button to reset contact list to an empty list.
    1. Give the sprite an appropriate costume.
    2. Make the script ask the user for confirmation (so no one accidentally deletes their lists).
      3. For example, clicking, "Clear List" might ask the user, "Enter c to clear the list, or enter anything else to cancel."
  4. The sprites' questions might be covered up by the contact list watcher on the stage. Use hide variable and show variable to hide the contact list watcher until the end of each sprite's script.
