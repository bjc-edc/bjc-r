Data Types

  • Paul or Brian, please review the following preface on data types. --MF
  • BH: Variables do not have types. Data have types. Any variable can hold any datum, regardless of type. Also, Snap! is not Scratch -- lists are not comparable to variables; they're comparable to numbers or procedures.


A data type is, just as the name suggests, what kind of data something is. "Noun phrase" is a data type; so is "phrase" (which also includes verb phrases and prepositional phrases). "Person" could be a data type in a Contacts app.
 

The abstract data types in the last paragraph exist only in the mind of the programmer. Other primitive data types are built into each programming language. The primitive types in Snap! are number, text string, Boolean, list, command, reporter, and predicate. (We haven't used the last three as data types yet.) You can get a reminder of Snap!'s primitive types from the pulldown menu in the is () a ()? block.
