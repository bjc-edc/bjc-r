Public Key Encryption

The weak part of the methods so far is that the key needs to be kept secret. But keeping the key private is almost as hard as keeping the message private. In 1976 Ron Rivest, Adi Shamir, and Len Adleman devised an ingenious method (now known as RSA) that allows a person to announce publicly how anyone can encode a message to him so that only he is able to decipher it. The information, which is given away to everybody, is called public key. The information which is needed for deciphering is known only by the person for whom the message is intended. This information, therefore, is called the private key. If I want to send Tanya a secret message, I encipher it with her public key. Then no one else can read it, even if he or she has her public key. Only her private key can undo the encryption, and you can't figure out the private key from the public key.
