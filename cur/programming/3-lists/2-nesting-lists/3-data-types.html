A data type is, just as the name suggests, what kind of data something is.

Each programming language provides some "primitive" data types. The primitive types in Snap! are number, text string, Boolean, list, command, reporter, and predicate (but we haven't used the last three as data types yet.) You can get a reminder of Snap!'s primitive types from the pulldown menu in the is () a ()? block.

Abstract data types (or ADTs), like "noun phrase" exist only in the mind of the programmer. For example:

  • Your reporter noun phrase (in Lab 1) was a data type that joined 3 text strings from various lists.
  • Also prepositional phrase (in Lab 1) was a data type that joined a text string and a noun phrase.
  • You could create a person data type for your Contacts List app.
  • You might want a student data type if you were collecting data on your classmates for a yearbook.
