Data Types

This picture needs updating for Snap! 4.1, which has a "sprite" data type.

A data type is, just as the name suggests, what kind of data something is.

Each programming language provides some primitive (built in) data types. The primitive types in Snap! are number, text string, Boolean, list, command, reporter, and predicate (but we haven't used the last three as data types yet). You can get a reminder of Snap!'s primitive types from the pull-down menu of the is () a ()? block (shown right).

Abstract data types (or ADTs) such as "noun phrase" are custom data types that exist only in the mind of the programmer. For example:

  • Your reporter noun phrase (in Lab 1) constructs a data type that joins three text strings from various lists.
  • Similarly, prepositional phrase constructs a data type made out of a preposition and a noun phrase.
  • If you were collecting data on your classmates for a yearbook, you might want a student constructor.
  • For your Contacts List app, you will create a contact data type.
  • On this page, you will use a point constructor to simplify your Draw Shape project.
