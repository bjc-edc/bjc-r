    Try to describe the algorithm in enough detail so that someone else could use your description to sort a list.
  1. Design (don't code) a recursive algorithm for sorting the names.
    • First consider your general strategy; how would you sort a group of names, step-by-step.
    • Then consider how you would tell a computer, which is not as smart as you and can only perform specific procedures, how to sort a list.
    • It may help to keep in mind that recursion works by solving a smaller, similar subproblem and using that result to help solve the original problem.
    If you get stuck, discuss the problem with another group.
  2. Now build your algorithm, as a sort reporter that takes a list as input and outputs a new list of the same elements in alphabetical or numeric order. Test your algorithm using both names and numbers.
    3. Comparing numbers and strings can be done using the same blocks, so your sorting algorithm should work equally well for names and numbers.
  3. Share your sorting algorithms. Describe some similarities and differences of different sorting algorithms.

There are many different algorithms to sort a list. Sorting algorithms generally give the same result, but the code and efficiency can vary greatly from one algorithm to another. The next page explores one simple sorting algorithm.
