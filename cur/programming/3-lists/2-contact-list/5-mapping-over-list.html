The block takes two inputs: a function (a reporter with a blank input slot) and a list, and it reports a new list where every item is the result of calling the function with an item from the original list as input. For example:

You choose the function that describes the change, and map applies that function to each item in the input list and then reports the list of changed values.

    Here is a quick review of higher order functions.

    • Map performs a function on every item of a list and reports the list of changed items.
    • Keep uses a predicate function (a true/false question) to check every item in a list and reports the items that make the predicate true. (You learned about keep on Unit 2 Lab 3 Page 5: Keeping Items from a List.)

    • Combine uses a combining function (a function with two inputs) to report the results of combining all the items in a list using that function. (You learned about combine on Unit 2 Lab 4 Page 3: More Mathematical Reporters.)

The orange oval blocks that name variables are reporters, like other oval blocks. But most reporters implement functions, which means that when you call the same block with the same inputs, you always get the same value. 2+3 is always 5. The random block, and reporters you write using random, are different; calling them twice with the same inputs may report different answers.

Variable blocks are one step further away from functions; a program can choose to change the value that a variable block will report by using set or change to give the variable a new value.



It turns out that any program that changes the values of variables can be rewritten in a way that doesn't set or otherwise change the values of variables. There are advantages to this functional programming style, which avoids certain kinds of otherwise-common bugs, but some kinds of programs are hard to write functionally. You aren't really ready to think about this yet, but we'll come back to functional programming later in the year.
