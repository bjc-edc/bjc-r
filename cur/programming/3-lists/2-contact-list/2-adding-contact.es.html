      Usamos "entrada" vagamente para referirnos a las casillas vacías en un bloque que se llenan con valores. Pero entrada también significa información ingresada en un programa por el usuario, como en el bloque preguntar y esperar. La entrada del programa también puede provenir de tablas de datos, sonidos, imágenes, videos u otros programas.

      Del mismo modo, salida del programa significa cualquier dato enviado desde su programa al usuario oa cualquier dispositivo. Por lo general, la salida depende de la entrada..

La mayoría de los programas que ha escrito tienen un script que está a cargo, utilizando bloques de bucle para seguir ejecutándose para siempre o solo durante mucho tiempo. Este proyecto es diferente: no sucede nada hasta que haces clic en un botón (recuerda de la Unidad 1 que esto se llama un evento), que inicia un script que solo se ejecuta brevemente para realizar cualquier acción que requiera el botón. Este tipo de estructura de programa se llama programación impulsada por eventos.

En esta actividad, refinará el manejo de los nombres de las personas. Cuando busca a una persona en particular, quiere ver su nombre como lo ha estado viendo en esta página, primero el nombre: "Alphie Preston". Pero suponga que desea ordenar sus contactos. Por lo general, eso lo hace el apellido, por lo que la lista sería algo así como

  • Abdel, Gamal
  • Anderson, Betsy
  • Preston, Alphie
Por lo tanto, debe poder mostrar un nombre primero para la visualización normal, pero primero el apellido para la clasificación.

Antes de comenzar, guarde su proyecto nuevamente, con el nombre U3L2-TIF. Esto mantendrá su trabajo sobre estos problemas separado de los problemas regulares.

  1. El primer paso es crear un resumen de datos escriba para los nombres. Necesitará un constructor que tome el nombre y el apellido como entradas e informe una lista de ellos para representar la parte del nombre de un contacto..
  2. Leer un nombre: El siguiente paso es modificar el programa que escribiste arriba para que pida por separado el nombre y el apellido, construya un nombre (es decir, una lista) usando el constructor que acabas de escribir, y usa esa lista como entrada de nombre para el bloque contacto con nombre. Este es un tipo de datos abstractos (nombre) dentro de otro tipo de datos abstractos (contacto).
  3. Escribiendo un nombre: Tu bloque name from contact ahora reportará una lista, como debería. Pero algunas otras partes del programa, como la parte que muestra un contacto al usuario, quieren mostrar el nombre como una cadena de texto: "Betsy Anderson". La parte que ordena la lista de contactos también quiere una cadena: "Anderson, Betsy". Escribe dos bloques y que unen las dos partes del nombre con un espacio y/o una coma según sea necesario.
Este proceso de tres pasos (crear un ADT, usarlo para leer valores de ese tipo, usarlo para escribir valores de ese tipo) es muy común en la programación. Volverá a seguir los mismos tres pasos en la sección Llévelo más lejos en la página 4.
