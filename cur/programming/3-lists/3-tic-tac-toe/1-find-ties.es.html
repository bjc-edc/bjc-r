Recordando los Movimientos y Posibles Ganancias

En este laboratorio, continuará con su Proyecto Tic-Tac-Toe para hacer que el programa analice el tablero de juego para detectar triunfos y empates.

En esta página, te prepararás permitiendo que el programa registre los movimientos que se realizan y enséñale los grupos ganadores de tres casillas.

Cuando escribiste Tic Tac Toe por primera vez, era fácil detectar movimientos ilegales (moverse a un cuadrado que ya lleva un disfraz de X u O) porque cada sprite podía probar eso sin conocer el resto del tablero de juego. Pero para detectar victorias y empates, el programa necesita una forma de probar el estado del tablero de juego completo, no solo de una pieza individual. Entonces, usará una variable global, , para haga un seguimiento de si el cuadrado en cada posición está vacío o lleno con una X o una O.

Aprendió sobre variables globales en Unidad 2 Práctica de laboratorio 1, página 4: Llevar la puntuación con variables globales.

Cada sprite necesita conocer su propia posición en el tablero para que pueda usar el bloque objeto de para reemplazar esa posición con una X o una O. Entonces, creará una variable de sprite , , y cada clon tendrá su propia versión de esta variable.

Una variable sprite es como una variable global en el sentido de que no pertenece a un script en particular, pero sí pertenece a un sprite en particular.
