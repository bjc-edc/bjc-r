Tic-Tac-Toe: Detect Tie Games

In this lab, you will continue the tic-tac-toe project from Unit 2 to allow the program to analyze the game board to detect wins and ties.
On this page, you'll start with ties, because they're easier to find.
In Unit 2, you may have done either or both of two versions of the tic-tac-toe project: the one-sprite version in Lab 4 and the nine-sprite version in the optional projects.
  • In the one-sprite version, it took some calculation to find out which square the player moved into, but very little extra effort to check for illegal moves.
  • In the nine-sprite version, knowing which square was automatic, because the player moved by clicking a different sprite for each square. But collecting the moves in one place to detect illegal moves required messages between sprites.
There's no obvious better choice between the two.

Both versions end up with a global variable board whose value is a nine-item list representing the current state of the game board. As a result, you can now build on either starting point in this lab.

  1. Open your project U2L4-TicTacToe (Lab 4 version) or U2Opt-TicTacToe (optional project version) and save it as U3L3-TicTacToe. That preserves the old version while you modify it here.
  2. The definition of a tie game is that there are no more empty squares, and neither player has won. You're going to detect wins on the next page, so for now, just declare a tie (switch the sprite's costume to Turtle and have it say "TIE GAME!") when there are no empty squares after a move. (Don't click unless you really, really need a super big hint.)
  3. Save your project.
