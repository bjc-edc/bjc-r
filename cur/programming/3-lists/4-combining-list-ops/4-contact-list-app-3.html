Contact List App: Name Order

Now you'll fix the name order issue: whether say the family name first and then the given names, or the given names first and then the family name. To do this, you will create two different types of contact: family-first and given-first. Each contact will be represented as a list whose first item is a type tag, either the word "family-first" or the word "given-first."

  1. Load your contact list app from before; it should be saved as "U3L2-ContactList"
  2. Write two new constructors:

    Both should report a four-item list containing the type tag, the name, the address, and the phone number.

  3. Decide how to present this choice in the user interface. You could have two buttons "Add family-first contact" and "Add given-first contact." Or you could have a single button that asks the user to enter the type (maybe with examples), and then the name, address, and phone. Or.... Build the button(s) for the user interface.
  4. Then create a new selector called contact type that takes a contact as input, and reports its type tag.
  5. Now fix the address and phone number selectors to use the new contact structure.
  6. Finally, fix the two name selectors to check the type tag and report the name properly:
    Note: There's no comma in the sorting form of a family-first name; it's exactly the same as the display form.
    Name forms vs. name types
      display form sorting form
    family-first type   Sun Yat-Sen   Sun Yat-Sen
    given-first type   Abraham Lincoln   Lincoln, Abraham
  7. Clear your contact list, add some contacts of both types.
  8. Make sure the internal representation is what you expect, and then make sure that your and blocks still work, without changing them at all.
