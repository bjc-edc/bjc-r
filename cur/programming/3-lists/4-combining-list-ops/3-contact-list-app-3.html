Contact List App: Name Order

Now you'll revise your contact list app to fix the name order issue: whether to say the family name first and then the given name(s) or the given names first and then the family name.

To do this, you will create two different versions of the contact constructor: family-first and given-first. Each contact will be represented as a list whose first item is a type tag, either the word "family-first" or the word "given-first."

  1. Load your contact list app from before; it should be saved as U3L2-ContactList. Save it with the new name U3L4-ContactList.
  2. Write two new constructors:

    Both should report a four-item list containing the type tag, the name, the address, and the phone number.

  3. Decide how to present this choice in the user interface, and implement your decision.
    • You could have two buttons "Add family-first contact" and "Add given-first contact."
    • Or you could have a single button that asks the user to enter the type (maybe with examples) and then the name, address, and phone.
  4. Clear your contact list and add a few entries in the new format.
  5. So that your program can determine which type a contact is, create a new selector called that takes a contact as input and reports its type tag.
  6. Now, fix the address and phone number selectors to use the new contact structure.
  7. Finally, fix the two name selectors (display-form and sorting-form) to check the type tag and report the name properly:
    Note: There's no comma in the sorting form of a family-first name; it's exactly the same as the display form. (Recall, in "Sun Yat-Sen" the family name is "Sun.")
