Hints for Binary Conversion

The biggest hint is that converting to binary should work from right to left, because the rightmost bit is always the ones column, while the leftmost bit's value changes depending on how many digits it has.

The following exercises are hints; use as many or few as you need. Remember, the input to your block is a number, so you should do arithmetic on it; the value reported by your block is a text string, so you should make it using string operators.

    Try doing these calculations by hand, and look for connections. The connections may help you understand how to write your code.
  1. Explain why 13 is 1101 in binary.
  2. Write 26 in binary. Write 27 in binary. How are each related to writing 13 in binary?
  3. What is the quotient and remainder when you divide 27 by 2?
    4. BH suggests to delete hints 4 and 5.
  4. Start with 26, and divide by 2: the quotient is 13 and the remainder is 0. Continue with 13, repeating the process. Use this process to build the binary representation of 26.
  5. Repeat the previous problem, starting from 107. What is the binary representation of 107?
Remember, you'll want to use the and blocks. For binary, the quotient and remainder when dividing by 2 are the key.
