Here is a list of other formatting we can use:
  • Use the "code" tag for code. Reference blocks (e.g. pen down) using the "code" tag or an image of the actual block: .)
    • Note: The code tag has been used all over the place in the previous version of BJC. Please try to keep it to only actual code blocks.
    • Re: variables and mathematics—If you are referencing a mathematical variable that you have made into a snap variable (hence, a block) then use the code tag (or an image of the block). If you are referencing the plus/minus/etc. block, use the code tag (or an image of the block). If you are just discussing a mathematical idea, then use regular notation: e.g. x + y = 7.
  • Use the "strong" tag for bold. Introduce new terms using the strong tag.
  • Use the "em" tag for italics. The "em" tag is for emphasis.
  • References to text in the app (such as the "OK" button or the "Motion" palette) should be in quotes.
