Finishing Your First Snap! App

More than one script can control a sprite. You currently have just one script, which runs when Alonzo is clicked. Now you'll make a second script that runs when is clicked. The new script will get Alonzo to welcome the player to your game.

Blocks that look like this can be used to detect an event—clicking on the sprite, pressing a key, clicking the —and tell a script to start when that event occurs. Because these blocks can sit only on top of a script (as a way to start the script), Snap! programmers call them "hat blocks."

You can stop all running scripts by clicking the red stop sign at the top right of your screen.

    You can do this using just 3 blocks including:
  1. Open your ClickAlonzo project in the window below.
  2. Create a second script that makes Alonzo move to (190, -130) and say a welcome message whenever the green flag is clicked. (Alonzo might say, "Welcome to 'Click On Alonzo!' Keep clicking me if you can!") .
  3. Make that message disappear when the player clicks on Alonzo.
    The block with an empty input will make the sprite say nothing.
  4. Check that your app does everything you want:
    • When the green flag is clicked, Alonzo should move to (190, -130) and say its message.
    • When Alonzo is clicked, the message should disappear, and Alonzo should move to a random spot on the stage and face the other way.
 
  1. So far, Alonzo waits in one place to be clicked. Increase the challenge by adding code so that after the first click and from them on, Alonzo moves to a random position on the stage every two seconds (even if not clicked).
    Translating ideas into code:
    • "Every 2 seconds" means wait 2 seconds and repeat.
    • In Snap! programming, "from then on" uses the forever block.
    • What does "after the first time Alonzo is clicked" tell you about the structure of the code?
  2. Give the player some sense of how well they are doing:
    1. Put in the script that runs when the player clicks on Alonzo. This makes Alonzo fade a bit each time he's clicked.
    2. Put another block in your program to change the ghost effect by -5 any time Alonzo moves without being clicked.
