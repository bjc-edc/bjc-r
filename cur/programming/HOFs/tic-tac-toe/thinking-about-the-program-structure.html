Remember that this block only represent the strategy in which the computer chooses the next move; we are not programming any visual parts at the moment.

Before we approach to make the strategy in BYOB, think about how do you play tic-tac-toe. We have many strategy rules that has priority over another. We will represent these rules with "if" statements:

  • If there is a winning move, we take that move.
  • If the opponent has a winning move, we move to the square that blocks it.
  • If none of the above applies, then choose the best free square.

And if you're daring, you may also do this:
  • If we can make a move that allows us to win on the next turn, we take that move.
  • If there is a move that the opponent can make that will make them win in two moves, then block it.
The last two allows the computer to play a perfect game of tic-tac-toe.

You can see that we are going to make many helper blocks to complete the strategy. We need to check each condition sequentially and make sure we get what we need to get.
