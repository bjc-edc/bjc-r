In this project you will create a timer that reports the number of seconds elapsed in both decimal and binary notations.

  1. Study the Binary Counter linked here to gain some instinct how the binary system relates to the decimal system.
  2. Review the examples in the Appendix, if needed, and discuss as a class how you can convert a number in base 10 to its equivalent in base 2. Come up with a simple algorithm for this procedure.
  3. Examine the student file linked here with which you will be working. Note that 3 sprites representing the decimal digits, 8 sprites representing the binary digits, with all the necessary code and costumes have been created for you. To see the actual costumes, you will need to go to the Costumes area for each sprite. Another sprite representing the “Timer” has also been created for which you will need to fill in some code.

In this project you will generate the binary number representation of decimal numbers 0 through 255. While it takes 3 digits to represent 255 in base 10, it will require 8 digits in base 2. (Why?)

If you look at the Timer sprite code, you will see that Snap!’s own “timer” block (from the “Sensing” menu) is used as a time keeper. The decimal and binary digits are computed and stored in lists and then a broadcast is made to the sprites representing the digits to put on the appropriate costumes (0 and 1 for the binary digits and 0 through 9 for the decimal digits)

The “warp” block is used to ensure that all the computation within the block is finalized prior to the refreshing of the screen .
