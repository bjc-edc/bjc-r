Different people might draw this diagram slightly differently, so don't try to memorize all the levels. The key abstraction levels to remember are software, digital computer hardware, and underlying analog circuit components.

Software Domain

At the highest level are application programs—the computer software. For the most part, neither the programmer nor the user thinks at all about how the hardware inside a smartphone or computer (the wires, glass, and plastic) actually fit together or work. Using software is an abstraction—a way of thinking about the computer without thinking about how it works.

Similarly, a driver is thinking at a high level of abstraction when using the brake pedal. As long as the engineers and mechanics designed and built the car right, drivers don't need to know all the details about how the brake system works—they just use it at a high level.

Digital Domain

Below the software is the hardware (the guts of the computer). The hardware's job is to run whatever programs the user requests. For hardware engineers, the abstraction focuses on how to make best use of components (memory, chips, logic gates, etc.) that already have some useful built in behaviors, and often also assembling these into slightly fancier components. Engineers mostly don't have to think about how these digital components work inside—they just think of circuits being either on or off (1 or 0), with no in-between values.

Analog Domain

At the lowest level, "noise" in the electronic circuits means that there are in-between values (like "on a little bit"). Electrical engineers must think about the laws of electronics (physics) to design circuits so that these in-between values stay close enough to 0 or 1 to be treated as if they really are 0 or 1.
