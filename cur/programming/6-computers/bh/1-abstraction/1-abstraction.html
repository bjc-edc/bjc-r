Software Domain

The programs you write and use are software, but underneath these programs is more software that helps them work. In this lab, we'll look at four layers of abstraction in the software domain: applications that you write and use, programing languages that are used to write them, libraries of useful functions that hide messy details, and operating systems that interface with the hardware.

Software is an abstraction—a way of thinking about the computer without thinking about how it works.

Similarly, a driver is thinking at a high level of abstraction when using the brake pedal. As long as the engineers and mechanics designed and built the car right, drivers don't need to know all the details about how the brake system works—they just use it at a high level.

The circuitry inside a computer is complicated, and to make building it possible, engineers think about it at several different levels of abstraction: architecture (the instructions that software can send that the hardware will understand), components (such as memory and ) that implements the architecture, those components are built out of integrated circuits (chips), which are the back rectangles you can see on this circuit board, those ICs are designed around fundamental building blocks that implement Boolean functions (logic gates).

Engineers mostly don't have to think about how these digital components work inside—they just think of circuits being either on or off (1 or 0), with no in-between values.

Below the software is the hardware (the guts of the computer). The hardware's job is to run whatever programs the user requests. For hardware engineers, the abstraction focuses on how to make best use of components (memory, chips, logic gates, etc.) that already have some useful built in behaviors, and often also assembling these into slightly fancier components. Most of the enginneering work is based on the assumption that any the biggest jump in abstraction is between the ones and zeros that everyone talks about and the way ... most of the work is dome in the digital domain, which is the levels of abstraction in which we pretend that only ones and zeros are possible.

In the different levels of abstraction in software, each level of abstraciton is a different piece of software, but the level of abstraction in the digital domain are four ways of thinking about the same physical object.

Analog Domain

Logic gates, which are the lowest abstraction level of the digital domain, operate on ones and zeros. In physical reality, those logic gates are built out of transitors, a type of circuit component. Transitors aren't like light switches that are either on or off. There can be in-between values (like "on a little bit"). So, electrical engineers have to think about the laws of electronics to design digital circuits so that in-between values don't happen.
