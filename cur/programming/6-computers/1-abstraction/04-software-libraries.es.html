Dominio de software: Librerías

En esta página, aprenderás que una librería es un paquete de procedimientos escritos por otros programadores que ayuda a resolver un problema.

Supongamos que un programador quiere que una imagen se mueva en una pantalla que también tiene otras imágenes En Snap!, que está incorporada: importa una imagen como un sprite disfraz, y luego muévelo moviendo el sprite. La abstracción del sprite permite a un programador mover una imagen sobre otras imágenes sin pensar en lo que sucede donde las imágenes se superponen o en cómo recuperar partes de la imagen después de que se cubren. No todos los lenguajes tienen la abstracción de los sprites.

Otras tareas comunes (como saber dónde está el puntero del ratón, hacer aritmética con números enormes o controlar el sonido) también tienen partes complejas. Si los programadores tuvieran que tratar con todos estos detalles en cada programa, ningún software se desarrollaría. Y como muchas aplicaciones necesitan las mismas habilidades, los programadores escriben librerías para que otros programadores las utilicen.

Vocabulario: Software de librerías
AAP-3.D.1, AAP-3.D.2, AAP-3.D.3
  • El software de librerías es un paquete de procedimientos que puedes importar a tu programa. Una librería es una especie de abstracción: no tienes que saber ninguno de los detalles de cómo está codificada. Puedes construir bibliotecas por ti mismo, o puedes usar una que alguien más haya escrito.

Lenguajes vs. Librerías

A menudo cuando la gente compara lo que los diferentes lenguajes de programación pueden hacer, están realmente comparando librerías. Por ejemplo, las personas pueden pensar que les gusta JavaScript porque pueden utilizar este para programar páginas web, pero esa no es una propiedad de JavaScript. Realmente este es una librería de páginas web incorporado en el navegador que te permite programar páginas web.
