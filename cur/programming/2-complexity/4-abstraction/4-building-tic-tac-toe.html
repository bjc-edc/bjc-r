Building a Tic-Tac-Toe Board

On this page, you'll build an interactive display for a simple Tic-Tac-Toe game between two human players.

In later units, you'll use this as the starting point for an ongoing project working toward playing against the computer.

  2. A Tic-Tac-Toe board is three rows of three squares each. If you use squares of size 100, the entire board will not quite fill up the height of the stage. You're going to start by drawing a single square, then use that to draw a row of squares, and then use that to draw the entire board. Be sure to test each block by itself before building a block that uses it.
    1. Build to make a single square tile. You can call any square or polygon block you've created earlier, or use primitive blocks directly.
      Why abstraction? Why build draw a tile when you could use draw square or just the script inside it? Suppose you use squares for several purposes in a large project; maybe you have a Sudoku (9x9 board) player in addition to Tic-Tac-Toe. If you later decide to create the tiles differently (maybe with thicker borders, filled with random colors, or by using stamp), it makes sense to edit draw tile (changing draw square would change all the squares).
