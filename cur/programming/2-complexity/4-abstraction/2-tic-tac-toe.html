Tic Tac Toe Phase 1


Tic Tac Toe Phase 1: You will create the basic Human versus Human game where the computer simply takes care of displaying the X’s and O’s and the users themselves decide on a win or a draw. Playing with the final product will feel a lot like two players playing the game with paper and pencil. In this phase, you will create the Tic Tac Toe board by cloning a clickable square sprite with three costumes: Empty, X and O.

To get an overview of the three phases of Tic Tac Toe, click here.

  1. Play with this example to get a feel for the Phase 1 version of Tic Tac Toe.
  2. Open this starter project to begin your Phase 1 Tic Tac Toe. A sprite, named Square, has already been created for you with three costumes: Empty, X and O.
  3. You will be using the cloning feature in Snap! to create the 3x3 Tic Tac Toe board as it is a speedier alternative to creating and keeping track of 9 separate sprites: Square1, Square2, … Square9. Study this example file and do the task given there to gain some experience with the basics of cloning in Snap!.
  4. Create a new sprite called Board and use it to build the Tic Tac Toe board with clones of Square using the following blocks.

    The second of these blocks does not appear by default in your Control menu but you can import it via the File menu’s "Import tools" option.

    Experiment until you can position the Square’s clones correctly on the stage building the board row by row without overlap.

  5. After cloning nine times to create the board, hide the original Square so it does not interfere with the game as a tenth square. Be sure to show the original Square initially, as it will be hidden by a previous run and thus its clones would not be visible.
  6. By default, each Square clone should have the Empty costume. Now create code that will allow each Square clone to put on the proper costume when clicked: X or O depending on whose turn it is. Also think, how should a square behave when it is clicked while already wearing an X or O costume.
    (Hint: If you keep track of the number of moves made so far with a variable named counter, you will be able to use the oddness/evenness of this count to determine whose turn it is. You can use the mod block to in the Operators menu to determine oddness/evenness.)
    The mod function gives the remainder when the first input is divided by the second. For example, reports 0, as 6 divided by 2 leaves a remainder of 0. Also, reports 1, as 7 divided by 2 leaves a remainder of 1. Thus, you can use a number mod 2 to determine if the number is odd or even.

You are now ready to pursue Phase 2 and Phase 3 of Tic Tac Toe.
  1. Phase 2: Human versus Human game where the computer decides on a win or a draw. In this version, the computer starts gaining awareness of the game and begins to meaningfully contribute to the game. Tic Tac Toe Phase 2 example.
  2. Phase 3: Human versus Computer game where the computer not only decides on a win or a draw but it also has some AI (Artificial Intelligence) features that allow it to “think” about a proper next move and simulate the moves of an intelligent human opponent. Tic Tac Toe Phase 3 example.
