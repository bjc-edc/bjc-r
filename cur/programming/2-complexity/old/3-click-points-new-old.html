You will combine parts of two projects: your work from the U2L2-DrawShape project you saved on the previous page, and a starter project for this page.

Export U2L2-DrawShape blocks
  1. Load your U2L2-DrawShape project.
  2. Choose "Export blocks" from the File menu to save the blocks on your computer. (Save all the blocks.)

Set the stage background
  1. Load this starter project. Below the actual stage are a "Stage" button and a "Sprite" button. Click the stage button.
  2. Now click the "Backgrounds" tab near the top of the window.

    You will find two non-empty backgrounds:
    • A graph of median U.S. household income by year (source: Federal Reserve Bank of St. Louis)
    • A capital "E" (font: Century Gothic Bold).
    The graph is typical of data processing, which is one topic of Unit 3. The E continues the activity from the previous page. Choose either background by clicking on it. (Later, you can also drag some other picture of a graph or letter into the project with the stage selected to get a different background.)
  3. Now go back to the Scripts tab, and choose the "Sprite" button under the stage to get back to the sprite's scripts.

Import your blocks
  1. Now choose "Import" from the File menu, and import the file U2L2-DrawShape.xml to add your blocks to the starter project.

Alphie and Betsy develop code to implement their ideas.
  1. The project file you loaded contains three unfinished scripts, one for each of Betsy's steps.
    • In the first script, variable point list should be given an appropriate initial value.
    • In the second script, you need a way to tell the sprite to stop following the mouse. Replace the forever block with one that will loop only until the user presses the space key.
    • In the third script, the stamp block is already in place, but you have to write the code that will add the mouse's position to the list of points. Use the data abstraction for points.

  1. This is a good quick-and-dirty solution to the problem of entering a bunch of points into a list interactively. But it has a couple of weaknesses:
    • Even after you tell the sprite to stop following the mouse, the "when I am clicked" script is still ready to run, and so if you click directly on the sprite, it will still add points to the list that you may not want.
    • The list of points is always kept in the same global variable. So it's messy to create lists for several different letters.
