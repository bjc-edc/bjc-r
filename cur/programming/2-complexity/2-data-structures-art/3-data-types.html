Abstract Data Types

On this page you will create a new data type, point, to make your U2L2-DrawShape code even more flexible and easier to read, debug, and use.
: binary sequence

Just as the name suggests, a data type is what kind of data something is. Every kind of information in the computer—whether it is a number, text, list, or an instruction to perform—is stored as a sequence of 0s and 1s (a binary sequence). So, for a program to work, the computer needs to be told how to interpret any such sequence of 0s and 1s.

Each programming language provides some primitive (meaning "built in") data types. In Snap! the primitive types are number, text string, list, and some other types you haven't yet used as data. (The is () a ()? block's pull-down menu shows all of Snap!'s primitive types.)

The word "abstract" often sounds as if it means something harder to understand, but in Computer Science, its meaning is almost the opposite. Abstract data types (or ADTs) are things that you, the programmer, create to make your program easier for people to read, debug, improve, and use.
