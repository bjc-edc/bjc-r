Data Types

On this page you will create a new data type, point, to make your code even more flexible and easier to read, debug, and use.

Just as the name suggests, a data type is what kind of data something is. Every kind of information in the computer is stored as a sequence of 0s and 1s (binary sequence). For a program to work, the computer needs to know whether to interpret any such sequence as a number, as text, as a list, or as something else....

Each programming language provides some primitive (built in) data types. In Snap! the primitive types are number, text string, Boolean, list, sprite, command, reporter, and predicate. (You haven't used all of these as data yet). This is the information the computer needs in order to interpret sequences of 0s and 1s. You can get a reminder of Snap!'s primitive types from the pull-down menu of the is () a ()? block.

Abstract data types (or ADTs) are for people, not for the computer. You (the programmer) create them to make your program easier for people to read, debug, improve, and use. On this page, you will build a point data type to clean up your U2L2-DrawShape project.
