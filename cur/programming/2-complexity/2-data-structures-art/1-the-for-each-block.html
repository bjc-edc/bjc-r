Yes, it is possible for you to define your blocks in a way that indicates what type of data they expect as inputs: a number, a list, or some other type. Some languages require the data type to be indicated. In Snap! it's an option. It is not necessary but, like assigning a color to a block, it can be a helpful reminder of what the block does and what type of input it expects. You have already seen input slots of several shapes, indicating different expected data types.

Notice in the video that there are many input types you can specify (Text, Number, Boolean, etc.) The list input type is chosen because a point is a list of two coordinates.
