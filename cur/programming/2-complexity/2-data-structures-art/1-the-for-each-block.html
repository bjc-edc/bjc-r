Connect the Dots

What students do: They build a script that takes a list of points as input and connects the dots. Useful for drawing and, later, for plotting data points.
What students learn: They think about problem decomposition in the form of tool-building, and creating and debugging an algorithm. First they build a block that goes to one point, using item of to decompose a pair of coordinates into the x and y components. Then they build a script that uses for each to go to all points, then a block that is defined by that script. In each case, they focus on the nature and use of the input.

Follow-up: I think that after this, students should use the thinking that they've done in problem 5. They should have the experience of clicking on some data points on an imported line graph (perhaps from a news article), reading the coordinates, making a list of them, and having Snap plot that line graph.

Yes, it is possible for you to define your blocks in a way that indicates what type of data they expect as inputs: a number, a list, or some other type. Some languages require the data type to be indicated. In Snap! it's an option. It is not necessary but, like assigning a color to a block, it can be a helpful reminder of what the block does and what type of input it expects. You have already seen input slots of several shapes, indicating different expected data types.
Notice in the video that there are many input types you can specify (Text, Number, Boolean, etc.) The list input type is chosen because a point is a list of two coordinates.
