On the previous page, you drew a shape based on a list of points we gave you. On this page, you will automate the process of getting coordinates by clicking on the desired points on the screen.

  3. Now go back to the Scripts tab, and choose the "Sprite" button under the stage to get back to the sprite's scripts.

Betsy: I can think of three things our program has to do:

  1. Initialize the point list variable to an empty list.
  2. Have the sprite follow the mouse.
  3. Every time the mouse is clicked, add its position to the point list, and give the user some visible feedback that the click has been processed.

Alphie: We did the second of those in Unit 1.

Betsy: Yes, and we did the other two in that "greet player" project, also in Unit 1.

Alphie: ... except for the part about "visual feedback." What did you mean by that?

Betsy: Well, the user is going to be following the outline of a shape, and clicking at turning points. That may take a while, and users get nervous if the computer doesn't seem to be paying attention. So I thought we should draw something at each of the points where the user clicks, a little circle or something.

Alphie: I have an idea! There's a block that paints a picture of the sprite itself on the stage. That arrowhead thingy will point right at the point where the user clicked.

Alphie and Betsy develop code to implement their idea.

  1. In the project file you loaded, the sprite has three scripts, one for each task in Betsy's list. Each of these scripts is incomplete.
    • In the first script, variable point list should be given an appropriate initial value.
    • In the second script, you need a way to tell the sprite to stop following the mouse. Replace the forever block with one that will stop looping when the user presses the space key.
    • In the third script, the stamp block is already in place, but you have to write the code that will add the mouse's position to the list of points. Respect the data abstraction for points.
  1. This is a good quick-and-dirty solution to the problem of entering a bunch of points into a list interactively. But it has a couple of weaknesses:
    • Even after you tell the sprite to stop following the mouse, the "when I am clicked" script is still ready to run, and so every mouse click thereafter will add points to the list that the user didn't want.
    • The list of points is always kept in the same global variable. So it's messy to create lists for several different letters.
