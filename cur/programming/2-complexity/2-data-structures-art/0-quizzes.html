  • A data type is what type of data something is (number, text string, list, etc.). For example, Betsy noticed that the data type for the second input to is a list.
  • Each programming language provides some primitive data types (data types that are built-in). For example, Snap! provides numbers, text (words and sentences), Booleans, lists, and some you haven't yet used as data. This block's pull-down menu shows all of Snap!'s primitive types:
  • An abstract data type (or ADT) is a custom data type that's meaningful to your program. It's not built into the language; you develop it as you code. For example Gamal, Betsy, and Alphie just made up a question answer abstract data type.

The word "abstract" is often used casually to mean something harder to understand or more complex, but in computer science, its meaning is almost the opposite. ADTs are things that you, the programmer, create to make your program easier for you and others to read, debug, and improve.

Traversing a list means looking at each item of the list. For each is iterative (that is, repetitive, like for), but for each traverses the list without using index numbers.

  1. Use for each to traverse your list, asking each question, checking each answer, and letting the user know if they are right or wrong on each item.
