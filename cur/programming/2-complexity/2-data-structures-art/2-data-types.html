On this page you will create a new data type, point, to make your U2L2-DrawShape code easier to read, debug, and use.
: binary sequence

Every kind of information in the computer—whether it is a number, text, list, or an instruction to perform—is stored as a sequence of 0s and 1s (a binary sequence). So, for a program to work, the computer needs to be told how to interpret any such sequence of 0s and 1s.

Just as the name suggests, a data type is what kind of data something is. Each programming language provides some primitive (meaning "built in") data types. In Snap! the primitive types are number, text string, list, and some other types you haven't yet used as data. (The is () a ()? block's pull-down menu shows all of Snap!'s primitive types.)

An abstract data type (or ADT) is a kind of data that's meaningful in your program, but not built into the language. You implement it as part of your program. Examples might be "sentence" in Unit 1 Lab 2 or "angle" in Unit 1 Lab 3. In this lab, the point is a useful ADT.

In other situations, the word "abstract" is often used to mean something harder to understand, but in computer science, its meaning is almost the opposite. ADTs are things that you, the programmer, create to make your program easier for people to read, debug, improve, and use.
