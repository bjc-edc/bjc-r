Nested functions (that is, functions used inside of other functions, like the way you used divisible? and numbers inside of keep) can be hard to read. Snap! offers another way to write code with multiple functions. You can use the function from the "Frequency Distribution Analysis" library to write your code one function at at time.

The pipe function sends the input number data through numbers (1) of () to report a list of numbers and then that list goes through keep items such that ((input number) divisible by ()?) from () to get the numbers from that list that are divisible by the input number.

Notice that the empty input slots in each function are filled by the output of the previous function (or filled by the starting data in the case of the first function):

  • The empty slot in the function is filled by input number.
  • The empty list input slot () in is filled by the output of numbers.
    • As usual, the blank input slot in the predicate for keep (that is, in ) is filled by each item of keep's input list (here, the output of numbers) so that keep can decide whether each number will be kept.
It's like a pipe of made of pieces connected together; the input number goes in one end and works through each function, producing a new output at each step.

Pipe is a higher order function; it is a function that takes functions as input. You've seen a higher order functions already: keep (in Unit 2 Lab 3).
