Una biblioteca de software es una colección de procedimientos que se pueden usar en programas.

El uso de bibliotecas simplifica el desarrollo de nuevos programas. Cuando utilizas procedimientos que ya sabes que funsscionan correctamente, reduce la cantidad de tiempo que necesitas dedicarle a la codificación, la cantidad de posibles errores que puede tener tu código y la cantidad de tu proyecto que necesitas probar.

    Exportación de bloques

    1. Abra el proyecto Snap! que tiene los bloques que quieres exportar. (En este caso, tu proyecto U2L3-Predicados).
    2. Elija "Exportar bloques..." en el menú de archivo Snap! ().
    3. Selecciona solo los bloques que deseas exportar. (En este caso, quieres , , , and .)
      Puedes deseleccionar los bloques que no necesitas exportar, o puedes hacerle clic con el botón derecho al fondo, elegir "ninguno" y luego seleccionar solo los que quieres.
      4. Snap! las bibliotecas y los proyectos Snap! se exportan como archivos XML.
    4. Haz clic en "Aceptar". Se descargará un archivo XML.
  2. Importa estos bloques a tu proyecto U2L4-BibliotecaMatemática, pruébalos, resuelva cualquier problema y guárdelo.

    Importación de bloques

    1. Abra el proyecto Snap! que tiene los bloques que quieres exportar. (En este caso, tu proyecto U2L4-BibliotecaMatemática).
    2. Importe el archivo XML arrastrando el archivo XML desde tu carpeta de descargas a la ventana de Snap!.
      También puedes importar el archivo XML seleccionando "Importar..." en el menú del archivo Snap!, ubicando el archivo XML en tu computadora y haciendole clic en "Abrir".
    3. Haz pruebas para los bloques que importaste:
      1. Encuentra los bloques importados al final de las paletas que los contienen.
      2. Hazle clic a cada bloque importado para verificar de que se ejecuta correctamente.
Estás creando una biblioteca matemática. Snap! tiene bibliotecas para muchas cosas, por ejemplo una que da acceso a diferentes partes de palabras y oraciones...

Depuración dependencias

Tu creaste el bloque par? usando otro bloque personalizado, ¿divisible por?. Si desea usar par? en otro proyecto algún día, debe exportar ambos bloques para que par? funcione correctamente.

If you ever see this red Obsolete! block in code you have imported, it means that a required block was not exported. You'll have to go back to the original project and export again being sure to select all of the blocks needed by the blocks you want.

Si alguna vez miras este bloque rojo Obsolete! en el código que has importado, significa que no se exportó un bloque requerido. Entonces vas a tener que volver al proyecto original y exportar nuevamente, asegurándote de seleccionar todos los bloques necesarios para los bloques que deseas utilizar.

: IPAs
AAP-3.D.4, AAP-3.D.5

Una interfaz de programa de aplicación (IPA) documenta lo que un programador necesita saber sobre el uso de la biblioteca: es una descripción del propósito, las entradas y las salidas de cada procedimiento (pero no sus algoritmos).

¿Qué es una IPA web?

Un tipo común de IPA es una web IPA en la que la biblioteca existe en la computadora de otra persona. por ejemplo, el IPA de las Mapas de Google describe cómo incrustar un mapa de Google en tu propio sitio web.

Una web IPA es solo una llamada a un procedimiento en otra máquina. Por ejemplo, estas son tres notaciones diferentes para la misma llamada de procedimiento, que busca el número de búsquedas de "BJC" en EE. UU.:
