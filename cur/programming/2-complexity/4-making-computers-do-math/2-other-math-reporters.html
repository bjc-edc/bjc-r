In this project, you will create mathematical blocks that combine several numbers, such as how sum from () to () combines numbers using addition:
  1. Open your U2L4-MathTools project if it isn't open already.

The triangular number function adds all of the whole numbers from 1 through the input number. So triangular number (n) reports 1 + 2 + 3 + \cdots + (n - 1) + n.

The factorial function multiplies all of the whole numbers from 1 through the input number. In other words, (n) ! reports 1 \times 2 \times 3 \times ... \times (n - 1) \times n.

To develop these procedures, you will use to get a list of the numbers to add or multiple and use the combine block to add or multiply them all together. The block takes an operation and a list as input and reports the result of combining all the elements in the input list using the input operation. For example, the expression below reports: 0 + 1 + 2.

Combine is a higher order function; it is a function that takes a function as input. You've seen two higher order functions already: keep (in Unit 2 Lab 3) and map (in Unit 3 Lab 1).

The definition for a reporter looks much like the procedure definition for a command. The procedure definition for this custom square roots reporter

would be written as either text or an image, like this:

PROCEDURE squareRoots (number)
{
    positiveRoot ← sqrt(number)
    RETURN [positiveRoot, -1 * positiveRoot]
}

In many languages (including Snap!) variables must be declared in some way (in Snap!, you can click the "Make a variable" button or use the script variables block). But the AP's language doesn't include variable declarations, so you won't see them on the exam.

Also, the procedure 
sqrt
isn't built in to the AP's language so it is written in lower case like other programmer-defined procedures.
