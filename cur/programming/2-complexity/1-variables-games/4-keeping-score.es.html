Mantener la Puntuación con variables globales

En esta página, utilizará una variable global para llevar la puntuación en su juego Clic Alonzo.

En su juego de adivinanzas de números, usó variables de secuencia de comandos para almacenar información (el valor del número secreto) que solo necesitaba una secuencia de comandos. Para mantener la puntuación en Clic Alonzo, más de un script necesitará acceder a esa información, por lo que usará una variable global.

: Variable Global

Una variable global es una variable que todos los scripts del programa pueden utilizar.

¿Cuándo se necesitan variables globales?

La mayoría de las veces, las variables de secuencia de comandos son una mejor opción porque ninguna otra parte del programa puede cambiarlas y, por lo tanto, reducen la posibilidad de errores. Pero si varias secuencias de comandos necesitan acceso a una variable o si la información de esa variable debe guardarse con su proyecto, utilice una variable global.

  1. Abra su proyecto U1L1-ClicAlonzo, juegue y revise el código para recordar cómo funciona.
  2. "U2L1-ClicAlonzo" Asegúrate de cambiar el nombre de tu proyecto con "U2" en el nombre del archivo.
    3. Si guarda un trabajo nuevo con un nombre de archivo anterior, Snap! sobrescribirá su proyecto original. Utilice "Guardar como..." desde el menú Archivo Snap! para guardar con el nuevo nombre. Cuando cargue un proyecto anterior y esté a punto de hacer un gran cambio, acostúmbrese a guardar primero, con el nuevo nombre, antes de cambiar nada.
  3. Cree una variable global llamada puntuación. Así es como.

    4. Haciendo un Variable Global

    1. Clic en la paleta Variables. (No es un bloque; no puede arrastrarlo al área de secuencias de comandos).
    2. Escriba el título de su variable. En este caso, es puntuación.
    3. Haga clic en "Aceptar."
  4. Utilice la variable puntuación para realizar un seguimiento de la puntuación del jugador:
    1. Inicialice puntuación a 0 al comienzo del juego.

      2. Establecer el valor inicial de una variable se conoce como inicialización la variable.

      AAP-1.B.1
    2. Haz que el programa cambie la puntuación en 1 cada vez que se haga clic en el sprite.
      3. (lo que significa ) se escribiría como 
      puntuación ← puntuación + 1
      o .
  5. Probar y depurar. Juega el juego lo suficiente para asegurarte de que la variable de puntuación funcione.
  6. En este momento, el juego nunca termina, pero cambiarás eso. Decide la puntuación que utilizarás para comprobar si el jugador ha ganado.
  7. Use un condicional para determinar cuándo el jugador alcanza ese puntaje. Cuando lo hacen:
    1. Haz que Alonzo deje de dar saltos.
    2. Haz que Alonzo aparezca de nuevo. (Tendrá que restablecer el efecto fantasma).
    3. Felicite al jugador por unos segundos.
  8. Probar y depurar. Tomen turnos para jugar y corrijan cualquier error.

    Sugerencia de Depuración: Visualización de Variables

    Cuando está depurando su código, puede ser útil ver el valor de una variable en diferentes puntos de su programa. Esto se hace de manera diferente para las variables locales y globales:

    • Las variables globales tienen casillas de verificación que controlan la visualización de la variable observador en el escenario (que se muestra a continuación).
    • Si se trata de una variable local (secuencia de comandos), puede usar el bloque mostrar variable dentro de la secuencia de comandos para mostrar un observador en el escenario. (y use hide variable para ocultarla de nuevo).
