En el idioma del examen, ← operador realmente hace una copia del valor que está asignando. Entonces, por ejemplo, la instrucción 
palabraSecreta ← "supercalifragilisticoespialidoso"
asigna memoria para "supercalifragilisticoespialidoso" nuevamente; la palabra ya estaba almacenada en la declaración del código, y ahora se almacena una copia en la variable 
palabraSecreta
. Este no es el caso en Snap! y la mayoría de los otros lenguajes, donde solo hay una copia que comparten la variable y el código, lo cual es una forma más eficiente para que operen los lenguajes de programación.
Esto no es cierto. En primer lugar, todo el problema de la copia surge solo para datos mutables, por lo que realmente deberíamos hablar de esto con respecto a las listas. En segundo lugar, Snap! en realidad hace una copia en el caso de una lista literal en el código, porque si el programa modifica la lista que se ha asignado a la variable, el programador ¡probablemente no quiere que se cambie el código! Si asigna a una variable el resultado con valores de lista de alguna expresión que no sea simplemente llamando a lista con elementos constantes, es cuando no hacemos una copia. Esto es demasiado complicado para que lo arregle esta noche.

: Variable Local

Una variable local se puede configurar o usar solo en el entorno en el que se define. Este término incluye entradas a procedimientos y variables creadas por el bloque para o variables de script.

En álgebra, a veces se usa una variable para algo cuyo valor aún no conoces, y el objetivo es averiguar su valor. En la programación tú decides ls valores de las variables.

AAP-1.A.1
Aprendiste sobre las variables de entrada en Unidad 1 Práctica de laboratorio 3 Página 3: Bloques con entradas.
Cuando asigna un valor a una variable, la variable contiene ese valor, no de dónde proviene. Por ejemplo, si manzanas = 2 y usted , entonces plátanos mantendrá el valor 2 y no recordará que haya venido de manzanas. Esta es la razón por que 
a ← a * 2
significa algo. (Suponga que a = 8. Primero calcule el valor de 
a * 2
, es decir, 16, y luego reemplace el valor anterior de a con 16). Hasta ahora, las únicas variables que ha usado son variables de entrada, y nunca asigna un valor a una entrada porque el valor lo da el código que lo llama. Pero una variable de script no tendrá un valor hasta que le des uno con conjuntos.
