The procedure definition for the custom polygon command

would be written as

The procedure 
turn_clockwise
isn't built in to the AP's language so it is written in lower case like other programmer-defined procedures.
PROCEDURE polygon (numberOfSides, length)
{
    REPEAT numberOfSides TIMES
    {
        MOVE_FORWARD (length)
        turn_clockwise (360 / numberOfSides)
    }
}

Notice that the hat block, , would be written as 

PROCEDURE polygon (numberOfSides, length)
. The word 
PROCEDURE
tells you that that line of the code is like a hat block; the variable names in the parentheses on that line are the inputs that the procedure takes.
