Debugging Reference Page


On this page, you will learn about the basics of debugging in Snap! and some programming principles and habits that will help you minimize bugs in your programs.

What is Debugging?

Debugging is the art of identifying errors in computer programs and finding ways of resolving them.

Debugging Principles

  • Recognize the existence of a bug
  • When you experience a bug, do not delete your project and start over.
  • Reproduce several times the error that the bug creates
  • Isolate the piece of code producing the error
  • Identify the specific cause of the error within that code
  • Debug by subtraction not addition. That is do not add code that works around the bug
  • Fix the error
  • Test thoroughly including consideration of special cases (Note: Should be move down under good programming habits; it's an anti-bugging strategy)
  • Keep a log of bugs you encounter throughout the year and add any new bugs found to this log. Include in the log any test cases that helped you identify the bugs.
  • Love your bugs! They give you insight into programming and a make you better at it!
