GPS coordinates on a map are "backwards" compared to the way Snap! plots points. In Snap!, and in mathematics, the first coordinate is plotted horizontally, and the second vertically. However, this isn't the order GPS gives.

When writing GPS coordinates, the first number is latitude, measures north-south, and the second, longitude, measures east-west. When plotting GPS coordinates, plot them as (longitude, latitude), reversing the written order.

Transform the data

You can do it your own way instead if you like.

To see the variation in the GPS coordinates, you'll need to transform them. Transforming data and information can lead to new knowledge and new means to communicate it. These problems will lead you through one way to transform the GPS coordinates.

    Subtracting -74 is the same as adding 74.
  1. Every point's longitude is around -74 and every point's latitude is around 40.7. This does not help us tell the points apart. Subtract -74 from every longitude and subtract 40.7 from every latitude, using a list operation.
  2. Replot the data after the transformation. What happens?
  3. The data is still too close together. Change your list operation so that, after subtracting, the new values are multiplied by a factor. Adjust this factor and replot until you can see the shape of the data. What does the plot tell you about the GPS information?
    Making Snap! do the work is better, because then the program can be used on any set of GPS points in this area, and not just these. That's abstraction.
