In this project you will create a timer that reports the number of seconds elapsed in both decimal and binary notations.

  1. Study the Binary Counter created in Snap! linked here until you gain some instinct how the binary system relates to the decimal system.
  2. Review the examples in the Appendix, if needed, and discuss as a class how you can convert a number in base 10 to its equivalent in base 2. Come up with a simple algorithm for this procedure.
  3. Examine the student file linked here with which you will be working. Note that 3 sprites representing the decimal digits, 8 sprites representing the binary digits, with all the necessary costumes (0-1 for binary 0-9 for decimal) and code have been created for you. Also created is a sprite representing the “Timer”. This is the only sprite for which you will need to fill in some code. Snap!’s own “timer” block (from the “Sensing” menu) will be used as a time keeper.
  • In this project you will generate the binary number representation of decimal numbers 0 through 255. While it takes 3 digits to represent 255 in base 10, it will require 8 digits in base 2. (Why?)
  • If you look at the Timer sprite code, you will see that the computed decimal and binary digits are stored in lists and then a broadcast is made to the sprites representing the digits to put on the appropriate costumes (0 and 1 for the binary digits and 0 through 9 for the decimal digits)
    • The “warp” block is needed as all the computation must be finalized prior to a screen refresh.

Note that last part of the code is used to pad the front of the number with zeros. For example, as we are allocating 3 digits to represent the numbers, the number “42” is written as “042”.
