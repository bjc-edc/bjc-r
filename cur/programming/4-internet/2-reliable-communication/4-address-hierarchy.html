6.2.2C Hierarchy in the DNS helps that system scale.

The devices on the Internet locate one another using domain names we type in (like snap.berkeley.edu) and IP addresses behind the scenes (like 128.32.189.18). Before the Internet, there were only small networks of computers (like the Arpanet which peaked around 200 computers), and every computer knew the name of all the other computers on their network. That worked for small networks, but it's not realistic for the 3 billion computers on the Internet now. So now, addresses on Internet are hierarchical to allow the system to scale.

A hierarchy is a ranking arrangement with the broadest or most powerful rank at the top and with things below ranked according to status or inclusion. It's often depicted as a pyramid. For example in biology, we use a taxonomic hierarchy: kingdom, phylum, class, order, family, genus, species. And on computers, we store files in a hierarchy of folders within folders.

Domain Name System (DNS) Hierarchy

Just as the path in a URL helps locate a specific file in a hierarchy of folders on the server, domains are named with hierarchical addresses that help an expanding system locate specific computers.

The hierarchy of the domain name system simplifies the process of finding the computer with the desired domain name, which helps the whole system make connections faster.

An example of this simplification: when you visit snap.berkeley.edu, your computer only has to know where to find an edu name server. That server only has to know where to find the berkeley.edu server. And that server directs your computer to snap.berkeley.edu.

The last segment of a domain name (like .edu) may be a country code (such as .mx for Mexico) or a category code (like .gov for government). The last two segments of a domain name (like berkeley.edu) is the domain. Additional levels of the domain name are subdomains. For example:

  • snap.berkeley.edu is a subdomain of berkeley.edu
  • store.parks.ca.gov is a subdomain of ca.gov

According to the domain name system (DNS), which of the following is a subdomain of the domain example.com?
about.example.com
Correct! about.example.com is a subdomain on the example.com domain.
example.co.uk
example.co.uk would be used for a commercial site in the United Kingdom.
example.com.org
example.com.org is not a valid address because it has two category codes (.com and .org).
example.org
example.org is a completely different domain name that would be on a .org (not .com) name server.

Internet Protocol (IP) Address Hierarchy

6.1.1G The domain name system (DNS) translates domain names to IP addresses.
6.2.1B The domain name syntax is hierarchical.
6.2.1C IP addresses are hierarchical.

When we type in a domain name, the browser queries the domain name system (DNS) to find the IP address of the server we want. Both the domain name syntax and IP addresses are hierarchical.
