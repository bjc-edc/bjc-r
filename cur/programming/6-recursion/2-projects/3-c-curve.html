Another well-known recursive shape is the Lévy C-curve. As in the snowflake algorithm, the base case is that the sprite draws a single line segment, starting on the left, going to the right, and pointing in the same direction in the beginning and at the end (to the right).

In the second level, the sprite repeats the previous level twice—first turning 45° left, then doing the previous level, then turning 90° right, then doing the previous level again, and finally turning 45° left to face the same direction again. You'll have to think about how much smaller the size input should be in the recursive calls.

  1. If the size of one level of the program is held by the variable size, what is the size passed through the recursive call to the next-lowest level? Use the Pythagorean formula: a^2 + b^2 = c^2.

In the third level, the sprite repeats the previous level twice, drawing 4 smaller lines. Notice that the 2nd and 3rd lines are in the same direction (they are collinear) so they appear to be one connected longer line.

  1. What in the algorithm makes these two lines end up drawn at the same angle?
    You may find it helpful to expand this list of commands by writing out the steps within each of the two recursive calls.
    Recall, the algorithm for each recursive level is:
    • turn 45° left
    • recursively call the previous level at the smaller size
    • turn 90° right
    • recursively call the previous level at the smaller size
    • turn 45° left
