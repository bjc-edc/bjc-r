Recursive Mondrian

Recursion is not just for fractals...

  1. Do a quick search for the Dutch painter Mondrian.

Mondrian's paintings were not created recursively, but we can use recursion to create similar images:

If you look closely at this picture, you'll see that the corners of the outside border are missing. This happens when you use a wide pen and choose "Flat line ends," as we did for this project. Don't worry about it.

  1. Load this project. It has several blocks already created and the script that calls Mondrian already set up.
  2. Begin by programming the code for your base case, the rectangle block. It will need to draw a rectangle with a black border and use a very thick line of color to paint the inside. Be sure to preserve state transparency (put the sprite back where it started).
    The rectangle block already contains two custom blocks, set pen to black line, for the borders, and set pen to paint, which runs a random set pen color block from the colors list and sets the pen width according to the thickness you determine. You can look inside these two custom blocks to see how they work. Colors is set up before the outer call to Mondrian, like this:
  3. Click "r" a few times to be sure you have rectangle set up correctly. Each time, you should get a big rectangle with one of the five colors inside and a black border, like this:

  4. Then, program the code for the recursive case. The Mondrian block already contains some structure. Your job is to fill in all the blanks. Before you get started, think carefully about how the script is going to work.

    • You've already seen level 1, a filled-in rectangle. For level 2, Mondrian will randomly divide the area specified by its width and height inputs in two pieces, and draw a rectangle in each piece. For level 3, Mondrian will randomly divide the area in two pieces, and draw a level-2 Mondrian in each piece (for a total of four filled-in rectangles). Level 4 will create eight rectangles; level 5 will create 16; and so on.
    • The fraction variable will contain a random fraction between 1/5 and 4/5, specifying how to divide up the area at this level. You'll have to use that fraction to find the scaling factors for the sizes of the two pieces.
    • The algorithm will be slightly different based on whether an area is being divided up along the width or along the height. Notice that the if-else determines whether width or height is greater. Divide each area along the longer side.
