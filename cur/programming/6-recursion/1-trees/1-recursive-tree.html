Recursive Tree

The branching pattern on many plants has what is called fractal structure: the smaller parts are like miniature copies of the whole.

A fractal is an infinite repeating pattern made up of copies (or slight variations) of the same shape. In this picture, the green tree is (essentially) the same shape as the entire picture.

In this lab, you will build a tree using blocks within blocks to create the branches. Each branch will be a smaller copy of essentially the same tree.

These blocks all look exactly the same, don't they? So it makes sense to wonder if we can replace them all with a single tree block. The differences between numbered blocks are only that tree5 uses tree4, while tree4 uses tree3, and so on. We can use that fact to write a tree block that uses tree itself in its own script, provided that we replace the numbers in the names with another input to keep track of the level.

Using a block inside of itself is called recursion.

When tree is called with level 5, it should draw the same picture as tree5, so it will draw two trees of level 4; each of them will draw two trees of level 3, and so on.
