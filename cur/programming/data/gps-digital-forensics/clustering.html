Suddenly, your crime-fighter colleagues come in and have a hunch: Maybe there was more than one perpetrator? The messages were collected in a very short time and no major movement was observed. So they ask you to confirm or deny the hypothesis that the messages come from more than one source and if so from how many.

Clustering

Clearly, all coordinates have different values. So we have to find out if certain points are closer to other points and if that's a pattern. In other words:

Can we group the points into distinct clusters?

How many clusters are best? It wouldn't add much information if each point was in its own cluster, or if we grouped all of them into a single large cluster.

In order to do clustering, we need two main processes:

  • A method to how close points are to each other. This is called a distance metric. An easy to use metric is the Euclidean Distance d, which is accurate for XY coordinates on flat surfaces (as a disclaimer, GPS coordiates are not used on a flat surface—i.e., the Earth! But, for coordinates all in a small area, Euclidean distance does good enough).

  • A method to determine if a point belongs to a cluster or not. A cluster is a set of points, and we'll use our distance metric to compare the point in question (lets call it p1outside?) with the points that are in the cluster. That will give you a list of distances (as many as there are points in the cluster), so you'll need to use that list to figure out if p1outside? should be considered a member of the cluster.

    You can do this in various ways; let's choose an easy one: figure out if the minimum distance in your list is less than some predetermined value. What this means, then, is that p1outside? belongs to a cluster if the distance between it and the closet point in the cluster is "close enough".
