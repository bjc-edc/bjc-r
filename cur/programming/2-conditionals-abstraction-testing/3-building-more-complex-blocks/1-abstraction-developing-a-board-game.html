Abstraction: developing a board game

  2. Test it to make sure that it can make 4x4 arrays and 5x5 arrays, and that the size of the tiles can be controlled.

  3. Build the beginnings of a game.
    1. Set your sprite to (-150, 150), clear the stage, and run .
    2. Build a reporter where? that uses your between? block to report 5 if the sprite is dragged into the center cell and 0 if it is not in the center cell.
    3. Click on Make a variable and name it which tile?.

      That will automatically give you a watcher to display its value on the stage.

    4. Build a block display the tile that uses where? to set the variable which tile?. Use forever to make it keep reporting (and setting which tile?) as you drag the sprite around the stage.
    5. Extend where? to report 0 only if the sprite is dragged entirely off the game board. Run display the tile to test your improvements to where?.
    6. Now improve where? further, so that it reports 1 for the top left tile, 2 for the top middle tile, 3 for the top right tile, and so on.
    4. Notice that even though display the tile is the block that says what tile you are in, you don't have to change it at all when you improve where?. In a similar way, you will be able to improve the way that display the tile displays the information without changing the details of where?. Building separate blocks to specialize in separate functions is extremely valuable for clear, bug-free, improvable code.
