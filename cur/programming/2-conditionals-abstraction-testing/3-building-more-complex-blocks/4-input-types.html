Input Types

The max block has an unexpected behavior. It is intended to work only for numbers, yet you can type text into it!

You have already used to check for situations like this, but you can also define blocks in a way that allows only the appropriate input.

The words "input" and "argument" are often used interchangeably. Nobody is being disagreeable!
You can specify the input type that should be used in a block. For example, you can limit the max block to accept only numbers as arguments.

  1. Open the Block Editor for the max block and click on one of the input names (e.g., x). You whould get this pop-up box

  2. Then, click on the right-arrow in the pop-up box. The dialog box (shown below) lets you choose the input type you want. For this block, you want a numbers-only slot (shown selected below). As you can see, the shape of the slot tells people what type of input the block accepts.

    You can also specify a default value for the variable if you like. This is how the move block was defined always to start as with the default value 10, and not just with no default value specified.

  3. Modify both the x and y variables to take only numbers and to have the default values of 5 and 10, respectively. Your Block Editor should then look like this, with the default values shown in the header:

    Notice that the variable names are still x and y, even though the title line for the block looks different. It uses # to indicate that the input must be a number and it shows the default value that will be used.

    When you click OK, your block should look like this in the Operators tab and we can no longer type text as input.

    4. When an input type is restricted, a block can still receive improper values. Snap! doesn’t constantly check to see what types are passed as values and indicate when those do not match a block’s input type. Specifying the input type is an organizational tool to remind you what the domain (input values) of your blocks should be.

    When you are writing a complicated program that requires the user to input information—a number, their name, or something else—you will often want to check to make sure their information is the right kind, so that your program works properly.
