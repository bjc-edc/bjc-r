Magic 8 Ball: An Introduction to Storing Data in Lists


In the “Diary of a Wimpy Kid” series book titled “Hard Luck”, the hilarious main character, Gregg Heffley, decides to use a toy named Magic 8 Ball® to determine what to do about the tough choices he has to make in his life, such as whether to do laundry or clean up his room. In this project, you will build a program inspired by the Magic 8 Ball® to help you deal with the uncertainties of the future.
  1. Open the file linked here and play with the simulation to get a feel for the toy what you are trying to model.
  2. The possible answers that the toy gives to a YES/NO question is:
       Outlook uncertain  
       Probably yes
       No way
       It is certain
       Not likely
       Reply hazy
       May be
       Without a doubt

    What kind of a data structure would you use in your code to store and retrieve the answer choices?
  3. Open and examine this starter project file where two costumes and a sound file have already been created for you.
    4. If you get stuck, you can look at this page for some more information.
  4. Write code to model the toy. Feel free to change any aspects of the project to personalize the project and fit your creative needs. For example, consider changing the predictions of the ball; use new images for the costumes and may be even add another sound file.

  1. Trick your friends to think that your model of the Magic 8 Ball ® actually knows things and tells the truth. Create a version of your model that allows you to select among a certain type of choices when a particular key is secretly pressed. For example, when the key “1” is pressed only affirmative answers are chosen, when the key “2” is pressed only non-committal answers are chosen, when the key “3” is pressed only negative answers are chosen.
