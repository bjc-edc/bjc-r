On the last page, you used script variables to keep track of temporary information (the value of secret number) that was only needed inside one script. To keep score, you need to be able to access the current score from anywhere in your app, and so you'll use a global variable.

It's good programming practice to always use script variables (also known as local variables) unless you need the variable to be accessible to the whole program, in which case you'll use a global variable.
  1. Open your U1L1-ClickAlonzo project, play the game, and review the code so you remember how it works.
  2. "U2L1-ClickAlonzo"
  3. Right now, the game never ends, but now you will change that. Create a global variable named score. (Instructions are below.)

    4. Making a Global Variable

    1. Choose from the Variables palette.
    2. Type the title for your variable: score, and click OK.
  4. Use the score variable to keep track of the player's score:
    1. Initialize score to 0 at the beginning of the game.
    2. Make the program change the score by 1 whenever the sprite is clicked.
    3. If the sprite moves without being clicked, make the score go down by 1.
  5. Debug. Test the game a few times to make sure that the score variable does each of the three things described above.

Variables look like reporters, and both do report values, but they are different:

  • Variables always give the same value until the value is changed by the user or programmer.
  • Reporters may report different output but always by same process until the process is changed by the programmer.

Reporters always have the same behavior as was coded in at the beginning. They can't be changed by the user of the app; they can only be changed by a programmer changing the code. Variables, the other hand, can change while the program is running.

  1. Determine the score that you will use to test whether the player has won.
  2. Use a conditional to determine when the player reaches that score, and when they do:
    1. Make Alonzo appear again. (You'll have to reset the ghost effect.)
    2. Congratulate the player for a few seconds.
    3. Stop all scripts from running to end the game.
  3. Debug. Take turns testing the game, and fix any bugs.
