Developing a Number Guessing Game with Script Variables

In this lab, you will use two kinds of variables to manage data in your games:

  • Script variables that are temporary and end when the script ends, and
  • Global variables that can be accessed anywhere in the app.

On this page, you will develop a number guessing game that uses script variables to keep track of a secret number.

Using a Script Variable

A script variable exists only while the script is running.
Sometimes you will need extra variables to store information temporarily while your script is running. The script variables block lets you do that.

For example, imagine a number-guessing game in which the player tries to guess the computer's secret number. The computer needs to remember the secret number so that is can compare the player's guesses to it, but once the game is over, the secret number does not need to be remembered.

  1. "U2L1-NumberGuessing"
  2. Create a new block called Number guessing game. (You might choose to put it in the Looks palette since there will be a lot of conversation using the say block.)
  3. Inside this new block, create a script variable called secret number and set it to a random number between 1 and 10.

    Making a Script Variable

    1. Move a script variables block into the Scripting Area. You can find it in the Variables palette.
    2. Name the variable by clicking on the orange a and typing the name you want. Here, it should be called secret number.
    3. Set the initial value of the variable using set. The down-arrow lets you select the variable you want. Set secret number to a random number between 1 and 10.

    Setting the initial value of a variable is known as initializing the variable.

Checking the Player's Guess

Now the computer should ask the player to guess the secret number again and again until the player gets it right. You will use the repeat until block, which is a loop (just like repeat, forever, and for), but it only repeats until a certain condition is met. In this case, it should repeat until the player's answer is the secret number.

Repeat until is a conditional block just like if. Conditionals control the code based on a condition called a predicate. Predicates are hexagonal-shaped reporters that always report a Boolean value (that is, they can only report either or ). Predicates help conditionals (such as or ) decide when to do something.

  1. Use repeat until to ask the player to guess the secret number until their answer equals the secret number.
  2. After the players guesses the secret number, make the computer congratulate the player.
  3. Debug. Take turns playing the game, and fix any problems with the code before moving on.
    Use join to merge the text "You guessed it! My secret number was" with the value of the secret number variable.
  1. When the computer congratulates the player for guessing correctly, have the computer say the number. For example, it might say, "You guessed it! My secret number was 7."
  1. Use more conditional blocks and predicates to make the computer tell the player whether their guess is too big or too small: "That's too big. Try again."
    How should these conditionals work and where in the code should they go?
  2. Debug. Play your game a few more times, and check that it performs as expected:
    • The game should ask the player to guess its secret number.
    • The game should say whether the answer was too big or too small or congratulate the player if they guess the number.

Since script variables only exist while the script is running, the secret number variable will not exist when Number guessing game is not running. When the game is played again, the program will create the variable again and initialize it with a different random number.
