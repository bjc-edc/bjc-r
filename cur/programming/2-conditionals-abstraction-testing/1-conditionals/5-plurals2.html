In this lab, you will continue to improve your suffix s block so that it works correctly with more words. If you do not already have it, please reload it. (The suggested file name was U2Lab1-Plural.)

  1. Organize your list of words that plural gets wrong. Organize those words into categories according to their last letter. For example, some of the words it gets wrong will be words that end with the letter h. Try to find others it gets wrong. You will probably find at least two more categories.
  2. The idea of building a separate block to handle one special case and then incorporating that block in the bigger procedure isn't necessary for very simple cases (like "ends with x") that you test for only once. In more complex cases, like "ends with h," building your code in modules is extremely helpful in keeping your code clear and helping you debug it.
    Make a "specialist" block that is designed to work for just one category, the words that end with the letter h.
    You might name that block . This specialist block can be completely wrong about words that don't fit its specialty ("case" does not end in h), but it should work correctly for words that do fit its specialty . Test it for a variety of words to make sure it works the way you want.
  3. When you trust your new specialist block, edit the original suffix s block to use your new specialist. Test the revised suffix s block to make sure it still works for all the words it used to work for, and also the new ones.
